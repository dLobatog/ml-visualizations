Key Insight: Neural networks learn hierarchical feature representations through backpropagation.
Architecture: Input → Hidden layers (learn features) → Output. Each layer builds on lower layer representations.
$$\text{Forward pass: } h_l = \sigma(W_l \cdot h_{l-1} + b_l)$$
\(h_l\):Hidden layer output/activation at layer l. Shape: (batch_size, layer_width)
\(W_l\):Weight matrix for layer l. Shape: (input_dim, output_dim). Learned during training.
\(h_{l-1}\):Input to layer l (output from previous layer l-1). At layer 0, this is the raw input x.
\(b_l\):Bias vector for layer l. Shape: (output_dim,). Shifted learned parameter, one per output unit.
\(W_l \cdot h_{l-1} + b_l\):Linear transformation (affine). Without \(\sigma\) would just be linear combinations, stack of linear layers = one linear layer.
\(\sigma\):Activation function—introduces non-linearity. Examples: ReLU (max(0,x)), tanh, sigmoid. Critical: without it, network is just linear.
Backpropagation: Compute \(\frac{\partial L}{\partial W}\) by chain rule. Gradients flow backward. Updates all weights to reduce loss.
Activation function \(\sigma\): Introduces non-linearity. Without it, network is just linear transformation. ReLU, tanh, sigmoid common choices.
Why deep networks? Deeper networks learn more abstract representations. Trade-off: harder to train, need more data.
Neural network = function composition. Each layer learns a transformation. Depth = abstraction hierarchy.

Activation Functions Compared

Toggle between Sigmoid, ReLU, and Tanh. Watch the derivative plots (bottom row) — Sigmoid has near-zero derivatives at extremes (vanishing gradients!). ReLU has constant 1 in the positive region (stable gradients). Flat regions = gradient death.

Key Insight: Backpropagation efficiently computes gradients for all weights using the chain rule.
$$\frac{\partial L}{\partial W} = \frac{\partial L}{\partial \text{output}} \cdot \frac{\partial \text{output}}{\partial W}$$
\(\frac{\partial L}{\partial W}\):Gradient of loss w.r.t. weights. Tells how much each weight affects final loss. Used to update weights.
\(\frac{\partial L}{\partial \text{output}}\):Gradient of loss w.r.t. final output. How much the output contributes to loss.
\(\frac{\partial \text{output}}{\partial W}\):Gradient of output w.r.t. weights. How much changing W changes the output.
Chain rule (\(\cdot\)):Multiply gradients along the computation path. Example: if loss depends on output, and output depends on W, then \(\frac{\partial L}{\partial W} = \frac{\partial L}{\partial \text{output}} \times \frac{\partial \text{output}}{\partial W}\)
Chain rule in networks: Error at output flows backward through layers. Each layer contributes to gradient via local \(\frac{\partial h}{\partial W}\).
Computational complexity: One forward pass + one backward pass. Backward cost ≈ 2× forward. Much cheaper than numerical gradients.
Vanishing gradients: Gradients multiply as they flow backward. With small derivatives (<0.1), deep layers get tiny gradients. Training stalls.
Exploding gradients: Gradients multiply, grow exponentially. Weights become NaN/Inf. Solution: gradient clipping.
Use ReLU instead of sigmoid/tanh to avoid vanishing gradients. Use batch normalization to stabilize gradient flow.
Backpropagation = chain rule applied recursively. Efficient but suffers from vanishing/exploding gradients in deep networks.
👉 Try this: Click Sigmoid. At extremes (x=-10, x=10), derivative ≈ 0 (white/flat). In the middle (x=0), derivative = 0.25 (steepest). Switch to ReLU — derivative is constant 1 for x>0 (no vanishing!). This is why ReLU enabled deep networks.

Vanishing & Exploding Gradients

Key Insight: Gradients vanish or explode because backprop multiplies derivatives through every layer via the chain rule.
$$\frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial h_n} \times \frac{\partial h_n}{\partial h_{n-1}} \times \cdots \times \frac{\partial h_2}{\partial h_1} \times \frac{\partial h_1}{\partial W_1}$$
The problem:Each \(\frac{\partial h_k}{\partial h_{k-1}}\) factor includes the activation derivative. For sigmoid, this is at most 0.25. Multiply 50 factors of 0.25 → gradient \(\approx 10^{-30}\) (vanished). For weight scale > 1, gradients explode exponentially.
Why depth matters:More layers = more multiplied factors. A 5-layer network with sigmoid barely trains. A 50-layer network with sigmoid is dead.
ReLU helps:ReLU derivative = 1 for positive inputs (no shrinkage!). But "dead ReLU" problem: if input is always negative, gradient = 0 permanently.
Residual connections fix it:y = F(x) + x. Gradient of skip connection = 1. Even if F's gradient vanishes, the gradient flows through the skip. This is why ResNets can be 100+ layers deep.
"Why can't we train a 50-layer sigmoid network?" → Chain rule multiplies derivatives. Sigmoid max derivative = 0.25. After 50 layers: 0.25⁵⁰ ≈ 10⁻³⁰. Gradient is effectively zero. Solution: ReLU + residual connections + careful initialization (He/Xavier).
Vanishing = multiplying numbers < 1 repeatedly. Exploding = multiplying numbers > 1 repeatedly. Residual connections = adding a highway that always carries gradient = 1.
👉 Try this: Set depth to 5 with Sigmoid — gradients are okay. Now drag to 25 — watch them collapse. Switch to ReLU — they stabilize. Switch to Residual — rock solid even at depth 30.

Key Insight: Convolution is template matching — a small kernel slides across the input, computing local dot products to detect features (edges, textures, shapes).
$$\text{output}[i,j] = \sum_a \sum_b \text{kernel}[a,b] \times \text{input}[i+a, j+b] + \text{bias}$$
kernel[a,b]:Small learnable weight matrix (e.g., 3×3). Each kernel detects one pattern — vertical edges, horizontal edges, corners, etc.
input[i+a, j+b]:Local patch of input centered at position (i,j). Convolution exploits spatial locality — nearby pixels are more related.
Stride:How far the kernel moves each step. Stride 1 = overlap (detailed features). Stride 2 = skip pixels (downsample, fewer parameters).
Padding:Add zeros around input border. "Same" padding preserves spatial size. "Valid" = no padding, output shrinks.
Output size:(input_size - kernel_size + 2×padding) / stride + 1. E.g., 32×32 input, 3×3 kernel, stride 1, padding 1 → 32×32 output.
Why dropout works: Prevents neurons from co-adapting (relying on specific other neurons). Forces network to learn robust features.
Effect: Trains ensemble of ~2^n subnetworks. At test time, average them (or use all units with scaling).
Dropout rate p: Typical: 0.5 for hidden layers, 0.1-0.2 for input. Higher p = stronger regularization but risk underfitting.
At test time: No dropout! Use all activations. Scaling: multiply by (1-p) during training, OR divide by (1-p) at test (depends on implementation).
Dropout is a simple, powerful regularizer. Less important with batch norm, but still useful together. Monitor train vs test error.
Dropout = train multiple nets simultaneously, average at test time. Prevents co-adaptation. Ensemble effect = better generalization.

Normalization: Batch vs Layer vs RMS

Toggle between Batch Norm, Layer Norm, and RMSNorm. For each, the colored region shows which activations get normalized together. Batch Norm normalizes across batch (column), Layer Norm across features (row). Notice: Batch Norm behavior depends on batch size!

Key Insight: Convolutional layers learn local spatial patterns via weight sharing and local connections.
$$\text{Conv output: } y[i,j] = \sum w[a,b] \cdot x[i+a, j+b] + b$$
y[i,j]:Output activation at spatial position (i,j). Shape across all filters: (height_out, width_out, num_filters)
w[a,b]:Filter/kernel weights at local offset (a,b). Learned parameter. Size: (kernel_h, kernel_w). Same for all positions (weight sharing).
x[i+a, j+b]:Input values in the receptive field—rectangular region around position (i,j). Sliding window centered at (i,j).
\(\sum w[a,b] \cdot x[i{+}a, j{+}b]\):Dot product of filter and local patch. Computes correlation/match between filter and local region.
+ b:Bias term—learned scalar added to dot product. Same bias for all spatial positions.
Spatial locality:Only uses values in small receptive field around (i,j), not entire image. Efficient and captures local features.
Weight sharing: Same filter applied at every position. Much fewer parameters than fully connected. Captures translation invariance.
Spatial hierarchy: Early layers: edge, texture detectors. Middle: shape, part detectors. Deep: object detectors. Learns hierarchical representations.
Hyperparameters: Filter size (3×3, 5×5), stride (step between positions), padding (border handling), number of filters (output channels).
Pooling: Max/average pooling reduces spatial dimensions. Provides small translation invariance. Reduces parameters.
CNNs are the backbone of computer vision. Receptive field grows with depth. Understand how filter size and stride affect output dimensions.
CNN = local pattern matching with weight sharing. Captures spatial hierarchies. Deep = large receptive fields = global context.
👉 Try this: Select Batch Norm. Colored region highlights entire column (normalize across batch). Batch size = 32 activations, all scaled together. Switch to Layer Norm — colored region is entire row (normalize across features for each sample independently). RMSNorm normalizes only by variance, not mean.

RNN/LSTM/GRU: Recurrence & Gating

Toggle between Vanilla RNN, LSTM, and GRU. For Vanilla RNN, trace the hidden state flow and see it accumulate (or vanish) through timesteps. For LSTM, watch the cell state bypass (straight arrow) — this is the "memory" that avoids vanishing gradients. Gates control what flows.

5
Key Insight: Recurrent networks process sequences by maintaining hidden state that captures context from previous steps.
$$h_t = \sigma(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b_h)$$
\(h_t\):Hidden state at timestep t. Acts as memory. Shape: (batch_size, hidden_dim). Updated every step based on current input and previous state.
\(h_{t-1}\):Hidden state from previous timestep t-1. Carries information forward in sequence.
\(x_t\):Input at timestep t (e.g., word embedding, single token). Shape: (batch_size, input_dim)
\(W_{hh}\):Recurrent weight matrix (hidden-to-hidden). Shared across all timesteps. Shape: (hidden_dim, hidden_dim)
\(W_{xh}\):Input-to-hidden weight matrix. Shape: (input_dim, hidden_dim). Also shared across timesteps.
\(b_h\):Bias vector for hidden state. Shape: (hidden_dim,)
\(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b_h\):Linear combination of previous hidden state and current input. Combines memory with new information.
\(\sigma(\ldots)\):Activation function (tanh common for RNNs). Bounds hidden state, typically to (-1,1) range.
Hidden state h_t: Memory that carries information from previous positions. Initialized to zero, updated at each step.
Sequence processing: At each timestep, network sees current input + hidden state from previous. Learns dependencies across sequence.
Vanishing gradient problem: Gradients flow back through many timesteps. Same multiplication of derivatives (<0.1) causes vanishing gradients.
LSTM/GRU solutions: Gated mechanisms control information flow. Cell state = long-term memory. Much better at learning long-range dependencies.
LSTMs widely used in practice. But Transformers (attention) often better for many tasks now. Sequence length matters: quadratic memory for attention.
RNN = process sequence step-by-step, carrying memory. Long dependencies = vanishing gradients. LSTM = gates solve this via cell state.
👉 Try this: Vanilla RNN, 20 timesteps: hidden state accumulates or decays rapidly (recurrent arrow shows values exploding or vanishing). LSTM, same setup: hidden state flows through cell state (long straight arrow bypasses gates), gradients survive 20 steps. GRU: simpler than LSTM but similar solution.

Flash Attention: Tiled IO-Aware Computation

Watch the "Standard Attention" animation — it materializes the full N×N attention matrix (memory bottleneck!). Toggle to "Flash Attention" — computation is tiled into blocks (no full matrix in memory). Despite the same result, tiling reduces slow memory access (HBM) dramatically.

Key Insight: ResNets add skip connections that allow direct gradient flow and enable training of very deep networks.
$$y = F(x) + \color{#ff6b6b}{x} \quad \text{(instead of just } y = F(x)\text{)}$$
x:Input to residual block. Shape: (batch_size, feature_dim)
F(x):Transformation by residual block—typically 2-3 layers (conv, norm, activation). Learns feature changes.
+ x:Skip/shortcut connection. Identity mapping directly adds input to output. Enables information to bypass block.
y = F(x) + x:Output is residual transformation F(x) PLUS identity. Network learns F(x) = (desired_output - x), i.e., the residual/difference.
Dimension requirement:F(x) and x must have same shape for addition. If different, use 1×1 conv to match dimensions.
Skip connection benefit: Gradients bypass blocks, flow directly through identity connection. Solves vanishing gradient for deep networks.
Training depth: Without residuals: hard to train networks >50 layers. With residuals: can train 150+ layers easily.
Residual block interpretation: F(x) learns the residual (difference from identity). Network learns what to add/change, not entire function from scratch.
Dimensions matching: If F changes dimensions (stride or filters), use 1×1 projection convolution for x. Or zero-pad/subsample x.
ResNets revolutionized deep learning. Skip connections are now standard everywhere (Transformers, dense nets, etc.).
ResNet = learn residuals, not full function. Skip connection = highway for gradients. Enables very deep networks.
👉 Try this: Sequence length = 100. Standard Attention materializes 100×100 matrix (10K values in slow memory). Flash Attention: tiles into 4×4 blocks (only 25 values in fast memory at a time). Result is identical, but memory bandwidth drops 4-10x. This is why Flash Attention made long sequences practical.

Key Insight: Attention mechanisms compute weighted combinations of values based on query-key similarities, enabling long-range dependencies.
$$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
\(Q\) (Query):Matrix of query vectors, shape \((\text{seq\_len}, d_k)\). "What am I looking for?" Linear projection of input.
\(K\) (Key):Matrix of key vectors, shape \((\text{seq\_len}, d_k)\). "What does each position represent?" Compared against queries.
\(V\) (Value):Matrix of value vectors, shape \((\text{seq\_len}, d_v)\). "What to output?" Weighted by attention scores.
\(QK^T\):Dot product of queries and keys. Shape: (seq_len, seq_len). Measures similarity between each query and all keys. Unnormalized attention scores.
\(\sqrt{d_k}\):Scaling factor. \(d_k\) = dimension of keys (usually 64). Prevents attention scores from becoming too large (softmax gradient issues).
\(\text{softmax}(QK^T/\sqrt{d_k})\):Convert scaled scores to probability distribution over sequence positions. Each query sums to 1 across all keys. Attention weights.
\(\text{softmax}(\ldots) \cdot V\):Weighted sum of values. Each value is weighted by its corresponding attention score. Output shape: \((\text{seq\_len}, d_v)\)
Query, Key, Value: Q = what we're looking for. K = what each position represents. V = what to output. Learned linear projections of input.
Attention weights: Softmax(QK^T) = probability of attending to each position. Learned during training based on query-key similarity.
Multi-head attention: Multiple attention heads attend to different aspects. Concatenate outputs. Richer representations than single attention.
Computational cost: Attention is \(O(n^2)\) in sequence length. For long sequences, use approximations: local attention, sparse attention, linear attention.
Attention is core of Transformers. Self-attention: each position attends to all positions. Enables parallelization unlike RNNs.
Attention = soft selection of values based on relevance. Queries ask questions, keys match queries, values are answers.
👉 Try this: Move compute budget left (favor parameters, 80/20 split) — loss plateaus (data bottleneck). Move right (80/20 data favoring) — loss plateaus (parameter bottleneck). The diagonal (50/50 split) is optimal. Chinchilla discovered this: equal compute to parameters and data.

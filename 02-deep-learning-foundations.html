What Is a Neuron?

A neuron is a tiny decision-maker. Each weight says how much it cares about that input. The bias shifts the threshold. ReLU says: if the sum is positive, pass it through; if negative, output zero.

Key Concepts:
Weights (w): Control how much each input matters. Large positive weight = strong positive influence. Negative weight = inhibitory.
Bias (b): Shifts the activation threshold. A large negative bias makes the neuron harder to activate; positive bias makes it easier.
ReLU activation: ReLU(z) = max(0, z). If the weighted sum is positive, pass it through unchanged. If negative, output zero. This introduces non-linearity.
$$\text{output} = \text{ReLU}(w_1 \cdot x_1 + w_2 \cdot x_2 + b) = \max(0,\; z)$$
A single neuron computes a linear function followed by a non-linearity. Without the activation function, stacking neurons would still be linear.
💡 Real-world: Every neural network — from a spam filter to GPT-4 — is built from this basic unit. Understanding one neuron means understanding the building block of all deep learning.

From Neuron to Decision Boundary

Key Insight: One neuron = one straight line. That is why we need layers of neurons — each adds another line, and together they carve complex shapes. This is the key insight behind deep learning.
$$\text{Decision boundary: } w_1 x + w_2 y + b = 0$$
Linear separability: A single neuron can only classify data that is separable by a straight line (or hyperplane in higher dimensions).
XOR problem: XOR requires at least 2 neurons (a hidden layer) because no single line can separate the classes. This was the key insight that motivated multi-layer networks.
The XOR problem was historically important: Minsky & Papert (1969) showed perceptrons could not solve XOR, which led to the first AI winter. The solution? Hidden layers.

In the previous section, you moved the decision boundary by hand with sliders. But how does the network move it automatically? It compares its prediction against a known answer (a "label"), measures the error, and nudges the weights to reduce that error. Part B: Two Neurons Chained — The Birth of Backpropagation

Now chain two neurons. The first one is "hidden" — it doesn't touch the output directly. How do we fix its weight?

Click Train Step to see how errors flow backward through two neurons.
0.10
That's backpropagation. The error flows backward through the network — each neuron receives the error from the neuron it feeds into, scaled by the connecting weight. Then it uses that to update its own weight. The "chain rule" is just multiplying these error signals along the chain.
$$\frac{\partial L}{\partial w_1} = \underbrace{\frac{\partial L}{\partial \text{out}}}_{\text{error at output}} \times \underbrace{\frac{\partial \text{out}}{\partial h}}_{= w_2 \text{ (the connecting weight!)}} \times \underbrace{\frac{\partial h}{\partial w_1}}_{= \text{ReLU}'(z_1) \cdot x}$$
Backprop is just the chain rule applied systematically. The key insight: each node only needs its local gradient and the upstream error signal. No node needs to "see" the whole network.
💡 Real-world: This weight update rule is running right now on thousands of GPUs across Google, Meta, and OpenAI. A single training run for a frontier model performs this operation quadrillions of times. The math is simple — the scale is what makes it expensive ($100M+ per run).

Key Insight: Neural networks learn hierarchical feature representations through backpropagation.
Architecture: Input → Hidden layers (learn features) → Output. Each layer builds on lower layer representations.
$$\text{Forward pass: } h_l = \sigma(W_l \cdot h_{l-1} + b_l)$$
\(h_l\):Hidden layer output/activation at layer l. Shape: (batch_size, layer_width)
\(W_l\):Weight matrix for layer l. Shape: (input_dim, output_dim). Learned during training.
\(h_{l-1}\):Input to layer l (output from previous layer l-1). At layer 0, this is the raw input x.
\(b_l\):Bias vector for layer l. Shape: (output_dim,). Shifted learned parameter, one per output unit.
\(W_l \cdot h_{l-1} + b_l\):Linear transformation (affine). Without \(\sigma\) would just be linear combinations, stack of linear layers = one linear layer.
\(\sigma\):Activation function—introduces non-linearity. Examples: ReLU (max(0,x)), tanh, sigmoid. Critical: without it, network is just linear.
Backpropagation: Compute \(\frac{\partial L}{\partial W}\) by chain rule. Gradients flow backward. Updates all weights to reduce loss.
Activation function \(\sigma\): Introduces non-linearity. Without it, network is just linear transformation. ReLU, tanh, sigmoid common choices.
When to Use
Shallow Networks (1-3 layers)
Tabular data, simple regression/classification where features are already meaningful. Faster to train, less prone to overfitting on small datasets.
Examples: Credit scoring, click-through prediction, structured data with <100 features.
Deep Networks (5-50 layers)
Tasks requiring hierarchical feature learning from raw inputs. Need enough data to justify depth. Use residual connections for 20+ layers.
Examples: Image classification (ResNet), speech recognition, NLP embeddings, medical imaging.
Very Deep / Foundation Models (50+ layers)
Large-scale pretraining on massive datasets. Requires careful initialization, normalization, and residual connections. Billions of parameters.
Examples: GPT/LLaMA (Transformers), Vision Transformers (ViT), diffusion models for image generation.
Why deep networks? Deeper networks learn more abstract representations. Trade-off: harder to train, need more data.
Neural network = function composition. Each layer learns a transformation. Depth = abstraction hierarchy.

Activation Functions Compared

Toggle between Sigmoid, ReLU, and Tanh. Watch the derivative plots (bottom row) — Sigmoid has near-zero derivatives at extremes (vanishing gradients!). ReLU has constant 1 in the positive region (stable gradients). Flat regions = gradient death.

Key Insight: Activation functions introduce nonlinearity — without them, stacking layers would just be matrix multiplication (a deep network would collapse to a single linear layer).
$$\text{output} = \sigma(Wx + b)$$
\(\sigma\):The activation function. Applied element-wise after the linear transform. This is what makes neural networks able to learn curved, complex decision boundaries.
ReLU: max(0, x):Zero for negative inputs, identity for positive. Derivative is 1 (positive) or 0 (negative). Simple, fast, and avoids vanishing gradients. The default for hidden layers.
Sigmoid: 1/(1+e⁻ˣ):Squashes to (0,1). Max derivative is only 0.25 — after 10 layers, gradients shrink to 0.25¹⁰ ≈ 0.000001. First layers stop learning. Use only for output gates.
GELU / SiLU:Smooth approximations of ReLU with non-zero gradients for slightly negative inputs. GELU is standard in transformers (GPT, BERT). SiLU/Swish in LLaMA, EfficientNet.
Tanh:Zero-centered output (-1 to 1). Better than sigmoid for hidden layers but still saturates at extremes. Used in RNN/LSTM hidden states.
Why the derivative matters: During backprop, gradients multiply through each layer's activation derivative. If derivatives are small (sigmoid), gradients vanish. If derivatives are ~1 (ReLU), gradients flow unchanged. This is why activation choice determines whether deep networks can train at all.
Dead neurons: ReLU outputs exactly 0 for negative inputs. If a neuron's weights shift so all inputs are negative, it never activates again — it's "dead." Leaky ReLU (small slope for negatives) prevents this.
When to Use
ReLU / Leaky ReLU
Default choice for hidden layers in most networks. Fast to compute, avoids vanishing gradients. Use Leaky ReLU if you encounter dead neurons.
Examples: CNNs (ResNet, VGG), MLPs, any deep feedforward network. Leaky ReLU for GANs.
GELU / SiLU (Swish)
Smooth approximations of ReLU used in modern Transformers. Slightly better performance due to non-zero gradients for negative inputs. GELU is standard in BERT/GPT.
Examples: GELU in Transformers (BERT, GPT, ViT). SiLU/Swish in EfficientNet, LLaMA.
Sigmoid
Output layer only for binary classification (outputs probability 0-1). Never use in hidden layers of deep networks (vanishing gradients). Also used in gates (LSTM, attention).
Examples: Binary classifiers, LSTM gate activations, logistic regression output.
Tanh
Zero-centered output (-1 to 1). Used in RNN hidden states and when you need bounded, centered outputs. Better than sigmoid for hidden layers but still suffers from saturation.
Examples: RNN/LSTM hidden state activations, actor outputs in reinforcement learning (bounded actions).
Use ReLU instead of sigmoid/tanh to avoid vanishing gradients. Use batch normalization to stabilize gradient flow.
Backpropagation = chain rule applied recursively. Efficient but suffers from vanishing/exploding gradients in deep networks.

Vanishing & Exploding Gradients

Set "Network Depth" to 2 and observe gradient magnitudes across layers (the color bars). Increase depth to 20 with Sigmoid — watch gradients fade to white at early layers (vanishing!). Switch to ReLU — gradients stay colorful. Adjust learning rate to see explosion.

10
0.80
Key Insight: Gradients vanish or explode because backprop multiplies derivatives through every layer via the chain rule.
$$\frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial h_n} \times \frac{\partial h_n}{\partial h_{n-1}} \times \cdots \times \frac{\partial h_2}{\partial h_1} \times \frac{\partial h_1}{\partial W_1}$$
The problem:Each \(\frac{\partial h_k}{\partial h_{k-1}}\) factor includes the activation derivative. For sigmoid, this is at most 0.25. Multiply 50 factors of 0.25 → gradient \(\approx 10^{-30}\) (vanished). For weight scale > 1, gradients explode exponentially.
Why depth matters:More layers = more multiplied factors. A 5-layer network with sigmoid barely trains. A 50-layer network with sigmoid is dead.
ReLU helps:ReLU derivative = 1 for positive inputs (no shrinkage!). But "dead ReLU" problem: if input is always negative, gradient = 0 permanently.
Residual connections fix it:y = F(x) + x. Gradient of skip connection = 1. Even if F's gradient vanishes, the gradient flows through the skip. This is why ResNets can be 100+ layers deep.
When to Use
Gradient Clipping
When gradients explode (loss spikes, NaN weights). Clips gradient norm to a max value. Essential for RNNs and Transformer training. Does not help vanishing gradients.
Examples: RNN/LSTM training (clip at 1.0-5.0), LLM pretraining (clip at 1.0), any unstable training run.
Residual Connections
When building networks deeper than ~10 layers. Skip connections add identity path so gradient = 1 minimum. Enables 100+ layer networks. No downsides -- always use in deep nets.
Examples: ResNet, Transformers (every layer), U-Net (encoder-decoder skips), DenseNet.
Careful Initialization (He/Xavier)
Always. Xavier init for tanh/sigmoid layers (preserves variance). He init for ReLU layers (accounts for half-zeroed outputs). Wrong init = dead or exploding network from step 1.
Examples: He init for all ReLU CNNs. Xavier for Transformer linear layers. Orthogonal init for RNNs.
"Why can't we train a 50-layer sigmoid network?" → Chain rule multiplies derivatives. Sigmoid max derivative = 0.25. After 50 layers: 0.25⁵⁰ ≈ 10⁻³⁰. Gradient is effectively zero. Solution: ReLU + residual connections + careful initialization (He/Xavier).
Vanishing = multiplying numbers < 1 repeatedly. Exploding = multiplying numbers > 1 repeatedly. Residual connections = adding a highway that always carries gradient = 1.
👉 Try this: Set depth to 5 with Sigmoid — gradients are okay. Now drag to 25 — watch them collapse. Switch to ReLU — they stabilize. Switch to Residual — rock solid even at depth 30.

linear = nn.Linear(256, 128)
nn.init.kaiming_normal_(linear.weight, mode='fan_in', nonlinearity='relu')

Convolution Operation Step by Step

Click "Next Step" to watch the kernel slide one position at a time. See the element-wise multiplication and summation at each position. The output is built position-by-position. This is why 3×3 kernels are efficient (only 9 parameters per filter).

1
0
Key Insight: Convolution is template matching — a small kernel slides across the input, computing local dot products to detect features (edges, textures, shapes).
$$\text{output}[i,j] = \sum_a \sum_b \text{kernel}[a,b] \times \text{input}[i+a, j+b] + \text{bias}$$
kernel[a,b]:Small learnable weight matrix (e.g., 3×3). Each kernel detects one pattern — vertical edges, horizontal edges, corners, etc.
input[i+a, j+b]:Local patch of input centered at position (i,j). Convolution exploits spatial locality — nearby pixels are more related.
Stride:How far the kernel moves each step. Stride 1 = overlap (detailed features). Stride 2 = skip pixels (downsample, fewer parameters).
Padding:Add zeros around input border. "Same" padding preserves spatial size. "Valid" = no padding, output shrinks.
Output size:(input_size - kernel_size + 2×padding) / stride + 1. E.g., 32×32 input, 3×3 kernel, stride 1, padding 1 → 32×32 output.
Why dropout works: Prevents neurons from co-adapting (relying on specific other neurons). Forces network to learn robust features.
Effect: Trains ensemble of ~2^n subnetworks. At test time, average them (or use all units with scaling).
Dropout rate p: Typical: 0.5 for hidden layers, 0.1-0.2 for input. Higher p = stronger regularization but risk underfitting.
At test time: No dropout! Use all activations. Scaling: multiply by (1-p) during training, OR divide by (1-p) at test (depends on implementation).
When to Use
Small Kernels (3x3)
Default choice for modern CNNs. Two stacked 3x3 convs have the same receptive field as one 5x5 but with fewer parameters (18 vs 25) and more non-linearity. VGGNet proved this.
Examples: ResNet, VGG, U-Net, most modern vision architectures. Use 1x1 convs for channel mixing (Inception).
Large Kernels / Dilated Convolutions
When you need large receptive fields without deep stacking. Dilated convolutions increase field without adding parameters. Large kernels (7x7) used in first layer to capture low-level features.
Examples: ResNet first layer (7x7), WaveNet (dilated), semantic segmentation (DeepLab), ConvNeXt (7x7 depthwise).
Depthwise Separable Convolutions
Mobile/edge deployment where compute is limited. Splits into depthwise (spatial) + pointwise (channel mixing). ~8-9x fewer FLOPs than standard convolutions with minimal accuracy loss.
Examples: MobileNet, EfficientNet, on-device inference, real-time video processing.
Stride 2 Conv vs Pooling
Use stride-2 convolutions for learned downsampling (modern default). Max pooling for translation invariance. Average pooling for global feature aggregation before classifier head.
Examples: Stride 2 in ResNet downsampling. Global average pooling before final FC layer. Max pool in older architectures (AlexNet).
Dropout is a simple, powerful regularizer. Less important with batch norm, but still useful together. Monitor train vs test error.
Dropout = train multiple nets simultaneously, average at test time. Prevents co-adaptation. Ensemble effect = better generalization.
👉 Try this: Click "Next Step" 5 times — kernel slides from left to right. After each step, output value = (sum of element-wise products). Try "Prev Step" to backtrack. This simple sliding dot product is the entire CNN operation — applied millions of times.
import torch.nn as nn

model = nn.Sequential(
    nn.Conv2d(3, 32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d(32, 64, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.AdaptiveAvgPool2d(1),
    nn.Flatten(),
    nn.Linear(64, 10)
)
import torch

print(model(x).shape)           # torch.Size([8, 10])

Normalization: Batch vs Layer vs RMS

Toggle between Batch Norm, Layer Norm, and RMSNorm. For each, the colored region shows which activations get normalized together. Batch Norm normalizes across batch (column), Layer Norm across features (row). Notice: Batch Norm behavior depends on batch size!

Key Insight: Normalization stabilizes training by controlling the distribution of activations at each layer — without it, activations can drift wildly as weights update, slowing or breaking training.
$$\hat{x} = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta$$
\(\mu, \sigma^2\):Mean and variance computed over a specific set of activations. WHICH activations you normalize over is the key difference between methods.
\(\gamma, \beta\):Learned scale and shift parameters. After normalizing to zero-mean unit-variance, the network can learn to undo the normalization if needed. This preserves representational power.
BatchNorm (across batch):Computes mean/variance across all samples in a batch for each feature. Requires batch size ≥ 32 to get stable statistics. Different behavior at train (batch stats) vs test (running average). Breaks at batch_size=1.
LayerNorm (across features):Computes mean/variance across all features for each sample independently. Batch-size independent — works with batch=1. Same behavior at train and test. The standard for Transformers.
RMSNorm:Like LayerNorm but skips the mean subtraction — only divides by root-mean-square. ~10-15% faster with similar performance. Used in LLaMA, Mistral, and most modern LLMs.
GroupNorm: Splits channels into groups, normalizes within each group. Works with any batch size. Used in object detection (Mask R-CNN) and generative models where batch sizes are small.
Why it works: Without normalization, small weight changes in early layers cause large activation shifts in later layers ("internal covariate shift"). Normalization keeps activations in a stable range, allowing higher learning rates and faster convergence.
Choosing normalization: CNNs with large batches → BatchNorm. Transformers/sequences → LayerNorm or RMSNorm. Small batches or detection → GroupNorm. When in doubt, LayerNorm is the safest default.
When to Use
Batch Normalization
CNNs and fixed-batch training. Normalizes across the batch dimension. Requires sufficiently large batches (32+). Behaves differently at train vs test time (uses running stats at test). Do not use with small batches or variable-length sequences.
Examples: ResNet, EfficientNet, any CNN with batch size >= 32. Standard in computer vision pipelines.
Layer Normalization
Transformers and sequence models. Normalizes across features for each sample independently. Batch-size independent -- works with batch=1. Same behavior at train and test time.
Examples: All Transformer models (GPT, BERT, ViT), RNNs, online/streaming inference, any variable-length input.
RMSNorm
Modern LLMs. Like LayerNorm but skips mean-centering -- only normalizes by root-mean-square. ~10-15% faster with comparable quality. Simpler gradient computation.
Examples: LLaMA, Gemma, Mistral, PaLM. Preferred in large-scale language model pretraining.
Group Normalization
When batch size must be small (1-8) but you still want normalization. Divides channels into groups and normalizes within each group. Batch-size independent like LayerNorm.
Examples: Object detection (small batches due to large images), video models, 3D medical imaging, distributed training with small local batches.
Choosing normalization: Use BatchNorm for CNNs with fixed batch sizes ≥32. Use LayerNorm for transformers and any model where batch size varies or equals 1 (like autoregressive generation). BatchNorm breaks at batch_size=1 because you can't compute statistics from one sample.
CNNs are the backbone of computer vision. Receptive field grows with depth. Understand how filter size and stride affect output dimensions.
💡 Real-world: RMSNorm (a simplified LayerNorm) is used in LLaMA, Mistral, and most modern transformers. It's 10-15% faster than LayerNorm with no quality loss — at the scale of training a 70B model, that saves weeks of GPU time.
CNN = local pattern matching with weight sharing. Captures spatial hierarchies. Deep = large receptive fields = global context.

RNN/LSTM/GRU: Recurrence & Gating

Toggle between Vanilla RNN, LSTM, and GRU. For Vanilla RNN, trace the hidden state flow and see it accumulate (or vanish) through timesteps. For LSTM, watch the cell state bypass (straight arrow) — this is the "memory" that avoids vanishing gradients. Gates control what flows.

5
Key Insight: Recurrent networks process sequences by maintaining hidden state that captures context from previous steps.
$$h_t = \sigma(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b_h)$$
\(h_t\):Hidden state at timestep t. Acts as memory. Shape: (batch_size, hidden_dim). Updated every step based on current input and previous state.
\(h_{t-1}\):Hidden state from previous timestep t-1. Carries information forward in sequence.
\(x_t\):Input at timestep t (e.g., word embedding, single token). Shape: (batch_size, input_dim)
\(W_{hh}\):Recurrent weight matrix (hidden-to-hidden). Shared across all timesteps. Shape: (hidden_dim, hidden_dim)
\(W_{xh}\):Input-to-hidden weight matrix. Shape: (input_dim, hidden_dim). Also shared across timesteps.
\(b_h\):Bias vector for hidden state. Shape: (hidden_dim,)
\(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b_h\):Linear combination of previous hidden state and current input. Combines memory with new information.
\(\sigma(\ldots)\):Activation function (tanh common for RNNs). Bounds hidden state, typically to (-1,1) range.
Hidden state h_t: Memory that carries information from previous positions. Initialized to zero, updated at each step.
Sequence processing: At each timestep, network sees current input + hidden state from previous. Learns dependencies across sequence.
Vanishing gradient problem: Gradients flow back through many timesteps. Same multiplication of derivatives (<0.1) causes vanishing gradients.
LSTM/GRU solutions: Gated mechanisms control information flow. Cell state = long-term memory. Much better at learning long-range dependencies.
Cell state highway: The cell state C_t acts as a gradient highway — it's a straight path where gradients can flow unchanged across many timesteps. Without it, gradients vanish after ~10 steps (same problem as deep networks with sigmoid).
When to Use Each Architecture
Vanilla RNN
Short sequences where long-range memory isn't critical. Rarely used in practice today due to vanishing gradients.
Examples: Simple character-level language models, short time-series with <10 steps, toy sequence tasks for learning.
LSTM
Long sequences requiring memory over hundreds of steps. The go-to RNN before Transformers. Cell state acts as a "conveyor belt" for long-range information.
Examples: Machine translation (seq2seq), speech recognition, music generation, handwriting synthesis, time-series forecasting (stock prices, weather).
GRU
Similar tasks to LSTM but when you want faster training and fewer parameters. 2 gates instead of 3 = ~25% fewer weights. Often matches LSTM quality.
Examples: Real-time speech synthesis, chatbot response generation, sensor data on edge devices, any LSTM task where compute budget is tight.
RNNs vs. Transformers: When Does Each Win?
Use RNNs (LSTM/GRU) When...
• Streaming/online inference — processing one token at a time with O(1) memory per step
• Very long sequences where quadratic attention is too expensive (e.g., continuous sensor data, audio waveforms)
• Low-resource or edge deployment — much smaller model footprint
• Sequential decision-making where order strictly matters (robotics control loops)
Use Transformers When...
• You need to capture dependencies across the full sequence simultaneously (not just left-to-right)
• Parallelizable training matters — Transformers train on all positions at once, RNNs are sequential
• You have large data and compute — Transformers scale better with more parameters
• Most NLP tasks today (LLMs, BERT, GPT) — Transformers dominate
In interviews, know that LSTMs were the gold standard before 2017 (Attention Is All You Need). Today Transformers dominate NLP, but RNNs still appear in streaming, on-device, and time-series settings. State Space Models (Mamba, S4) are a modern alternative that combine RNN-like O(1) inference with Transformer-like parallelism during training.
Vanilla RNN = broken for long sequences. LSTM = gold standard RNN (3 gates + cell state). GRU = cheaper LSTM (2 gates, no cell state). Transformer = parallel + global attention but O(n²). Pick based on sequence length, latency needs, and compute budget.
👉 Try this: Set sequence length to 10, step through all timesteps in Vanilla RNN — watch the gradient bars collapse to near-zero. Switch to LSTM — same 10 steps but gradients stay healthy. That's the entire reason LSTMs exist.
import torch
import torch.nn as nn

lstm = nn.LSTM(input_size=128, hidden_size=256, num_layers=2,
               batch_first=True, dropout=0.1)
x = torch.randn(32, 10, 128)  # batch=32, seq_len=10, features=128
output, (h_n, c_n) = lstm(x)
# output: (32, 10, 256) - all hidden states
# h_n: (2, 32, 256) - final hidden state per layer

# GRU: same API, no cell state
gru = nn.GRU(128, 256, num_layers=2, batch_first=True)
output, h_n = gru(x)  # no c_n returned

LSTM has 3 gates (forget, input, output) + cell state. GRU has 2 gates (reset, update), fewer params.

Flash Attention: Tiled IO-Aware Computation

Watch the "Standard Attention" animation — it materializes the full N×N attention matrix (memory bottleneck!). Toggle to "Flash Attention" — computation is tiled into blocks (no full matrix in memory). Despite the same result, tiling reduces slow memory access (HBM) dramatically.

64
Key Insight: Flash Attention is an IO-aware algorithm that computes exact attention while dramatically reducing memory reads/writes to slow GPU memory (HBM).
$$\text{Standard: } O(N^2) \text{ HBM memory} \quad \Rightarrow \quad \text{Flash: } O(N) \text{ HBM memory}$$
GPU memory basics:GPUs have two memory types: SRAM (fast but tiny, ~20MB) and HBM (slow but large, ~40GB). Standard attention builds an N×N matrix that lives in slow HBM. Flash Attention computes attention in small tiles that fit in fast SRAM — same math, 2-4× faster.
The bottleneck:Standard attention materializes the full \(N \times N\) attention matrix in HBM (slow GPU memory). Memory, not compute, is the bottleneck.
Memory hierarchy:GPU SRAM is fast (~19 TB/s) but small (~20 MB). HBM is large (~40 GB) but slow (~1.5 TB/s). Flash Attention exploits this gap.
Tiling strategy:Splits Q, K, V into blocks that fit in SRAM. Computes attention tile-by-tile, never materializing the full \(N \times N\) matrix in HBM.
Online softmax:Uses online softmax trick to compute exact softmax across tiles without storing full attention scores. Rescales running max and normalizer.
IO complexity:Standard attention: \(O(N^2)\) HBM reads/writes. Flash Attention: \(O(N^2 d / M)\) where M = SRAM size. Fewer HBM accesses by factor of M/d.
Same result: Flash Attention produces mathematically identical output to standard attention. It is an exact algorithm, not an approximation.
Wall-clock speedup: 2-4x faster than standard attention in practice. Enables training with much longer sequences (4K to 64K+ tokens).
Backward pass: Recomputes attention from blocks during backward pass instead of storing it. Trades extra compute for reduced memory.
When to Use
Flash Attention
Default choice for any Transformer training or inference. Always use when available -- it is exact (not approximate) and strictly faster. Especially critical for long sequences (4K+ tokens) where standard attention runs out of memory.
Examples: LLM pretraining (GPT-4, LLaMA), long-context models (100K+ tokens), ViT training, any model using self-attention on modern GPUs (A100, H100).
Standard Attention
Only when Flash Attention is unavailable (unsupported hardware, custom attention patterns like sparse/cross-attention with complex masking). Also useful for debugging since the full attention matrix is materialized and inspectable.
Examples: Prototyping/debugging attention patterns, older GPUs without Flash support, non-standard attention masks, attention visualization/interpretability research.
Multi-Query / Grouped-Query Attention
Inference optimization. MQA shares one K,V head across all query heads. GQA uses a few K,V groups (between MQA and MHA). Reduces KV-cache memory at inference with minimal quality loss.
Examples: GQA in LLaMA 2/3, Mistral. MQA in PaLM, Falcon. Critical for serving LLMs at scale with long contexts.
Sparse / Linear Attention
Extremely long sequences (10K-1M+ tokens) where even Flash Attention's O(N^2) compute is too expensive. Trades some quality for sub-quadratic complexity. Active research area.
Examples: Longformer (local + global), BigBird (sparse), Mamba/S4 (state-space, linear), genomics (100K+ base pairs), full-document retrieval.
Flash Attention is IO-aware: it optimizes data movement between fast SRAM and slow HBM, not the number of FLOPs. Same math, fewer memory transfers.
Flash = tiling + online softmax + recomputation. Never stores full N x N matrix. Exact, not approximate.
Try this: Sequence length = 100. Standard Attention materializes 100x100 matrix (10K values in slow memory). Flash Attention: tiles into 4x4 blocks (only 25 values in fast memory at a time). Result is identical, but memory bandwidth drops 4-10x. This is why Flash Attention made long sequences practical.
💡 Real-world: Flash Attention (Dao, 2022) made it practical to train models with 100K+ context windows. Without it, the O(N²) memory cost of attention would require prohibitively expensive hardware. It's now the default in PyTorch and every major training framework.
import torch
import torch.nn.functional as F

# Standard attention (what Flash Attention optimizes)
def attention(Q, K, V):
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / d_k**0.5
    weights = F.softmax(scores, dim=-1)
    return weights @ V

batch, heads, seq_len, d_k = 2, 8, 64, 64
Q = K = V = torch.randn(batch, heads, seq_len, d_k)
out = attention(Q, K, V)  # (2, 8, 64, 64)

# PyTorch built-in Flash Attention (auto-selects best kernel)
out_flash = F.scaled_dot_product_attention(Q, K, V)

F.scaled_dot_product_attention auto-uses Flash Attention when available. O(n) memory vs O(n^2).

Scaling Laws: Loss vs Compute/Data/Parameters

Drag "Compute Budget" slider — the optimal point follows Chinchilla scaling (allocate equal compute to parameters and data tokens). If you allocate too much to parameters (left), loss plateaus (data bottleneck). If too much to data (right), loss plateaus too (parameter bottleneck). The diagonal is optimal.

Key Insight: Neural scaling laws reveal power-law relationships between loss, model size (N), and data (D), enabling compute-optimal training.
$$L(N, D) \approx \left(\frac{N_c}{N}\right)^{\alpha} + \left(\frac{D_c}{D}\right)^{\beta} + L_{\infty}$$
\(L(N, D)\):Test loss as a function of parameters N and data tokens D. Follows smooth power laws over many orders of magnitude.
\(\alpha \approx 0.34\):Exponent for model size scaling. Loss decreases as \(N^{-0.34}\). Diminishing returns: 10x params gives ~2x loss reduction.
\(\beta \approx 0.28\):Exponent for data scaling. Loss decreases as \(D^{-0.28}\). Data scaling is slightly less efficient than parameter scaling.
What the exponents mean:The exponents (α≈0.34, β≈0.28) are empirical — fitted to thousands of training runs by Kaplan et al. (2020). They differ because parameters and data don't help equally: doubling parameters helps more than doubling data. This means big models trained on less data can match smaller models trained on more.
\(L_{\infty}\):Irreducible loss (entropy of natural language). Even infinite model + data cannot go below this floor (~1.69 nats).
Chinchilla optimal: For a fixed compute budget C, allocate roughly equal compute to parameters and data. Optimal N and D both scale as \(C^{0.5}\). Train a smaller model on more data.
Compute relation: \(C \approx 6ND\) (FLOPs for a forward+backward pass). Doubling compute should double both N and D for optimal training.
Chinchilla vs GPT-3: GPT-3 (175B params, 300B tokens) was undertrained. Chinchilla (70B params, 1.4T tokens) matched its performance with 4x less compute at inference.
Practical impact: Scaling laws enable predicting large model performance from small runs. Guides billion-dollar training decisions before committing full compute.
When to Use
Chinchilla Scaling (Balanced)
When training cost is your main constraint and you want the lowest loss per FLOP. Allocate compute equally to model size and data tokens. Train smaller models on more data. Best for one-time training runs.
Examples: Chinchilla (70B, 1.4T tokens), LLaMA 1 (65B, 1.4T tokens). Research labs optimizing training efficiency.
Over-Training (Inference-Optimized)
When inference cost matters more than training cost. Train a smaller model on far more data than Chinchilla-optimal. Higher training cost per FLOP, but cheaper to serve. Dominant strategy for production LLMs.
Examples: LLaMA 2 (70B, 2T tokens), LLaMA 3 (8B, 15T tokens -- 200x overtrained). Phi-3 (3.8B, 3.3T tokens). Any model deployed at scale.
Scaling Law Extrapolation
Before committing to a large training run, train small models (1M-1B params) at various compute budgets and fit power laws. Predict large model performance without training it. Saves millions in compute.
Examples: OpenAI scaling papers, Chinchilla study, any team planning a 10B+ parameter training run. Used to set hyperparameters and estimate final loss.
Compute-Optimal Architecture Search
Use scaling laws to compare architectures at small scale before scaling up. If architecture A has better scaling exponent than B, it will dominate at large scale even if worse at small scale.
Examples: Comparing MoE vs dense models, choosing attention variants, depth vs width trade-offs, deciding between Transformer and SSM architectures.
Chinchilla showed most large models were undertrained (too many params, too little data). Optimal: ~20 tokens per parameter. Scaling laws let you predict loss before training.
Scaling = power laws. More params + more data = lower loss, but diminishing returns. Chinchilla = balance params and data equally.
Try this: Move compute budget left (favor parameters, 80/20 split) -- loss plateaus (data bottleneck). Move right (80/20 data favoring) -- loss plateaus (parameter bottleneck). The diagonal (50/50 split) is optimal. Chinchilla discovered this: equal compute to parameters and data.
💡 Real-world: Scaling laws let labs predict performance BEFORE spending $100M on a training run. OpenAI used them to plan GPT-4: train small models, fit the power law, extrapolate to the target size. It's the closest thing to a crystal ball in ML.
import torch
import torch.nn as nn

model = nn.Linear(256, 10)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=100)

for epoch in range(100):
    x = torch.randn(32, 256)
    loss = nn.CrossEntropyLoss()(model(x), torch.randint(0, 10, (32,)))
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    scheduler.step()
    if epoch % 25 == 0:
        print(f"Epoch {epoch}: loss={loss.item():.3f}, lr={scheduler.get_last_lr()[0]:.6f}")

AdamW + cosine LR schedule is the standard training recipe for modern deep learning.

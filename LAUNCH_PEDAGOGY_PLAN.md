# ML Visualizations Launch + Pedagogy Plan

Created: 2026-04-02

## Saved From Prior Review

### Product / launch priorities

1. Reposition the product.
Use a product name or a sharper value proposition on the landing page, and demote the personal-brand framing.

2. Fix the truth gap.
Either bundle KaTeX/Prism locally or stop claiming "zero dependencies" and "works offline."

3. Add launch packaging.
Create an `og:image`, add stronger per-page metadata, and prepare one short demo clip/GIF for social sharing.

4. Reduce reference friction.
Add search, per-page table of contents, and deep links to sections so the library is easier to navigate.

5. Add academic scaffolding.
Per section: learning objective, 1-3 references, "when this breaks," and clearer trust signals for instructors/students.

## Content Improvements

1. Tighten scope per page.
Each page should make its intended depth explicit: interview-prep overview, practitioner intuition, or from-scratch derivation.

2. Distinguish intuition from rigor.
For each concept, clearly separate:
- Intuition
- Formal definition / equation
- Practical use
- Failure mode / caveat

3. Add prerequisite signaling.
Mark sections with lightweight difficulty or prerequisite labels so learners know what to read first.

4. Add "why this matters" hooks.
Open each major section with one real problem the concept solves in practice.

5. Add comparison frames.
When learners confuse neighboring ideas, add explicit contrast boxes:
- MLE vs MAP
- precision vs recall
- cross-entropy vs KL
- self-attention vs cross-attention
- bagging vs boosting

6. Add "common interview trap" callouts.
Highlight mistakes people commonly make under pressure, not just the correct explanation.

7. Add "what changes when scaling up?" notes.
Bridge toy visualizations to real systems so learners do not over-generalize from the small example.

8. Add references consistently.
Important claims, architectures, and historical notes should link to papers, docs, or canonical sources.

## Pedagogy Improvements

1. Use a consistent teaching rhythm.
Ideal section flow:
- See it
- Manipulate it
- Name it
- Formalize it
- Test it
- Generalize it

2. Make every interaction goal-directed.
Many controls are interesting, but the learning value jumps when each one has a short mission like:
"Set prior to 1% and explain why the posterior is still only ~50%."

3. Reduce passive reading density.
Some insight boxes are strong but long. Break them into smaller chunks so the learner alternates between action and explanation.

4. Make misconceptions first-class.
Each major topic should include:
- what beginners think
- why that is wrong
- what the correct mental model is

5. Add retrieval practice loops.
Instead of quizzes only at page level, include tiny in-line check questions after especially important sections.

6. Add transfer prompts.
After a concept is learned, ask where else it applies. This improves interview recall and conceptual flexibility.

7. Support multiple learner modes.
Each page should better support:
- skim mode
- study mode
- interview cram mode

8. Make visual encodings explicit.
State what color, position, width, and motion represent so learners do not infer the wrong meaning from the chart.

## Suggested Execution Order

2. Improve navigation and discoverability.
3. Standardize the teaching template for sections.
4. Add citations and academic trust signals.
5. Add stronger launch assets and sharing surfaces.

Self-Attention: Q, K, V Computation

Follow the full attention computation: input → Q,K,V projections → attention scores → softmax → weighted values → output. Click tokens to see what they attend to.

1.0
Key Insight: Self-attention is the core mechanism enabling Transformers to learn long-range dependencies.
Self-Attention(Q,K,V) = softmax(QK^T/√d_k)V
Q (Query):Query matrix, shape (seq_len, d_k). "What am I looking for?" Learned projection of input.
K (Key):Key matrix, shape (seq_len, d_k). "What does each position represent?" Compared against queries to determine relevance.
V (Value):Value matrix, shape (seq_len, d_v). "What information to extract?" Weighted and aggregated based on attention scores.
d_k:Dimension of keys (usually 64). Affects scaling factor and model capacity.
d_v:Dimension of values (usually 64). Output dimension per head. Total output = num_heads × d_v.
QK^T:Attention logits, shape (seq_len, seq_len). Measures similarity between each query and all keys. Unnormalized scores.
1/√d_k:Scaling factor prevents attention weights from becoming too confident. Variance of QK^T is O(d_k), scaling keeps gradient O(1).
softmax(...):Converts scaled scores to probability distribution. Each row sums to 1. Determines how much each position attends to others.
softmax(...) · V:Weighted sum of values. Output shape: (seq_len, d_v). Each position gets weighted combination of all value vectors.
Process: 1. Project input to Q, K, V. 2. Compute attention weights: softmax(QK^T). 3. Weight values: apply weights to V. 4. Concatenate heads.
Multi-head attention: h heads with different projections. Each head learns different attention patterns. Output: concat of all heads.
Scaling by √d_k: Prevents softmax from becoming too peaked. Variance of QK^T is O(d_k), so scale gradients stay O(1).
Masking in decoder: Prevent attending to future tokens. Set attention weights to -∞ for future positions before softmax.
Why Q, K, V not just use input directly? Learned projections provide flexibility. Different heads can attend differently.
Self-attention = compare all positions with each other. Learn what to pay attention to. Q asks, K answers, V provides info.

Multi-Head Attention

Each attention head learns different patterns. One head might focus on syntax, another on semantics, another on position.

4
Key Insight: Layer normalization and position-wise feed-forward networks stabilize Transformer training.
LayerNorm(x) = γ · (x - μ) / √(σ² + ε) + β
x:Input to layer normalization. Shape: (batch_size, seq_len, hidden_dim) or (batch_size, hidden_dim)
μ:Mean of x computed per example (across the feature dimension). Shape: (batch_size, seq_len) or (batch_size,)
σ²:Variance of x per example (across features). Measures spread of activation values.
ε:Small constant (e.g., 1e-5 or 1e-6) for numerical stability. Prevents division by zero when variance is very small.
(x - μ) / √(σ² + ε):Standardization step. Shifts to mean 0, scales to variance 1. Normalizes across features for each example.
γ (gamma):Learnable scale parameter. Shape: (hidden_dim,). Allows network to scale normalized activations. Learned during training.
β (beta):Learnable shift parameter. Shape: (hidden_dim,). Allows network to shift normalized activations. Learned during training.
γ · (...) + β:Affine transformation. Restores model expressiveness after normalization. If γ=1, β=0 everywhere, recovers standard normalization.
Why layer norm (not batch norm): Batch norm needs batch dimension. Transformers process variable-length sequences. LayerNorm normalizes per instance.
Position-wise FFN: Applied to each position independently. Two linear layers with ReLU: FFN(x) = max(0, xW₁ + b₁)W₂ + b₂.
FFN dimensions: First layer expands (4x hidden size). Second layer projects back. Adds expressiveness. Most parameters in Transformers.
Residual connections: Add input to output of each sublayer. Enables training of deep Transformers. Stabilizes gradients.
Transformer depth comes from stacking. Each layer adds computational power but also parameters and memory.
LayerNorm normalizes each example independently. FFN expands then contracts. Residual connections = skip highways.

KV-Cache: Efficient Autoregressive Inference

During generation, cache K and V from previous tokens to avoid recomputation. Watch the cache grow as new tokens are generated.

Key Insight: Language model pretraining learns general linguistic patterns from large unlabeled text via next-token prediction.
P(x_{t+1} | x_1, ..., x_t) = softmax(W · h_t + b)
P(x_{t+1} | x_1, ..., x_t):Conditional probability distribution over next token x_{t+1}, given all previous tokens. Output shape: (vocab_size,). Sum to 1.
x_{t+1}:Next token to predict. Element of vocabulary (0 to vocab_size-1). Loss = cross-entropy between true next token and this distribution.
x_1, ..., x_t:All previous tokens (context). Model processes entire sequence, attends to all previous positions via self-attention masks.
h_t:Hidden representation after processing sequence up to position t. Shape: (batch_size, hidden_dim). Contains context from all previous tokens.
W:Output projection matrix. Shape: (hidden_dim, vocab_size). Projects hidden state to logit space (unscaled scores for each token).
b:Output bias. Shape: (vocab_size,). Learned scalar offset per token.
W · h_t + b:Logits (unnormalized scores for each vocab token). Shape: (vocab_size,). No probability constraints yet.
softmax(...):Converts logits to valid probability distribution. Exponentiates, normalizes. Output: (vocab_size,) with elements in [0,1], sum=1.
Causal language modeling: Predict next token given previous tokens. Masks future positions. Loss = cross-entropy on next token prediction.
Scale matters: Larger models → better performance. 7B → 13B → 70B. More data helps too. Chinchilla scaling: match model size and data.
Emergent abilities: Chain-of-thought reasoning, few-shot learning appear at scale. Model size threshold matters.
Compute-optimal training: Optimal allocation: train to model convergence (2T tokens, where T = model parameters). LLaMA follows this principle.
Pretraining is expensive but transfer learning is powerful. Fine-tuning on specific task often very effective. Few-shot prompting often works!
LM pretraining = predict next token from massive unlabeled text. Learns grammar, facts, reasoning. Scale → emergent abilities.

Transformer Block Architecture

The complete flow through a decoder block: LayerNorm → Attention → Residual → LayerNorm → FFN → Residual.

Key Insight: Instruction tuning and RLHF align models with human preferences and make them helpful, harmless, honest.
Instruction tuning: Fine-tune pretrained LM on (instruction, response) pairs. Teaches model to follow directions. Simple but effective.
RLHF (Reinforcement Learning from Human Feedback): 1. Collect human preference data (which output better?). 2. Train reward model. 3. Use RL to optimize policy (LM) against reward.
Reward model training: Model learns to predict human preferences. Trained on paired outputs (better, worse). Loss = negative log likelihood of preference.
RL loss: PPO or similar policy gradient. Maximize: E[reward] - KL(π_new || π_old). KL term prevents diverging from pretrained model.
RLHF is expensive (human feedback needed) but crucial for alignment. Alternative: DPO (direct preference optimization, no reward model).
Instruction tuning = teach format. RLHF = optimize for human preferences. Alignment = make models helpful, safe, honest.

Inference Optimization: Speculative Decoding & Quantization

Speed up inference with draft models (speculative decoding) and reduce memory via quantization (FP32 → INT8 → INT4).

Key Insight: Quantization reduces model size and inference latency by using lower-precision numbers instead of float32.
Precision levels: float32 (32 bits). float16 (16 bits, 2x smaller). int8 (8 bits, 4x smaller). int4 (4 bits, 8x smaller).
Quantization methods: Post-training quantization: quantize after training. Quantization-aware training: simulate quantization during training.
Trade-offs: int4: 8x smaller, but quality loss. int8: ~4x smaller, minimal loss. Depends on layer sensitivity.
Mixed precision: Keep critical layers (embeddings, attention output) in higher precision. Quantize feed-forward networks heavily. Optimal quality/speed.
Quantization crucial for deployment on edge devices. int8 very practical. int4 requires careful tuning but very memory efficient.
Quantization = smaller, faster, less accurate. int8 sweet spot: 4x smaller, minimal loss. int4 when size critical.

Key Insight: Scaling laws govern how model performance improves with more parameters, data, and compute.
Loss(N) ≈ a / N^α (model size scaling)
Loss(N):Test loss when training with N parameters. Lower = better performance. Asymptotically approaches optimal loss.
N:Number of model parameters (weights, biases). Measured in millions (M) or billions (B). Ranges from 1M to 400B+ for LLMs.
a:Scaling constant (empirical fit). Depends on task, data quality, architecture. Fitted from experimental data.
α:Scaling exponent for model size, typically 0.07-0.1. Log-log slope. α = 0.1 means 10x params → ~2x loss reduction.
≈:Approximation sign. Empirical power law, not exact theory. Holds across wide range of N (millions to hundreds of billions).
Loss(D) ≈ b / D^β (data scaling)
Loss(D):Test loss when training with D tokens of data. Larger D → lower loss. Follows similar power law to model size.
D:Number of training tokens (data quantity). For 70B parameter model, optimal D ≈ 1.4 trillion tokens (Chinchilla scaling).
b:Scaling constant for data dimension. Empirically fitted. Typically larger coefficient than model scaling constant a.
β:Scaling exponent for data, typically 0.1-0.2. β = 0.15 means 10x more data → ~1.4x loss reduction. Slower than model scaling.
Observed exponents: α ≈ 0.07-0.1 (model), β ≈ 0.1-0.2 (data). Both improve loss, but slowly (power law).
Chinchilla/Compute optimal: Optimal allocation: increase model and data together. ~20 tokens per parameter. Avoid: huge model on small data or vice versa.
Implications: Performance improves continuously with scale. No ceiling expected. Bigger models generalize better despite more capacity.
Practical impact: Predicting loss on held-out data. Choosing model size. Budget allocation (compute, parameters, data).
Scaling laws are empirical (fit from data) not proven. But remarkably consistent across orders of magnitude. Useful for planning.
Scaling laws = loss improves as power law with model/data size. Chinchilla = balance model and data. Bigger = better.

RAG: Retrieval-Augmented Generation

Query → Embed → Retrieve → Rerank → Augment Prompt → Generate. Compare chunking strategies: fixed, semantic, recursive.

Key Insight: Knowledge distillation transfers knowledge from large teacher models to smaller student models.
Goal: Make smaller model mimic large model. Smaller = faster inference, deployable on edge devices.
KD training: Loss = α · cross_entropy(student, hard_labels) + (1-α) · KL(student_logits, teacher_logits). Soft targets from teacher.
Temperature scaling: Higher temperature softens probability distributions. Makes soft targets more informative. Typical: T=4.
Results: Student can match large part of teacher performance at much smaller size. 10x compression: ~5-10% quality loss.
Distillation effective for deployment. Student learns from teacher's mistakes too (soft probability targets reveal model uncertainty).
Knowledge distillation = student learns from teacher. Soft targets more informative than hard labels. Trade: quality for size/speed.

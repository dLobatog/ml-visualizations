Interview Tips: β-VAE

What it is: VAE with weighted KL divergence. Higher β = more disentanglement, lower reconstruction. Trade-off is explicit and unavoidable.

ELBO = E[log p(x|z)] - β·KL(q(z|x) || p(z))
ELBO (Evidence Lower Bound):Log-likelihood lower bound; maximize to improve both reconstruction and latent consistency
E[log p(x|z)]:Reconstruction term; expected log-likelihood of data given latent; higher = better reconstruction
x:Input data (e.g., image)
z:Latent representation; lower-dimensional encoding of x
KL(q(z|x) || p(z)):Kullback-Leibler divergence; regularization term; measures how much posterior q(z|x) deviates from prior p(z) ≈ N(0, I)
β:Weighting parameter ∈ [0, ∞); β=1 standard VAE, β>1 encourages disentanglement, β<1 prefers reconstruction
Posterior Collapse: q(z|x) = p(z) when β too high → encoder useless, no info in z
Posterior Collapse:When KL term dominates, encoder learns to ignore x; latent z becomes independent of input, learned nothing
KL Divergence:For Gaussian VAE: KL = 0.5 * Σ(μ² + σ² - 1 - log(σ²)); when σ²→1, μ→0, KL→0 (collapse)
β=1 baseline. High β = disentangled (each z dimension independent). Low β = good reconstruction. Can't have both.
Disentanglement: Each latent dimension encodes independent factors. E.g., one dim = style, another = content. Interpretable.
Tuning β: critical, often overlooked. Wrong β gives fuzzy reconstructions (low β) or collapsed latents (high β).
Posterior collapse: if KL → 0 after few epochs, you have collapse. Use KL warmup schedule to prevent.
Production rule: Start β=1, tune for your task. Use KL annealing. Evaluate disentanglement via downstream classification, not visual inspection. Gaussian prior for smooth interpolation.

Interview Tips: Diffusion Models

What it is: Learn to reverse noise addition. Forward adds noise T steps: x_t = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε. Reverse: x_t → x_{t-1} iteratively. Stable vs GANs. High quality samples.

Forward: q(x_t|x_0) = N(√ᾱ_t·x_0, (1-ᾱ_t)I) | Reverse: denoise ε ~ N(0,I)
q(x_t|x_0):Forward process distribution; conditionally Gaussian given clean x_0
x_0:Clean original data
x_t:Noisy data at diffusion step t; interpolation between x_0 and pure noise
ᾱ_t (alpha_bar_t):Cumulative product of noise schedule; starts at ≈1 (t=0, clean), approaches 0 (t=T, full noise); ᾱ_t = ∏_{i=1}^t α_i where α_i = 1 - β_i
β_i:Noise variance at step i; β_i ∈ [0, 0.02] typically; defines how much noise added at each step
√ᾱ_t·x_0:Signal coefficient; how much original signal remains at step t; decreases over time
(1-ᾱ_t)I:Noise variance (covariance matrix); noise increases over time as ᾱ_t decreases
ε ~ N(0,I):Standard Gaussian noise used in forward process; mean 0, variance 1
Loss: ||ε_θ(x_t, t) - ε||² → predict added noise at step t
ε_θ(x_t, t):Neural network prediction of noise at step t given noisy input x_t; outputs estimated noise ε̂
ε:Ground truth noise that was added in forward process; sampled fresh for each training example
||...||²:Mean squared error (L2 loss); measures pixel-level reconstruction error between predicted and actual noise
θ:Neural network parameters (typically UNet or transformer); learned during training
Closed-form diffusion: add noise → learn to reverse → sample via T denoising steps. More stable than adversarial training.
T (diffusion steps): T=1000 high quality but slow. Trade-off: quality vs latency. DDIM/DPM-Solver enable fast sampling (few steps).
Variance schedule (linear, cosine, etc.) tunes quality. Wrong schedule = poor samples. Critical tuning knob.
Score matching = predict noise, not mean directly. More stable training than explicit mean prediction.
Production rule: Use DDIM/DPM-Solver for speed. Store noise schedule (reproducibility). Monitor FID/IS scores. Slower than GANs but more stable (trade latency for reliability).
Conditioning: Classifier-free guidance common for control but can be unstable. Fine-tuning via inpainting/text-to-image needs specific procedures.

5. Classifier-Free Guidance: Conditional Generation Control

w = 7.5
0 = unconditional, 1 = conditional only, >1 = amplified

How It Works: Classifier-Free Guidance

What it is: Steer generation without separate classifier. Reweight conditional vs unconditional predictions. w=0 (unconditional), w=1 (conditional), w>1 (amplify condition). Enables text-to-image control.

ε_guided = ε_θ(x_t, ∅) + w·(ε_θ(x_t, c) - ε_θ(x_t, ∅))
ε_θ(x_t, ∅):Network prediction for unconditional generation (condition c is None/dropped); baseline noise estimate
ε_θ(x_t, c):Network prediction for conditional generation given condition c (e.g., text prompt, image class)
c:Conditioning signal (e.g., text embedding from CLIP, class embedding); modulates generation direction
w (guidance scale):Scaling factor ∈ [0, ∞); w=0 = unconditional only, w=1 = conditional only, w>1 = emphasize condition; w=7.5 typical, w>15 = artifacts
ε_guided:Final guided noise prediction combining unconditional baseline with conditional offset amplified by w
Interpretation:w=0 ignores condition, w=1 only listens to condition, w=7.5 amplifies conditional signal 7.5x relative to unconditional
Importance reweighting: p(x_t|c) ∝ p(x_t) · [p(x_t|c)/p(x_t)]^w → sharpens distribution
p(x_t):Unconditional distribution; prior without any guidance
p(x_t|c):Conditional distribution given condition c; posterior after observing condition
[p(x_t|c)/p(x_t)]^w:Likelihood ratio raised to power w; reweights samples toward condition; higher w = stronger reweighting
Sharp distribution:Increasing w makes distribution more concentrated on high-likelihood samples; less diversity, higher fidelity to condition
w=7.5 is Stable Diffusion default. w>15 = artifacts. Trade w for diversity-vs-fidelity. Cannot invent info not in training.
Training requirement: Randomly drop condition c with prob ~10% during training so model learns both conditional + unconditional denoising. Critical!
High w (w>15) causes mode collapse, artifacts, saturated colors, over-sharpening. Guidance cannot "invent" outside training data.
Guidance is orthogonal to likelihood. Doesn't improve likelihood, just reweights sampling distribution toward condition.
Production rule: Start w=1.0, increase gradually. Expose w in UI. Monitor diversity (LPIPS) + adherence (CLIP). Combine with negative prompts, LoRA, prompt weights.
Task-specific: Photorealistic: w=7-12. Artistic: w=4-8. One-dimensional control; multidimensional requires other methods.

6. Latent Diffusion & Stable Diffusion

50 steps
More steps = better quality, slower. 20-100 typical.

Pipeline: Encode → Diffuse → Decode

What it is: Diffuse in compressed latent space not pixels. 3-stage: (1) VAE encode x→z (8-16x spatial reduction), (2) diffuse in z, (3) VAE decode z→x̂. 10-50x speedup vs pixel-diffusion, quality maintained.

Latent forward: q(z_t | z_0) = √ᾱ_t·z_0 + √(1-ᾱ_t)·ε (same form, compressed space)
z_0:Latent representation from VAE encoder; compressed version of original image x_0; shape ≈ H/8 × W/8 × D where D ∈ [4, 8]
z_t:Latent at diffusion step t; noisy version of z_0; same shape as z_0 but with added noise
ᾱ_t:Cumulative noise schedule in latent space; same as pixel-diffusion but operates on compressed features
√ᾱ_t·z_0:Signal component; original latent scaled by signal retention at step t
√(1-ᾱ_t)·ε:Noise component; Gaussian noise scaled by noise level at step t
Compression: 512×512 image → 64×64 latent + 4-8 channels = 50-100x compression
Input image:Original pixel space; 512×512×3 = ~786K values
VAE encoding:Spatial reduction 8× (512→64), channels reduced from 3 to 4-8; typically 8 channels for high quality
Latent space:64×64×8 = 32,768 values; ~24x fewer than pixel space
Compression ratio:50-100x total (accounting for spatial + channel reduction); enables 10-50x speedup in diffusion sampling
VAE compresses semantically (learns important features). Diffusion then structures instead of pixels. Latent space smoother, more stable.
VAE quality critical: Poor VAE = lost details in latent diffusion. Cannot generate outside VAE training distribution (latent space has boundaries).
Latent scaling (0.18 in Stable Diffusion) important. Posterior scaling matches diffusion model's expected statistics. Fine-tune diffusion, freeze VAE.
Text encoders (CLIP, T5) condition via cross-attention. Conditioning orthogonal to latent compression benefit. Can swap independently.
Production rule: Use pre-trained Stable Diffusion VAE (hard to train). Quantize: fp16 memory reduction, int8 for mobile. Monitor latent stats: mean≈0, var≈1.
Efficiency: 20-50 latent steps ≈ 100-200 pixel steps. DDIM 15-30 steps acceptable quality. Batch in latent: 40-100x faster vs pixel-diffusion.

7. VQ-VAE: Vector Quantization for Discrete Latents

K = 256 codes
More codes = better reconstruction, higher entropy
D = 64 dim

Discrete Latent Space: VQ-VAE

What it is: Replace continuous Gaussian z with discrete tokens via codebook quantization. Enables Transformer-based autoregressive generation. Two-stage: (1) tokenize via VQ-VAE, (2) autoregressive language model on tokens.

Quantize: ẑ = e_k where k = argmin_i ||z - e_i||² (nearest codebook entry)
z:Continuous latent vector from encoder; the value to be quantized
e_i:i-th entry (embedding) in the codebook; dimension matches z, one per discrete token
||z - e_i||²:L2 distance between z and each codebook entry e_i
k = argmin_i:Index of nearest codebook entry; discrete choice ∈ {0, 1, ..., K-1}
ẑ = e_k:Quantized representation; replaces continuous z with closest codebook entry (discrete token)
K:Codebook size; number of discrete tokens available; K ∈ [64, 1024] typically
Loss: ||x - dec(ẑ)||² + β₁||z - sg[e_k]||² + β₂||sg[z] - e_k||²
||x - dec(ẑ)||²:Reconstruction loss; penalizes decoder for not recovering original x from quantized latent ẑ
x:Original input
dec():Decoder network that reconstructs x̂ from latent representation
β₁||z - sg[e_k]||²:Codebook loss; moves codebook entries e_k toward encoder outputs z; sg[] = stop-gradient (no backprop to encoder)
β₁:Weight for codebook loss term; typically β₁=0.99; prevents codebook collapse
sg[] (stop-gradient):Detaches gradients; prevents backpropagation through this term; critical for stable training
β₂||sg[z] - e_k||²:Encoder commitment loss; pulls encoder outputs z toward selected codebook entries; sg[z] prevents codebook gradient
β₂:Weight for commitment loss; typically β₂=0.25; balances learning speed of encoder vs codebook
Discrete tokens = H×W grid of K-way choices. Transformers better at scaling than diffusion on tokens. Used in DALL-E, Jukebox.
Gradient flow: Straight-through estimator needed (argmin not differentiable). Encourages encoder→codebook (term 2), codebook→encoder (term 3 with EMA).
Codebook collapse: if K too large, some embeddings unused (reduce effective quantization). Use EMA updates + per-embedding loss penalty.
Entropy bottleneck: each token carries log K bits. Too small K = poor reconstruction. Too large K = entropy unbounded.
Production rule: K ∈ [512, 8192], 4096 typical. EMA updates: e_i = (m·e_i + n(a)·z_a)/(m + n(a) + ε). Monitor usage: if <80% used, increase K.
Two-stage benefits: Lightweight (int32 tokens). Flexible (swap tokenizer or LM independently). Slow generation (hundreds forward passes for HW tokens). Uses Transformer scaling laws.

8. U-Net vs DiT: Architecture for Diffusion

U-Net vs Diffusion Transformer (DiT)

U-Net: Conv encoder-decoder + skip connections. Local inductive bias (spatial structure). Time via FiLM, condition via cross-attention. Fast, proven, ~0.5-1 sec for 512×512.

DiT: Treats spatial as tokens (HW patches). Global receptive field (any→any attention). Self-attention + cross-attention. Scales better with compute (Transformer laws). Used Stable Diffusion 3, slower but higher quality.

U-Net
O(H·W·C) memory. Logarithmic receptive field. Local bias = efficient. Skip preserve details. Training fast, quality plateaus with scale.
DiT
O((HW)² · D) memory (quadratic bottleneck). Global receptive field. More compute but scales like LLMs. Init critical for stability.
U-Net skip design matters. Poor skips limit quality. DiT quadratic complexity prohibitive for 4K (use sparse/linear attention, pruning).
U-Net: improvements plateau with size. DiT: continues improving (LLM-like scaling). DiT = future-proof but engineering-intensive.
Production: <512×512 latency-critical: U-Net (0.5 sec with DDIM). >768×768 or max quality: DiT + memory opt. U-Net faster training (prototype), DiT needs tuning.
Fine-tuning: U-Net: LoRA works. DiT: LoRA + higher rank. Can mix: U-Net encoder + DiT decoder hybrid for benefits.
Trade-off: U-Net = proven, fast, efficient current resolutions. DiT = future-proof, scales better, requires more engineering. Conditioning: FiLM good U-Net, cross-attention better DiT.

Interview Tips: GANs

What it is: Generator G: z→fake. Discriminator D: real vs fake. Min-max game. Nash Eq = D can't distinguish. Fast sampling. Hard to train.

min_G max_D E[log D(x)] - E[log(1-D(G(z)))] = -2log(2) - 2·JS(p_data || p_gen)
min_G max_D:Min-max game: Generator G minimizes, Discriminator D maximizes (adversarial objectives)
D(x):Discriminator output for real data x; probability [0, 1] that input is real; ideally D(x) → 1
E[log D(x)]:Expected log-likelihood of real data; D wants to maximize (output → 1), G indifferent
G(z):Generator output; generates fake sample from random noise z ~ N(0, I); ideally indistinguishable from real
D(G(z)):Discriminator output for fake data; probability that generated sample is real; ideally D(G(z)) → 0
E[log(1-D(G(z)))]:D wants to maximize (detect fakes), G wants to minimize (fool D); when D(G(z)) → 1, this term → 0
JS(p_data || p_gen):Jensen-Shannon divergence between real and generated distributions; at Nash equilibrium = 0 (distributions match)
-2log(2) ≈ -1.39:Constant in GAN objective; Nash equilibrium value when JS divergence = 0
Wasserstein GAN: min_G max_D E[D(x)] - E[D(G(z))] s.t. ||D||_L ≤ 1 → better gradients
E[D(x)] - E[D(G(z))]:Wasserstein distance (critic network, not probability); measures distribution distance directly
D (Critic):No sigmoid; outputs real-valued score; higher for real data, lower for fake; 1-Lipschitz constraint ensures smooth gradients
||D||_L ≤ 1:Lipschitz constraint; 1-Lipschitz ensures gradients well-behaved (≤1 everywhere); prevents vanishing/exploding gradients
Practical enforcement:Weight clipping (clip weights to [-c, c]) or gradient penalty (adds regularization term); gradient penalty preferred
Generator wants D to say "real". Discriminator wants to separate. If training converges = Nash Eq = D output 1/2 everywhere (can't distinguish).
Mode collapse: Generator learns 1-few modes, ignores diversity. D overfits to G. Solutions: minibatch discrimination, unrolled D, diversity-promoting losses.
Vanishing gradients: D confident (output 0 or 1) = small gradients = training stalls. Use Wasserstein loss or Lipschitz constraint (spectral norm).
Training instability: G/D learning rates must be tuned together. No guarantee of Nash convergence (may cycle/diverge). Different archs need different schedules.
Production rule: Wasserstein + gradient penalty or spectral norm. Progressive training (4×4 → 8×8...). Monitor FID. Separate BN for G/D. Use self-attention, residual.
Trade-off: Fast sampling (<0.1 sec vs diffusion 1+ sec). Hard to train, lower quality. Use when realtime critical. StyleGAN SOTA faces. Diffusion dominates text-to-image now.

Interview Tips: Flow Models

What it is: Invertible transformations z→x via coupling layers. Exact likelihood (no lower bound like VAE). Sample z~N(0,I), compute x=f(z). Bidirectional generation.

log p(x) = log p(z) - log|det J_f| where J_f = ∂f/∂z (Jacobian)
p(x):Probability of data x; what we want to maximize (log-likelihood)
p(z):Prior distribution, typically N(0, I); probability of latent code z
f (flow):Invertible transformation z → x; must be bijective (one-to-one, onto)
J_f = ∂f/∂z:Jacobian matrix; partial derivatives of each output w.r.t. each input; size d×d where d = dimension
det J_f:Determinant of Jacobian; measures volume scaling of transformation; must be ≠ 0 for invertibility
|det J_f|:Absolute value of Jacobian determinant; always positive; quantifies how much transformation stretches/shrinks volume
log|det J_f|:Log absolute Jacobian determinant; correction term to account for volume change (change of variables formula)
Exact likelihood:No approximation; unlike VAE (ELBO lower bound), flows give true log p(x) without variational bound
Coupling: z'_2 = T_θ(z_1) * z_2 + S_θ(z_1) → Jacobian triangular, |det|=product diagonal
z = [z_1, z_2]:Latent split into two parts; z_1 kept fixed, z_2 transformed conditioned on z_1
T_θ(z_1):Transform (scale) network; outputs multiplicative factor for z_2; depends on z_1, no gradients through z_2
S_θ(z_1):Shift (translation) network; outputs additive bias for z_2; depends on z_1
z'_2 = T_θ(z_1) * z_2 + S_θ(z_1):Affine transformation of z_2 conditioned on z_1; invertible via z_2 = (z'_2 - S_θ(z_1)) / T_θ(z_1)
Jacobian triangular:Because z_1 → z'_1 identity (no change), z_2 → z'_2 affine; results in triangular Jacobian matrix
|det| = product diagonal:For triangular matrix, determinant = product of diagonal elements; diagonal = [1, 1, ..., 1, T_θ(z_1)] = T_θ(z_1)
O(d) complexity:Computing determinant = O(d) vs O(d³) for general Jacobian; huge speedup enables efficient exact likelihood
Invertibility constraint = exact likelihood but architectures restrictive. Triangular Jacobian = O(d) not O(d³) det. Bidirectional: encode x→z, modify, decode z→x.
Invertibility cost: Cannot use ReLU (non-monotonic in reverse). Coupling must be carefully partitioned. Architecture highly constrained vs diffusion/VAE.
Likelihood ≠ quality. Exact likelihood can be far from log-prob if overtrained. Monitor both likelihood + FID. Likelihood alone insufficient.
Jacobian efficiency critical. Coupling layers triangular = O(d) determinant. Naive = O(d³) via LU. Use special structure.
Architectures: Real NVP: affine coupling. Glow: invertible 1×1 convs. Neural Spline: monotonic spline couplings. Good balance expressive vs efficient.
Production rule: Monitor likelihood + FID. Multi-scale (flows at different resolutions). Variational dequantization for discrete data. Exact posterior: p(z|x) exact (no variation approx).
Trade-off: Exact likelihood + invertibility (anomaly detection, controllable gen). Slower sampling, restricted archs. Use when likelihood/invertibility crucial. Diffusion > quality+speed.

Interview Tips: β-VAE

What it is: VAE with weighted KL divergence. Higher β = more disentanglement, lower reconstruction. Trade-off is explicit and unavoidable.

$$\text{ELBO} = \mathbb{E}[\log p(x|z)] - \beta \cdot \text{KL}(q(z|x) \| p(z))$$
\(\text{ELBO}\) (Evidence Lower Bound):Log-likelihood lower bound; maximize to improve both reconstruction and latent consistency
\(\mathbb{E}[\log p(x|z)]\):Reconstruction term; expected log-likelihood of data given latent; higher = better reconstruction
x:Input data (e.g., image)
z:Latent representation; lower-dimensional encoding of x
\(\text{KL}(q(z|x) \| p(z))\):Kullback-Leibler divergence; regularization term; measures how much posterior \(q(z|x)\) deviates from prior \(p(z) \approx \mathcal{N}(0, I)\)
\(\beta\):Weighting parameter \(\in [0, \infty)\); \(\beta=1\) standard VAE, \(\beta>1\) encourages disentanglement, \(\beta<1\) prefers reconstruction
$$\text{Posterior Collapse: } q(z|x) = p(z) \text{ when } \beta \text{ too high} \rightarrow \text{encoder useless, no info in } z$$
Posterior Collapse:When KL term dominates, encoder learns to ignore x; latent z becomes independent of input, learned nothing
KL Divergence:For Gaussian VAE: \(\text{KL} = 0.5 \cdot \sum(\mu^2 + \sigma^2 - 1 - \log(\sigma^2))\); when \(\sigma^2 \to 1\), \(\mu \to 0\), \(\text{KL} \to 0\) (collapse)
β=1 baseline. High β = disentangled (each z dimension independent). Low β = good reconstruction. Can't have both.
When to Use
VAE (Standard, β=1)
When you need smooth latent interpolation and reconstruction quality matters. E.g., image editing, face morphing, data augmentation. Balanced reconstruction-regularization tradeoff.
β-VAE (β>1)
When interpretable, disentangled representations are critical. E.g., drug discovery (isolate molecular properties), style transfer (separate content/style), scientific data analysis where each latent should encode one factor.
VAE with β<1
When reconstruction fidelity is paramount over latent structure. E.g., autoencoder-style compression, pre-training for downstream tasks where latent quality matters less than output quality.
Disentanglement: Each latent dimension encodes independent factors. E.g., one dim = style, another = content. Interpretable.
Tuning β: critical, often overlooked. Wrong β gives fuzzy reconstructions (low β) or collapsed latents (high β).
👉 Try this: Set β to 0.1 and click Train Step—reconstructions stay sharp but latent becomes spaghetti. Set β to 1.0—smoother interpolation, slight blur. Set to 2.0—clean Gaussian but fuzzy outputs. The sweet spot (usually β=0.3-0.7) balances both.
Posterior collapse: if KL → 0 after few epochs, you have collapse. Use KL warmup schedule to prevent.
👉 Try this: Watch the KL Divergence stat as you train. If it drops to 0 (encoder ignored), collapse happened. Increase Latent Dimension or adjust β warmup to prevent. Collapse means z ≈ N(0,1) regardless of x—the encoder learned nothing.
Production rule: Start β=1, tune for your task. Use KL annealing. Evaluate disentanglement via downstream classification, not visual inspection. Gaussian prior for smooth interpolation. 
import torch
                        import torch.nn as nn
                        import torch.nn.functional as F
                        
                        class VAE(nn.Module):
                            def __init__(self, dim=784, latent=20):
                                super().__init__()
                                self.encoder = nn.Linear(dim, 128)
                                self.mu = nn.Linear(128, latent)
                                self.logvar = nn.Linear(128, latent)
                                self.decoder = nn.Sequential(
                                    nn.Linear(latent, 128), nn.ReLU(), nn.Linear(128, dim))
                        
                            def forward(self, x):
                                h = F.relu(self.encoder(x.view(x.size(0), -1)))
                                mu, logvar = self.mu(h), self.logvar(h)
                                z = mu + torch.randn_like(mu) * (logvar * 0.5).exp()
                                return self.decoder(z), mu, logvar

Reparameterization trick: z = mu + eps * sigma enables backprop through sampling.

Interview Tips: Diffusion Models

What it is: Learn to reverse noise addition. Forward adds noise T steps: \(x_t = \sqrt{\bar{\alpha}_t} \cdot x_0 + \sqrt{1-\bar{\alpha}_t} \cdot \epsilon\). Reverse: \(x_t \to x_{t-1}\) iteratively. Stable vs GANs. High quality samples.

$$\text{Forward: } q(x_t|x_0) = \mathcal{N}(\sqrt{\bar{\alpha}_t} \cdot x_0,\; (1-\bar{\alpha}_t)\mathbf{I}) \quad|\quad \text{Reverse: denoise } \epsilon \sim \mathcal{N}(0, \mathbf{I})$$
q(xt|x0):Forward process distribution; conditionally Gaussian given clean x0
x0:Clean original data
xt:Noisy data at diffusion step t; interpolation between x0 and pure noise
\(\bar{\alpha}_t\) (alpha_bar_t):Cumulative product of noise schedule; starts at \(\approx 1\) (t=0, clean), approaches 0 (t=T, full noise); \(\bar{\alpha}_t = \prod_{i=1}^t \alpha_i\) where \(\alpha_i = 1 - \beta_i\)
\(\beta_i\):Noise variance at step i; \(\beta_i \in [0, 0.02]\) typically; defines how much noise added at each step
\(\sqrt{\bar{\alpha}_t} \cdot x_0\):Signal coefficient; how much original signal remains at step t; decreases over time
(1-ᾱ_t)I:Noise variance (covariance matrix); noise increases over time as ᾱ_t decreases
\(\epsilon \sim \mathcal{N}(0,I)\):Standard Gaussian noise used in forward process; mean 0, variance 1
$$\mathcal{L} = \|\epsilon_\theta(x_t, t) - \epsilon\|^2 \;\rightarrow\; \text{predict added noise at step } t$$
\(\epsilon_\theta(x_t, t)\):Neural network prediction of noise at step t given noisy input x_t; outputs estimated noise \(\hat{\epsilon}\)
\(\epsilon\):Ground truth noise that was added in forward process; sampled fresh for each training example
\(\|\cdot\|^2\):Mean squared error (L2 loss); measures pixel-level reconstruction error between predicted and actual noise
\(\theta\):Neural network parameters (typically UNet or transformer); learned during training
Closed-form diffusion: add noise → learn to reverse → sample via T denoising steps. More stable than adversarial training.
When to Use
Diffusion Models
When sample quality is top priority and latency is acceptable (1-10 sec). E.g., text-to-image (DALL-E 2, Stable Diffusion), high-fidelity image synthesis, inpainting, super-resolution. Best FID scores among generative models.
GANs Instead
When real-time generation is required (<100ms). E.g., video game texture synthesis, real-time style transfer, interactive applications. Faster sampling but harder to train and lower diversity.
Flow Models Instead
When exact likelihood computation is needed. E.g., anomaly detection, density estimation, scientific applications requiring calibrated probabilities. Slower but mathematically exact.
T (diffusion steps): T=1000 high quality but slow. Trade-off: quality vs latency. DDIM/DPM-Solver enable fast sampling (few steps).
Variance schedule (linear, cosine, etc.) tunes quality. Wrong schedule = poor samples. Critical tuning knob.
👉 Try this: Select Cosine schedule on the dropdown—watch Signal Ratio and Noise Ratio evolve. Cosine spends more time at high SNR (fine details). Switch to Linear—uniform noise addition. Try both with Timestep slider to see signal preservation differences.
Score matching = predict noise, not mean directly. More stable training than explicit mean prediction.
👉 Try this: As you animate, the network predicts ε at each step t. That's score matching (noise prediction). It's more stable than predicting the mean μ directly. Try T=1000 (many steps, high quality, slow) vs T=50 (few steps, faster, slight quality loss).
Production rule: Use DDIM/DPM-Solver for speed. Store noise schedule (reproducibility). Monitor FID/IS scores. Slower than GANs but more stable (trade latency for reliability).
Conditioning: Classifier-free guidance common for control but can be unstable. Fine-tuning via inpainting/text-to-image needs specific procedures. 
import torch
                        
                        def linear_schedule(T=1000, beta_start=1e-4, beta_end=0.02):
                            return torch.linspace(beta_start, beta_end, T)
                        
                        def forward_diffusion(x0, t, betas):
                            alpha_bar = (1 - betas).cumprod(dim=0)[t]
                            noise = torch.randn_like(x0)
                            xt = alpha_bar.sqrt() * x0 + (1 - alpha_bar).sqrt() * noise
                            return xt, noise

Interview Tips: GANs

What it is: Generator G: z→fake. Discriminator D: real vs fake. Min-max game. Nash Eq = D can't distinguish. Fast sampling. Hard to train.

$$\min_G \max_D \;\mathbb{E}[\log D(x)] - \mathbb{E}[\log(1-D(G(z)))] = -2\log(2) - 2 \cdot \text{JS}(p_{\text{data}} \| p_{\text{gen}})$$
min_G max_D:Min-max game: Generator G minimizes, Discriminator D maximizes (adversarial objectives)
D(x):Discriminator output for real data x; probability [0, 1] that input is real; ideally D(x) → 1
E[log D(x)]:Expected log-likelihood of real data; D wants to maximize (output → 1), G indifferent
G(z):Generator output; generates fake sample from random noise z ~ N(0, I); ideally indistinguishable from real
D(G(z)):Discriminator output for fake data; probability that generated sample is real; ideally D(G(z)) → 0
E[log(1-D(G(z)))]:D wants to maximize (detect fakes), G wants to minimize (fool D); when D(G(z)) → 1, this term → 0
JS(p_data || p_gen):Jensen-Shannon divergence between real and generated distributions; at Nash equilibrium = 0 (distributions match)
\(-2\log(2) \approx -1.39\):Constant in GAN objective; Nash equilibrium value when JS divergence = 0
$$\text{WGAN: } \min_G \max_D \;\mathbb{E}[D(x)] - \mathbb{E}[D(G(z))] \;\text{s.t. } \|D\|_L \leq 1 \;\rightarrow\; \text{better gradients}$$
E[D(x)] - E[D(G(z))]:Wasserstein distance (critic network, not probability); measures distribution distance directly
D (Critic):No sigmoid; outputs real-valued score; higher for real data, lower for fake; 1-Lipschitz constraint ensures smooth gradients
\(\|D\|_L \leq 1\):Lipschitz constraint; 1-Lipschitz ensures gradients well-behaved (≤1 everywhere); prevents vanishing/exploding gradients
Practical enforcement:Weight clipping (clip weights to [-c, c]) or gradient penalty (adds regularization term); gradient penalty preferred
Generator wants D to say "real". Discriminator wants to separate. If training converges = Nash Eq = D output 1/2 everywhere (can't distinguish).
When to Use
Standard GAN
When real-time generation is critical and quality can be moderate. E.g., live video filters, game asset generation, real-time style transfer. Sub-100ms sampling is the key advantage over diffusion.
WGAN / WGAN-GP
When training stability matters more than speed. E.g., medical image synthesis, scientific data generation where mode coverage is critical. Wasserstein distance provides meaningful loss signal and gradient stability.
StyleGAN
When controllable high-resolution face/object generation is needed. E.g., synthetic data for privacy-preserving datasets, digital art tools, latent space editing. SOTA for faces; progressive training enables 1024x1024+.
Mode collapse: Generator learns 1-few modes, ignores diversity. D overfits to G. Solutions: minibatch discrimination, unrolled D, diversity-promoting losses.
Vanishing gradients: D confident (output 0 or 1) = small gradients = training stalls. Use Wasserstein loss or Lipschitz constraint (spectral norm).
👉 Try this: Standard GAN (BCE loss) vanishes gradients when D confidence = 0 or 1. Wasserstein GAN (WGAN) = Earth-Mover distance, smoother gradients. Toggle on canvas: watch vanilla GAN mode coverage collapse while WGAN stays stable. Spectral normalization enforces 1-Lipschitz discriminator.
Training instability: G/D learning rates must be tuned together. No guarantee of Nash convergence (may cycle/diverge). Different archs need different schedules.
👉 Try this: Set Generator LR=0.0002, Discriminator LR=0.0004 (typical). Balance is critical: if G too fast, D can't keep up (mode collapse). If D too fast, G starves (vanishing gradients). Use spectral norm, instance norm, gradient penalties to stabilize. Monitor loss ratio.
Production rule: Wasserstein + gradient penalty or spectral norm. Progressive training (4×4 → 8×8...). Monitor FID. Separate BN for G/D. Use self-attention, residual.
Trade-off: Fast sampling (<0.1 sec vs diffusion 1+ sec). Hard to train, lower quality. Use when realtime critical. StyleGAN SOTA faces. Diffusion dominates text-to-image now. 
import torch
                        import torch.nn as nn
                        
                        class Generator(nn.Module):
                            def __init__(self, z_dim=100, out_dim=784):
                                super().__init__()
                                self.net = nn.Sequential(
                                    nn.Linear(z_dim, 256), nn.ReLU(),
                                    nn.Linear(256, out_dim), nn.Tanh())
                            def forward(self, z):
                                return self.net(z)
                        
                        class Discriminator(nn.Module):
                            def __init__(self, in_dim=784):
                                super().__init__()
                                self.net = nn.Sequential(
                                    nn.Linear(in_dim, 256), nn.LeakyReLU(0.2),
                                    nn.Linear(256, 1), nn.Sigmoid())
                            def forward(self, x):
                                return self.net(x)
                        
                        G, D = Generator(), Discriminator()
                        z = torch.randn(32, 100)
                        fake = G(z)              # generate fake samples
                        score = D(fake.detach())  # discriminator scores

Train D on real+fake, then G to maximize D(G(z)). Use detach() to stop gradients to G when training D.

0 = unconditional, 1 = conditional only, >1 = amplified

How It Works: Classifier-Free Guidance

What it is: Steer generation without separate classifier. Reweight conditional vs unconditional predictions. w=0 (unconditional), w=1 (conditional), w>1 (amplify condition). Enables text-to-image control.

$$\epsilon_{\text{guided}} = \epsilon_\theta(x_t, \varnothing) + w \cdot (\epsilon_\theta(x_t, c) - \epsilon_\theta(x_t, \varnothing))$$
\(\epsilon_\theta(x_t, \varnothing)\):Network prediction for unconditional generation (condition c is None/dropped); baseline noise estimate
\(\epsilon_\theta(x_t, c)\):Network prediction for conditional generation given condition c (e.g., text prompt, image class)
c:Conditioning signal (e.g., text embedding from CLIP, class embedding); modulates generation direction
\(w\) (guidance scale):Scaling factor \(\in [0, \infty)\); w=0 = unconditional only, w=1 = conditional only, w>1 = emphasize condition; w=7.5 typical, w>15 = artifacts
\(\epsilon_{\text{guided}}\):Final guided noise prediction combining unconditional baseline with conditional offset amplified by w
Interpretation:w=0 ignores condition, w=1 only listens to condition, w=7.5 amplifies conditional signal 7.5x relative to unconditional
$$p(x_t|c) \propto p(x_t) \cdot \left[\frac{p(x_t|c)}{p(x_t)}\right]^w \;\rightarrow\; \text{sharpens distribution}$$
p(x_t):Unconditional distribution; prior without any guidance
p(x_t|c):Conditional distribution given condition c; posterior after observing condition
[p(x_t|c)/p(x_t)]^w:Likelihood ratio raised to power w; reweights samples toward condition; higher w = stronger reweighting
Sharp distribution:Increasing w makes distribution more concentrated on high-likelihood samples; less diversity, higher fidelity to condition
w=7.5 is Stable Diffusion default. w>15 = artifacts. Trade w for diversity-vs-fidelity. Cannot invent info not in training.
When to Use
Low Guidance (w=1-5)
When diversity and creative variation matter. E.g., artistic image generation, brainstorming visual concepts, generating training data augmentations. Samples are diverse but less adherent to prompt.
Medium Guidance (w=7-12)
Sweet spot for most production use cases. E.g., text-to-image products (Stable Diffusion default w=7.5), photorealistic generation, marketing content. Balances fidelity to prompt with visual quality.
High Guidance (w=12-20)
When strict adherence to condition is critical despite quality tradeoff. E.g., precise text rendering, logo generation, technical diagrams. Risk of artifacts and oversaturation above w=15.
Training requirement: Randomly drop condition c with prob ~10% during training so model learns both conditional + unconditional denoising. Critical!
High w (w>15) causes mode collapse, artifacts, saturated colors, over-sharpening. Guidance cannot "invent" outside training data.
👉 Try this: Set w=1.0 and generate—natural samples respecting condition. w=7.0—sharp, committed samples. w=20.0—oversaturated, distorted. Guidance amplifies what the model learned; it can't "imagine" beyond training data. Typical range: w=7-15 for best results.
Guidance is orthogonal to likelihood. Doesn't improve likelihood, just reweights sampling distribution toward condition.
👉 Try this: Guidance reweights—doesn't improve the model itself, just changes sampling strategy toward the condition. Think: "which samples from the model match the condition best?" Higher w = stricter filtering. Monitor diversity alongside quality.
Production rule: Start w=1.0, increase gradually. Expose w in UI. Monitor diversity (LPIPS) + adherence (CLIP). Combine with negative prompts, LoRA, prompt weights.
Task-specific: Photorealistic: w=7-12. Artistic: w=4-8. One-dimensional control; multidimensional requires other methods. 
import torch
                        
                        def guided_denoise(model, xt, t, cond, w=7.5):
                            """Classifier-free guidance: blend conditional + unconditional."""
                            eps_uncond = model(xt, t, cond=None)     # unconditional
                            eps_cond = model(xt, t, cond=cond)       # conditional
                            eps_guided = eps_uncond + w * (eps_cond - eps_uncond)
                            return eps_guided

6. Latent Diffusion & Stable Diffusion

Diffusion in latent space (not pixel space) enables fast generation. Increase Latent Scale to compress representations; the VAE reconstructs after denoising. Watch reconstruction quality vs speed tradeoff. CLIP text embeddings condition the process—adjust to see semantic control.

50 steps
More steps = better quality, slower. 20-100 typical.

Pipeline: Encode → Diffuse → Decode

What it is: Diffuse in compressed latent space not pixels. 3-stage: (1) VAE encode x→z (8-16x spatial reduction), (2) diffuse in z, (3) VAE decode z→x̂. 10-50x speedup vs pixel-diffusion, quality maintained.

$$q(z_t | z_0) = \sqrt{\bar{\alpha}_t} \cdot z_0 + \sqrt{1-\bar{\alpha}_t} \cdot \epsilon \quad \text{(same form, compressed space)}$$
\(z_0\):Latent representation from VAE encoder; compressed version of original image x_0; shape ≈ H/8 × W/8 × D where D ∈ [4, 8]
z_t:Latent at diffusion step t; noisy version of z_0; same shape as z_0 but with added noise
ᾱ_t:Cumulative noise schedule in latent space; same as pixel-diffusion but operates on compressed features
\(\sqrt{\bar{\alpha}_t} \cdot z_0\):Signal component; original latent scaled by signal retention at step t
\(\sqrt{1-\bar{\alpha}_t} \cdot \epsilon\):Noise component; Gaussian noise scaled by noise level at step t
$$512 \times 512 \text{ image} \rightarrow 64 \times 64 \text{ latent} + 4\text{-}8 \text{ channels} = 50\text{-}100\times \text{ compression}$$
Input image:Original pixel space; 512×512×3 = ~786K values
VAE encoding:Spatial reduction 8× (512→64), channels reduced from 3 to 4-8; typically 8 channels for high quality
Latent space:64×64×8 = 32,768 values; ~24x fewer than pixel space
Compression ratio:50-100x total (accounting for spatial + channel reduction); enables 10-50x speedup in diffusion sampling
VAE compresses semantically (learns important features). Diffusion then structures instead of pixels. Latent space smoother, more stable.
When to Use
Latent Diffusion
Default choice for production image generation. 10-50x faster than pixel-space diffusion. E.g., Stable Diffusion, SDXL for text-to-image, inpainting, img2img. Use when balancing quality and compute cost.
Pixel-Space Diffusion
When maximum quality is needed and compute is unlimited. E.g., research benchmarks, small-image tasks (32x32 CIFAR), or when VAE artifacts are unacceptable. Slower but avoids VAE compression artifacts.
Cascaded Models
When generating very high resolution (1024x1024+). E.g., Imagen uses base model + super-resolution stages. Combines latent efficiency at low res with pixel-level refinement at high res.
VAE quality critical: Poor VAE = lost details in latent diffusion. Cannot generate outside VAE training distribution (latent space has boundaries).
Latent scaling (0.18 in Stable Diffusion) important. Posterior scaling matches diffusion model's expected statistics. Fine-tune diffusion, freeze VAE.
👉 Try this: Adjust latent scale on slider—too small, noisy latents; too large, saturated colors. 0.18 is Stable Diffusion's magic number. The VAE encoder/decoder is frozen (pre-trained); only the diffusion U-Net is trained. This separation makes fine-tuning efficient.
Text encoders (CLIP, T5) condition via cross-attention. Conditioning orthogonal to latent compression benefit. Can swap independently.
👉 Try this: CLIP embeddings inject semantic control via cross-attention. Swap to T5 (better language understanding)—different semantic space, same diffusion. Test text control: change prompt semantics. The text encoder is modular—swap for better ones.
Production rule: Use pre-trained Stable Diffusion VAE (hard to train). Quantize: fp16 memory reduction, int8 for mobile. Monitor latent stats: mean≈0, var≈1.
Efficiency: 20-50 latent steps ≈ 100-200 pixel steps. DDIM 15-30 steps acceptable quality. Batch in latent: 40-100x faster vs pixel-diffusion.

7. VQ-VAE: Vector Quantization for Discrete Latents

Adjust Codebook Size (K)—more codewords = more expressive but entropy-bounded. Watch reconstruction quality vs codebook utilization. Lower K = information bottleneck (some samples map to same codeword). Increase K—utilization improves, but risk collapse if K too large.

K = 256 codes
More codes = better reconstruction, higher entropy
D = 64 dim

Discrete Latent Space: VQ-VAE

What it is: Replace continuous Gaussian z with discrete tokens via codebook quantization. Enables Transformer-based autoregressive generation. Two-stage: (1) tokenize via VQ-VAE, (2) autoregressive language model on tokens.

$$\hat{z} = e_k \;\text{where}\; k = \arg\min_i \|z - e_i\|^2 \quad \text{(nearest codebook entry)}$$
z:Continuous latent vector from encoder; the value to be quantized
e_i:i-th entry (embedding) in the codebook; dimension matches z, one per discrete token
\(\|z - e_i\|^2\):L2 distance between z and each codebook entry e_i
\(k = \arg\min_i\):Index of nearest codebook entry; discrete choice \(\in \{0, 1, \ldots, K-1\}\)
ẑ = e_k:Quantized representation; replaces continuous z with closest codebook entry (discrete token)
\(K\):Codebook size; number of discrete tokens available; \(K \in [64, 1024]\) typically
$$\mathcal{L} = \|x - \text{dec}(\hat{z})\|^2 + \beta_1\|z - \text{sg}[e_k]\|^2 + \beta_2\|\text{sg}[z] - e_k\|^2$$
\(\|x - \text{dec}(\hat{z})\|^2\):Reconstruction loss; penalizes decoder for not recovering original x from quantized latent ẑ
x:Original input
dec():Decoder network that reconstructs x̂ from latent representation
\(\beta_1\|z - \text{sg}[e_k]\|^2\):Codebook loss; moves codebook entries e_k toward encoder outputs z; sg[] = stop-gradient (no backprop to encoder)
\(\beta_1\):Weight for codebook loss term; typically \(\beta_1=0.99\); prevents codebook collapse
sg[] (stop-gradient):Detaches gradients; prevents backpropagation through this term; critical for stable training
\(\beta_2\|\text{sg}[z] - e_k\|^2\):Encoder commitment loss; pulls encoder outputs z toward selected codebook entries; sg[z] prevents codebook gradient
\(\beta_2\):Weight for commitment loss; typically \(\beta_2=0.25\); balances learning speed of encoder vs codebook
Discrete tokens = H×W grid of K-way choices. Transformers better at scaling than diffusion on tokens. Used in DALL-E, Jukebox.
When to Use
VQ-VAE + Autoregressive
When leveraging Transformer scaling laws for generation. E.g., DALL-E 1 (text-to-image via discrete tokens), Jukebox (music generation), video tokenization. Enables LLM-style generation on non-text modalities.
Continuous VAE Instead
When smooth interpolation in latent space is needed. E.g., face morphing, latent editing, style mixing. Continuous z allows gradient-based optimization in latent space. Simpler training (no codebook collapse).
RQ-VAE / FSQ
When codebook collapse is a problem or higher compression needed. Residual quantization stacks multiple codebooks for finer granularity. Finite Scalar Quantization avoids learned codebooks entirely. E.g., audio codecs (SoundStream, Encodec).
Gradient flow: Straight-through estimator needed (argmin not differentiable). Encourages encoder→codebook (term 2), codebook→encoder (term 3 with EMA).
Codebook collapse: if K too large, some embeddings unused (reduce effective quantization). Use EMA updates + per-embedding loss penalty.
👉 Try this: Set K=512 (reasonable). Watch utilization %—good coverage. Increase to K=2048—some codewords unused, collapse risk. Decrease to K=64—forced quantization, lower quality. K=256-512 typical for most datasets. Monitor: dead codewords = wasted capacity.
Entropy bottleneck: each token carries log K bits. Too small K = poor reconstruction. Too large K = entropy unbounded.
👉 Try this: K=64 → log(64)=6 bits per code. K=256 → 8 bits. K=1024 → 10 bits. Fewer bits = compression but info loss. More bits = fidelity but less compression. Tradeoff is fundamental. Choose K to balance your latent size constraint vs quality needs.
Production rule: \(K \in [512, 8192]\), 4096 typical. EMA updates: e_i = (m·e_i + n(a)·z_a)/(m + n(a) + ε). Monitor usage: if <80% used, increase K.
Two-stage benefits: Lightweight (int32 tokens). Flexible (swap tokenizer or LM independently). Slow generation (hundreds forward passes for HW tokens). Uses Transformer scaling laws. 
import torch
                        import torch.nn as nn
                        
                        class VectorQuantizer(nn.Module):
                            def __init__(self, K=256, D=64):
                                super().__init__()
                                self.codebook = nn.Embedding(K, D)
                        
                            def forward(self, z):
                                # z: (B, D) continuous latent
                                dist = torch.cdist(z.unsqueeze(1), self.codebook.weight.unsqueeze(0))
                                indices = dist.argmin(dim=-1).squeeze(1)  # nearest code
                                z_q = self.codebook(indices)
                                # Straight-through estimator: copy gradients
                                z_q_st = z + (z_q - z).detach()
                                return z_q_st, indices

Straight-through estimator copies gradients past the argmin; codebook updated via EMA or loss term.

8. U-Net vs DiT: Architecture for Diffusion

Toggle between U-Net (CNN-based) and DiT (Transformer-based). Watch latency and FLOP count. U-Net is deployed widely (fast), DiT is research (quadratic complexity). For 4K+ resolution, U-Net with sparse attention beats DiT. Scale-wise, DiT follows LLM-like improvements; U-Net plateaus.

U-Net vs Diffusion Transformer (DiT)

U-Net: Conv encoder-decoder + skip connections. Local inductive bias (spatial structure). Time via FiLM, condition via cross-attention. Fast, proven, ~0.5-1 sec for 512×512.

DiT: Treats spatial as tokens (HW patches). Global receptive field (any→any attention). Self-attention + cross-attention. Scales better with compute (Transformer laws). Used Stable Diffusion 3, slower but higher quality.

U-Net
O(H·W·C) memory. Logarithmic receptive field. Local bias = efficient. Skip preserve details. Training fast, quality plateaus with scale.
DiT
\(O((HW)^2 \cdot D)\) memory (quadratic bottleneck). Global receptive field. More compute but scales like LLMs. Init critical for stability.
When to Use
U-Net Architecture

                        class ResBlock(nn.Module):
                            def __init__(self, ch, t_dim):
                                super().__init__()
                                self.conv1 = nn.Conv2d(ch, ch, 3, padding=1)
                                self.conv2 = nn.Conv2d(ch, ch, 3, padding=1)
                                self.time_mlp = nn.Linear(t_dim, ch)  # FiLM conditioning
                                self.norm = nn.GroupNorm(8, ch)
                        
                            def forward(self, x, t_emb):
                                h = torch.relu(self.norm(self.conv1(x)))
                                h = h + self.time_mlp(t_emb).unsqueeze(-1).unsqueeze(-1)
                                h = torch.relu(self.norm(self.conv2(h)))
                                return x + h  # residual connection

Interview Tips: Flow Models

What it is: Invertible transformations z→x via coupling layers. Exact likelihood (no lower bound like VAE). Sample z~N(0,I), compute x=f(z). Bidirectional generation.

$$\log p(x) = \log p(z) - \log|\det J_f| \;\text{where}\; J_f = \frac{\partial f}{\partial z} \;\text{(Jacobian)}$$
p(x):Probability of data x; what we want to maximize (log-likelihood)
p(z):Prior distribution, typically N(0, I); probability of latent code z
f (flow):Invertible transformation z → x; must be bijective (one-to-one, onto)
\(J_f = \frac{\partial f}{\partial z}\):Jacobian matrix; partial derivatives of each output w.r.t. each input; size d×d where d = dimension
det J_f:Determinant of Jacobian; measures volume scaling of transformation; must be ≠ 0 for invertibility
|det J_f|:Absolute value of Jacobian determinant; always positive; quantifies how much transformation stretches/shrinks volume
log|det J_f|:Log absolute Jacobian determinant; correction term to account for volume change (change of variables formula)
Exact likelihood:No approximation; unlike VAE (ELBO lower bound), flows give true \(\log p(x)\) without variational bound
$$z'_2 = T_\theta(z_1) \cdot z_2 + S_\theta(z_1) \;\rightarrow\; \text{Jacobian triangular, } |\det| = \prod \text{diagonal}$$
z = [z_1, z_2]:Latent split into two parts; z_1 kept fixed, z_2 transformed conditioned on z_1
\(T_\theta(z_1)\):Transform (scale) network; outputs multiplicative factor for z_2; depends on z_1, no gradients through z_2
\(S_\theta(z_1)\):Shift (translation) network; outputs additive bias for z_2; depends on z_1
\(z'_2 = T_\theta(z_1) \cdot z_2 + S_\theta(z_1)\):Affine transformation of z_2 conditioned on z_1; invertible via \(z_2 = (z'_2 - S_\theta(z_1)) / T_\theta(z_1)\)
Jacobian triangular:Because z_1 → z'_1 identity (no change), z_2 → z'_2 affine; results in triangular Jacobian matrix
\(|\det| = \prod\) diagonal:For triangular matrix, determinant = product of diagonal elements; diagonal = \([1, 1, \ldots, 1, T_\theta(z_1)] = T_\theta(z_1)\)
\(O(d)\) complexity:Computing determinant = O(d) vs \(O(d^3)\) for general Jacobian; huge speedup enables efficient exact likelihood
Invertibility constraint = exact likelihood but architectures restrictive. Triangular Jacobian = O(d) not O(d³) det. Bidirectional: encode x→z, modify, decode z→x.
When to Use
Flow-Based Models
When exact likelihood is required (not just a lower bound). E.g., anomaly detection (threshold on log p(x)), density estimation for scientific data, lossless compression. Bidirectional mapping enables both generation and inference.
Diffusion Instead
When sample quality matters more than exact likelihood. E.g., image/video generation, creative applications. Diffusion has fewer architectural constraints and produces higher-quality samples at the cost of approximate likelihood.
VAE Instead
When you need a latent space for downstream tasks and exact likelihood is not critical. E.g., representation learning, data augmentation, latent-space optimization. Simpler architecture with ELBO approximation.
Invertibility cost: Cannot use ReLU (non-monotonic in reverse). Coupling must be carefully partitioned. Architecture highly constrained vs diffusion/VAE.
Likelihood ≠ quality. Exact likelihood can be far from log-prob if overtrained. Monitor both likelihood + FID. Likelihood alone insufficient.
👉 Try this: Flow models compute exact likelihood (Jacobian determinant). High likelihood doesn't guarantee good samples (overfitting to low-probability regions). Jacobian efficiency critical. Coupling layers triangular = O(d) determinant. Naive = O(d³) via LU. Use special structure.

Architectures: Real NVP: affine coupling. Glow: invertible 1×1 convs. Neural Spline: monotonic spline couplings. Good balance expressive vs efficient.
Production rule: Monitor likelihood + FID. Multi-scale (flows at different resolutions). Variational dequantization for discrete data. Exact posterior: p(z|x) exact (no variation approx).

import torch
                        import torch.nn as nn
                        
                        class AffineCoupling(nn.Module):
                            def __init__(self, dim):
                                super().__init__()
                                half = dim // 2
                                self.net = nn.Sequential(
                                    nn.Linear(half, 64), nn.ReLU(), nn.Linear(64, half * 2))
                        
                            def forward(self, x):
                                x1, x2 = x.chunk(2, dim=-1)
                                st = self.net(x1)
                                s, t = st.chunk(2, dim=-1)
                                z2 = x2 * s.exp() + t
                                log_det = s.sum(dim=-1)  # triangular Jacobian
                                return torch.cat([x1, z2], dim=-1), log_det
                        
                            def inverse(self, z):
                                z1, z2 = z.chunk(2, dim=-1)
                                st = self.net(z1)
                                s, t = st.chunk(2, dim=-1)
                                x2 = (z2 - t) * (-s).exp()
                                return torch.cat([z1, x2], dim=-1)

Affine coupling: half of dims fixed, half transformed. Triangular Jacobian gives O(d) determinant.

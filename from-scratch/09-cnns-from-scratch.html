9.1 The Problem: Why MLPs Fail on Images

An MLP flattens a 2D image into a 1D vector, destroying all spatial structure. Shift a pattern by 2 pixels and the MLP sees a completely different input.

Active pixel
Inactive pixel
Changed input positions
Key Insight: MLPs have no spatial awareness. A pixel at position (0,0) and (2,0) are completely unrelated parameters to an MLP.
$$\text{Flatten: } 7 \times 7 \text{ image} \rightarrow 49\text{-dim vector} \rightarrow \text{spatial relationships lost}$$
The Fundamental Problem: Shifting an image by 2 pixels changes almost every element of the flattened vector. The MLP must learn every possible translation separately.
Parameter Explosion: A 224×224×3 image = 150,528 inputs. A single hidden layer of 1000 neurons needs 150M weights just for the first layer.
When asked "why use CNNs instead of MLPs for images?", lead with two things: (1) spatial structure preservation and (2) parameter sharing. MLPs need separate weights for every pixel position.
MLPs see a flat list of numbers. CNNs see a grid of neighbors. That is the entire difference.
💡 Real-world: In 2012, AlexNet (a CNN) cut ImageNet error from 26% to 16%, shocking the AI field. Within 2 years, every phone camera, self-driving car prototype, and medical imaging system adopted CNNs. It's the architecture that started the deep learning revolution.

⚠️ Common Mistakes

  • Mistake: Thinking MLPs "cannot" learn image tasks. They can, but need exponentially more data and parameters to learn spatial patterns that CNNs get for free. Fix: Understand that CNNs encode an inductive bias (locality + translation invariance) that MLPs lack.
  • Mistake: Applying an MLP to raw pixels of a large image. Fix: Always use convolutional layers for spatial data. Reserve MLPs (fully-connected layers) for the classification head after spatial features have been extracted.
Running Example: Throughout this chapter, we will follow a tiny 8×8 handwritten digit as it passes through each stage of a CNN. Watch how raw pixel values transform into abstract features, then into a class prediction.

9.2 Convolution: Sliding a Filter

A convolution slides a small kernel across the input, computing a weighted sum at each position. This is the fundamental operation of CNNs.

Edit kernel values above or use preset buttons
Key Insight: Weight sharing — the SAME 3×3 kernel is used everywhere. This is what gives CNNs translation invariance.
$$\text{output}[i,j] = \sum_{m=0}^{2} \sum_{n=0}^{2} \text{input}[i+m,\, j+n] \times \text{kernel}[m,\, n]$$
Weight Sharing: Instead of 49 separate weights per output, we have 9 shared weights. A pattern detected at position (0,0) is also detected at (3,4).
Local Connectivity: Each output pixel depends on only a 3×3 patch of input, not the entire image. This encodes the prior that nearby pixels are more relevant.
Why share weights? An edge detector learned at the top-left should work at the bottom-right too (a vertical edge looks the same everywhere). Without sharing, a 224×224 image with 64 filters would need 64 separate detectors per pixel = 3.2 billion parameters. With sharing: just 64 × 9 = 576 parameters.
Convolution is really cross-correlation in most DL frameworks (no kernel flipping). The distinction only matters for the backward pass.
Convolution = slide, multiply, sum. Same kernel everywhere = translation equivariance.
import torch
import torch.nn as nn

# Input: 1 image, 1 channel, 7x7
x = torch.randn(1, 1, 7, 7)

# Conv: 1 input channel, 1 output channel, 3x3 kernel
conv = nn.Conv2d(in_channels=1, out_channels=1, kernel_size=3)
out = conv(x)  # shape: (1, 1, 5, 5)

print(conv.weight.shape)  # (1, 1, 3, 3) = 9 parameters
print(out.shape)           # 7 - 3 + 1 = 5

nn.Conv2d applies the same kernel at every spatial position (weight sharing).

📎 The chain rule from Ch 1: Backprop applies here too. Each output pixel is a function of 9 inputs and 9 weights, so backprop computes 9 local gradients per position. We will see this in Section 9.7.

9.3 Padding & Stride

Padding adds border pixels; stride controls how far the kernel jumps. Together they determine the output size.

0
1
Key Insight: The output spatial size is fully determined by a simple formula.
$$\text{output\_size} = \left\lfloor \frac{\text{input} + 2 \times \text{padding} - \text{kernel}}{\text{stride}} \right\rfloor + 1$$
"Same" Padding: Use \(p = \lfloor(k-1)/2\rfloor\) to keep the output size equal to the input size (when stride=1). For a 3×3 kernel, padding=1.
"Valid" Padding: No padding (p=0). The output shrinks by (k-1) in each dimension. For a 3×3 kernel on 7×7 input: output is 5×5.
Stride=2 halves the spatial dimensions. This is a common way to downsample instead of pooling.

⚠️ Common Mistakes

  • Mistake: Forgetting that stride=2 halves spatial dimensions. Fix: Always use the formula. For "same" padding with stride=1, use p = (k-1)/2.
  • Mistake: Using even-sized kernels (e.g., 4×4) which make "same" padding asymmetric. Fix: Stick to odd-sized kernels (3×3, 5×5, 7×7) so padding is symmetric on both sides.
Hyperparameter Guidance: Kernel Size & Padding
  • Kernel size: 3×3 is the modern standard (VGG showed two 3×3 layers = one 5×5 receptive field with fewer parameters).
  • Padding: Almost always use "same" padding (p=1 for 3×3) to avoid losing spatial information early.
  • Stride: Use stride=2 for downsampling (replaces pooling in many modern architectures like ResNet).

9.4 Multiple Channels & Filters

Real images have multiple channels (RGB). Each filter has one kernel per input channel, and multiple filters produce multiple output channels.

4
Key Insight: Each filter learns to detect one feature (edge, texture, color). C_out filters produce C_out feature maps. The network learns WHAT to look for.
$$\text{output}[f, i, j] = \sum_{c=0}^{C_{in}-1} \sum_{m,n} \text{input}[c, i+m, j+n] \times \text{kernel}[f, c, m, n] + \text{bias}[f]$$
Parameter Count: One filter: \(C_{in} \times k \times k\) weights + 1 bias. Total: \(C_{out} \times (C_{in} \times k \times k + 1)\).
1×1 Convolutions: A 1×1 kernel mixes channels without spatial computation — like a per-pixel fully-connected layer across channels.
When asked about parameter counts: a Conv2d(64, 128, 3) has 128 * (64 * 3 * 3 + 1) = 73,856 parameters. This is independent of the spatial size of the input.
📎 The weight-sharing concept connects to Ch 3: MLPs — in an MLP, every input-output pair has its own weight. In a CNN, the same kernel weights are reused at every spatial position, which is a much stronger inductive bias.

9.5 Pooling: Spatial Reduction

Pooling reduces spatial dimensions, building a hierarchy from local features to global understanding.

Key Insight: Max pooling keeps the strongest activation in each region — if a feature (edge, corner) was detected anywhere in the 2×2 window, it survives.
Max Pooling
Keeps the strongest signal. Provides slight translation invariance within the pool window. Most common in classification.
Average Pooling
Averages all values. Smoother but can dilute strong activations. Global Average Pooling (GAP) replaces FC layers in modern nets.
Max pool = "was this feature detected anywhere in this region?" Average pool = "how much of this feature is in this region?"

⚠️ Common Mistakes

  • Mistake: Stacking many conv layers without any downsampling (no pooling, no stride>1). Feature maps stay enormous, computation explodes, and the receptive field grows too slowly. Fix: Downsample every few layers using stride=2 conv or 2×2 max pool.
Hyperparameter Guidance: Pooling
  • Pool size: 2×2 with stride 2 is the standard. Reduces each spatial dimension by half.
  • Modern trend: Replace max pooling with strided convolutions (stride=2). This lets the network learn how to downsample.
  • Global Average Pooling: At the end of the network, use GAP instead of flatten+FC. Reduces parameters dramatically.

9.6 A Full CNN: Conv → ReLU → Pool → Conv → ReLU → Pool → FC

Watch our 8×8 digit flow through a complete CNN pipeline, with feature map sizes and parameter counts at each stage.

Key Insight: A CNN progressively reduces spatial size while increasing channel depth. This trades spatial resolution for semantic richness.
$$8 \times 8 \times 1 \xrightarrow{\text{Conv1}} 6 \times 6 \times 4 \xrightarrow{\text{Pool}} 3 \times 3 \times 4 \xrightarrow{\text{Conv2}} 1 \times 1 \times 8 \xrightarrow{\text{FC}} 3 \text{ classes}$$
Conv1 Params: 4 filters × (1 × 3 × 3 + 1) = 40 parameters
Conv2 Params: 8 filters × (4 × 3 × 3 + 1) = 296 parameters
FC Params: 8 × 3 + 3 = 27 parameters. Total: 363 parameters
When asked about CNN architecture design, describe the pyramid: spatial dimensions shrink (pool/stride), channel dimensions grow (more filters). Early layers detect edges, deeper layers detect complex patterns.

⚠️ Common Mistakes

  • Mistake: Making the first FC layer too large by flattening big feature maps. A 7×7×512 feature map flattened = 25,088 inputs. If FC has 4096 outputs, that is 100M+ parameters in one layer. Fix: Use Global Average Pooling (GAP) to collapse spatial dims: 7×7×512 → 512. Then FC: 512×num_classes.
  • Mistake: Forgetting to apply ReLU between conv layers. Without activation, stacking convolutions is equivalent to a single convolution (linear composition). Fix: Always apply a nonlinearity (ReLU, GELU) after each convolution.
📎 The ReLU activations here are the same ones from Ch 1: Backprop (Section 1.4). The "dying ReLU" problem applies in CNNs too — if a filter's output is always negative, its gradient is always zero.

9.7 Backprop Through Convolutions

Gradients flow backward through conv layers. The key insight: backprop through convolution is a convolution with the flipped kernel.

Forward values
Backward gradients
Flipped kernel
Key Insight: Each output pixel was influenced by a kernel-sized patch of input. So the gradient for each input pixel accumulates contributions from all output pixels it influenced.
$$\frac{\partial L}{\partial \text{input}[i,j]} = \sum_{m,n} \frac{\partial L}{\partial \text{output}[i-m, j-n]} \times \text{kernel}[m,n]$$
Gradient w.r.t. Input: Convolve the upstream gradient (padded) with the 180°-flipped kernel. This is a "full" convolution.
Gradient w.r.t. Kernel: Convolve the input with the upstream gradient. Each kernel element's gradient sums contributions from ALL spatial positions (because of weight sharing).
The gradient of the loss w.r.t. the kernel sums over all spatial positions because the same kernel was used everywhere. Weight sharing means gradient accumulation during backprop.
import torch
import torch.nn as nn

conv = nn.Conv2d(1, 1, 3, bias=False)
x = torch.randn(1, 1, 5, 5, requires_grad=True)
out = conv(x)
loss = out.sum()
loss.backward()

# x.grad: gradient w.r.t. input (full conv with flipped kernel)
# conv.weight.grad: gradient w.r.t. kernel (sum over positions)
print(x.grad.shape)           # (1, 1, 5, 5)
print(conv.weight.grad.shape) # (1, 1, 3, 3)

PyTorch handles the kernel flipping and gradient accumulation automatically in the backward pass.

📎 This is the chain rule from Ch 1: Backprop applied to the convolution operation. The "upstream gradient × local gradient" rule is identical — the local gradient just happens to be the kernel (or flipped kernel).

9.8 Modern Architectures: From LeNet to ResNet

Three decades of CNN evolution: deeper networks, smaller kernels, skip connections. Each innovation solved a specific problem.

Key Insight: The trend is clear — deeper networks with smaller kernels and skip connections.
VGG Insight
Two 3×3 convolutions have the same receptive field as one 5×5, but with fewer parameters and more nonlinearity. Three 3×3 = one 7×7.
ResNet Insight
Skip connections let gradients flow directly through identity paths. The layer only needs to learn the residual \(F(x) = H(x) - x\). This enables 100+ layer networks.
When asked "why are deeper networks better?", explain that depth = compositional feature hierarchy. But depth without skip connections hits vanishing gradients. ResNet solved this.
💡 Real-world: ResNet-50 is the backbone of most computer vision products (Google Photos, Tesla Autopilot, medical CT analysis). Its skip connections solved the degradation problem, enabling 152-layer networks. Before ResNet, networks deeper than ~20 layers performed worse, not better.
Hyperparameter Guidance: Modern CNN Design
  • Filter sizes: 3×3 is standard (sometimes 1×1 for channel mixing, 7×7 only at the very first layer).
  • Channel progression: Double at each downsampling: 64 → 128 → 256 → 512.
  • BatchNorm: After every conv layer, before activation. Stabilizes training and allows higher learning rates.
  • Skip connections: Use residual blocks (add input to output) for any network deeper than ~10 layers.
Hyperparameter Guidance: Training CNNs
  • Learning rate: Start with 0.1 for SGD+momentum, 0.001 for Adam. Use cosine annealing or step decay.
  • Data augmentation: Random crop, horizontal flip, color jitter. This is crucial for CNNs — it artificially expands the training set with spatial transformations.
  • Weight initialization: Kaiming/He initialization for ReLU networks. Xavier/Glorot for tanh/sigmoid.
📎 Skip connections in ResNet are the same concept as residual connections in Ch 6: Transformers. The idea of learning \(F(x) = H(x) - x\) instead of \(H(x)\) directly appears throughout modern deep learning. Also see Ch 4: Attention for how self-attention layers replaced convolutions in vision transformers.

⚠️ Common Mistakes

  • Mistake: Designing very deep networks without skip connections, then wondering why training loss plateaus. Fix: Always use residual connections for networks deeper than ~10 layers. The gradient signal needs a direct path to early layers.

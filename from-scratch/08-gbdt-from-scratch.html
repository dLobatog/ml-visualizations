Chapter 8: GBDTs from Scratch

Build gradient boosted decision trees step by step, one split at a time

8.1 A Single Decision Stump

The simplest possible tree: one split. We try every feature and every threshold, pick the one that best separates the classes.

Every gradient boosted model starts with a single, humble split. Before we can boost, we need to understand how a single decision tree node works. A "stump" is a tree with depth 1 -- just one question: "is feature X above threshold T?"
Class 0
Class 1
Best Split
Key Insight: A decision stump exhaustively searches all possible splits to find the one that best separates data.
$$\text{Gain} = \text{MSE}_{\text{parent}} - \frac{n_L}{n} \text{MSE}_L - \frac{n_R}{n} \text{MSE}_R$$
Brute Force Search: For each feature, sort values and try every midpoint as a threshold. Keep the split with highest gain.
Limitation: A single stump can only create axis-aligned boundaries. XOR patterns are impossible with one split.
Decision trees are greedy: they pick the locally best split at each node. This is not globally optimal, but it is fast and works well in practice.

⚠️ Common Mistakes

  • Mistake: Using GBDT for image/text data without feature engineering. Fix: Use neural networks for unstructured data -- they learn representations automatically. GBDT excels on tabular data with hand-crafted or naturally structured features.
  • Mistake: Not setting a random seed. Fix: Always set random_state (sklearn) or seed (XGBoost) for reproducibility. Tree splits can vary between runs due to subsampling and tie-breaking.

When to Use

Decision Stumps (Depth 1)
  • AdaBoost base learners (classic weak learner)
  • Ultra-fast feature screening -- which single feature splits best?
  • When data has one dominant predictive feature
  • Example: quick triage filter (e.g., "is age > 65?" for medical screening)
Deeper Splits (Depth 2+)
  • GBDT weak learners (typical depth 3-6 per tree)
  • When feature interactions matter (XOR-like patterns need depth >= 2)
  • Standalone decision trees for interpretable models
  • Example: clinical decision rules that doctors can follow step-by-step
from sklearn.tree import DecisionTreeClassifier
import numpy as np

X = np.random.randn(100, 2)
y = (X[:, 0] + X[:, 1] > 0).astype(int)

stump = DecisionTreeClassifier(max_depth=1)
stump.fit(X, y)
print(f'Feature: {stump.tree_.feature[0]}')
print(f'Threshold: {stump.tree_.threshold[0]:.3f}')
print(f'Accuracy: {stump.score(X, y):.2f}')

max_depth=1 creates a single split -- the simplest weak learner.

8.2 Growing a Full Tree

Recursively split each leaf to grow a deeper tree. Watch how depth controls complexity -- and overfitting.

Why not just use deeper trees? More depth means more splits, which means the tree can memorize training data perfectly. But memorization is not learning. The bias-variance tradeoff is at the heart of tree-based methods.
2
Class 0
Class 1
Key Insight: Tree depth controls the bias-variance tradeoff.
$$\text{Depth 1: underfitting} \quad \longleftrightarrow \quad \text{Depth } \infty\text{: overfitting}$$
Shallow Trees: High bias, low variance. Underfits complex patterns but generalizes well.
Deep Trees: Low bias, high variance. Memorizes training data, poor on unseen data.
In boosting, we WANT weak learners (shallow trees). We will combine many weak predictions to make a strong one.

⚠️ Common Mistakes

  • Mistake: Using max_depth too high on small datasets (each leaf needs enough samples). Fix: For datasets under 10k rows, keep max_depth at 3-4. With depth 8, a tree can have 256 leaves -- if you have only 500 samples, most leaves will have 1-2 points, pure memorization.
  • Mistake: Ignoring class imbalance. Fix: Use scale_pos_weight (XGBoost) or class_weight='balanced' (sklearn) when one class significantly outnumbers the other. Without this, the model may predict the majority class for everything.

When to Use

Shallow Trees (depth 2-4)
  • GBDT / XGBoost / LightGBM weak learners
  • High-dimensional or noisy data
  • When you boost hundreds of trees
  • Example: CTR prediction with 500 features -- each tree captures simple interactions
Medium Trees (depth 5-8)
  • Random Forest or moderate GBDT
  • Balanced bias-variance on clean tabular data
  • XGBoost default (max_depth=6)
  • Example: house price prediction with 30 meaningful features
No Depth Limit
  • Random Forest only (bagging prevents overfitting)
  • Never for boosting -- will memorize training set
  • Very small, clean datasets with no noise
  • Example: RF on Iris dataset for teaching purposes
from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import make_moons

X, y = make_moons(n_samples=300, noise=0.25, random_state=42)

for depth in [1, 3, None]:  # None = unlimited
    tree = DecisionTreeClassifier(max_depth=depth, random_state=42)
    tree.fit(X, y)
    print(f'Depth={str(depth):>4s}  '
          f'Nodes={tree.tree_.node_count:>3d}  '
          f'Accuracy={tree.score(X, y):.2f}')

Unlimited depth memorizes training data (100% accuracy, poor generalization).

8.3 Computing Residuals

The key insight of gradient boosting: after fitting a model, look at what it got WRONG. The residuals \(r_i = y_i - \hat{y}_i\) tell the next tree where to focus.

This is THE key insight of boosting. Instead of fitting a new tree to the original targets, we fit it to the errors (residuals) of the current model. Each new tree corrects the mistakes of all previous trees combined. This is gradient descent in function space -- the same core idea as backpropagation in neural networks (Ch 01), but applied to an ensemble of trees instead of layer weights.
1
True function
Tree prediction
Residual arrows
Key Insight: Residuals are the negative gradient of MSE loss, pointing toward the correct answer.
$$r_i = y_i - \hat{y}_i = -\frac{\partial}{\partial \hat{y}_i} \frac{1}{2}(y_i - \hat{y}_i)^2$$
Residual = Error: Positive residual means we predicted too low. Negative means too high. The next tree learns to correct this.
Gradient Connection: For MSE loss, the residual IS the negative gradient. This is why "gradient boosting" and "fitting residuals" are the same thing.
When asked "what does gradient boosting do?", say: it iteratively fits new trees to the residuals (errors) of the current ensemble. Each tree corrects the mistakes of the previous trees.

When to Use

MSE Residuals (Regression)
  • Standard regression tasks with Gaussian-like errors
  • Residual = negative gradient of squared loss
  • Sensitive to outliers (large residuals dominate)
  • Example: house price prediction, demand forecasting
Log-Loss Gradients (Classification)
  • Binary/multi-class classification
  • Pseudo-residuals are (y - predicted probability)
  • Trees predict log-odds corrections, not class labels
  • Example: churn prediction, spam detection, medical diagnosis
Huber / Quantile Residuals
  • When data has outliers that would distort MSE
  • Quantile loss for prediction intervals (not just point estimates)
  • Huber: smooth transition between L1 and L2
  • Example: delivery time prediction with occasional extreme delays
import numpy as np
from sklearn.tree import DecisionTreeRegressor

np.random.seed(42)
X = np.linspace(0, 10, 100).reshape(-1, 1)
y = np.sin(X.ravel()) + np.random.randn(100) * 0.2

# Step 1: initial prediction = mean
y_pred = np.full_like(y, y.mean())
# Step 2: compute residuals
residuals = y - y_pred
print(f'MSE before: {np.mean(residuals**2):.3f}')
# Step 3: fit tree to residuals
tree = DecisionTreeRegressor(max_depth=2).fit(X, residuals)
y_pred += 0.1 * tree.predict(X)  # learning rate 0.1
print(f'MSE after:  {np.mean((y - y_pred)**2):.3f}')

Each boosting step fits a tree to residuals: r_i = y_i - y_hat_i.

8.4 Fitting Residuals -- One Boosting Step

Fit a new tree to the residuals, then add it to the prediction (scaled by learning rate). Watch the prediction get closer to the target.

Why use a learning rate? Adding the full residual tree (learning rate = 1.0) can overshoot. A small learning rate (0.1) means each tree makes a small, conservative correction. More trees needed, but much better generalization.

Target
Key Insight: Each boosting step adds a small correction to the current prediction.
$$\hat{y}^{(t)} = \hat{y}^{(t-1)} + \eta \cdot h_t(x)$$
\(\eta\) (Learning Rate): Shrinks each tree's contribution. Small \(\eta\) = more trees needed, but better generalization (shrinkage regularization). This serves the exact same purpose as the learning rate in SGD (Ch 03) -- controlling step size to balance convergence speed and stability.
\(h_t(x)\) (New Tree): Fitted to the residuals \(r_i = y_i - \hat{y}_i^{(t-1)}\). It predicts the correction needed.
Small LR (0.1)
Conservative updates. Needs more trees but generalizes better. Less risk of overfitting.
Large LR (1.0)
Aggressive updates. Fewer trees needed but risk of oscillation and overfitting. Try "Break It" above!

⚠️ Common Mistakes

  • Mistake: Setting learning rate too high without reducing n_estimators. Fix: Learning rate and n_estimators are inversely related. If you increase lr from 0.01 to 0.1, reduce trees from 5000 to ~500. High lr + many trees = severe overfitting.
  • Mistake: Not using early stopping. Fix: Always set early_stopping_rounds (10-50) and provide a validation set. This saves compute and automatically finds the optimal number of trees. There is no reason to skip this in production.

When to Use

Small Learning Rate (0.01-0.1)
  • Production models optimizing for every basis point of accuracy
  • Pair with early stopping and thousands of rounds
  • More robust to hyperparameter mis-specification
  • Example: ad revenue optimization where 0.1% AUC improvement = millions of dollars
Large Learning Rate (0.1-1.0)
  • Fast iteration during model development
  • Small datasets that cannot support many trees
  • Latency-sensitive inference (fewer trees in ensemble)
  • Example: rapid A/B test model prototyping, mobile/edge deployment
from sklearn.ensemble import GradientBoostingRegressor
import numpy as np

X = np.random.randn(200, 5)
y = X[:, 0]**2 + 0.5 * X[:, 1] + np.random.randn(200) * 0.3

# learning_rate controls step size; n_estimators = number of trees
gbr = GradientBoostingRegressor(
    n_estimators=100, learning_rate=0.1,
    max_depth=3, random_state=42
)
gbr.fit(X, y)
print(f'R2 score: {gbr.score(X, y):.3f}')
print(f'Trees used: {gbr.n_estimators_}')

Small learning_rate + many trees = better generalization than large lr + few trees.

8.5 Building the Full Ensemble

Watch the full boosting loop: each round fits a tree to residuals and adds it to the ensemble. The prediction converges to the target.

Key Insight: The ensemble prediction is the sum of all individual tree predictions, each scaled by the learning rate.
$$\hat{y}(x) = \hat{y}_0 + \sum_{t=1}^{T} \eta \cdot h_t(x)$$
Additive Model: Each tree adds a small correction. The sum of many weak learners creates a strong learner. Compare with dropout in neural networks (Ch 03), which creates an implicit ensemble of sub-networks -- a different route to the same idea.
Convergence: Train loss always decreases. Validation loss decreases then increases (overfitting). This gap motivates regularization and early stopping.

When to Use

Few Trees (10-50)
  • Simple patterns or very noisy data
  • High learning rate (0.3+) configurations
  • Latency-critical serving (each tree adds inference time)
  • Example: real-time bidding with 5ms latency budget
Moderate Trees (100-500)
  • Most tabular ML tasks with lr=0.05-0.1
  • Good balance of accuracy and inference speed
  • Standard for Kaggle-competitive models
  • Example: fraud detection, churn prediction, loan default
Many Trees (1000+)
  • Very low learning rate (0.01) for maximum accuracy
  • Large, complex datasets with subtle patterns
  • Always use early stopping to prevent waste
  • Example: insurance pricing with millions of policies and 200+ features
from xgboost import XGBRegressor
from sklearn.model_selection import train_test_split
import numpy as np

X = np.random.randn(1000, 10)
y = np.sum(X[:, :3]**2, axis=1) + np.random.randn(1000) * 0.5
X_tr, X_val, y_tr, y_val = train_test_split(X, y, test_size=0.2)

xgb = XGBRegressor(n_estimators=500, learning_rate=0.05,
                   max_depth=4, early_stopping_rounds=10)
xgb.fit(X_tr, y_tr, eval_set=[(X_val, y_val)], verbose=False)
print(f'Best round: {xgb.best_iteration}')
print(f'Val RMSE: {xgb.best_score:.4f}')

Early stopping halts training when validation loss stops improving.

8.6 Gradient Descent in Function Space

Gradient boosting IS gradient descent -- but in the space of functions, not parameters. Each tree is a step that reduces the loss.

The deep connection. In regular gradient descent, we update parameters: \(\theta \leftarrow \theta - \eta \nabla_\theta L\). In gradient boosting, we update the function itself: \(F \leftarrow F - \eta \nabla_F L\). Each tree approximates the negative gradient of the loss.
0
Loss reduction path
Current loss
Gradient direction
SGD (Parameter Space)
$$\theta_{t+1} = \theta_t - \eta \, \nabla_\theta \mathcal{L}$$
Update weights directly
Boosting (Function Space)
$$F_{t+1}(x) = F_t(x) - \eta \, h_t(x)$$
Add a tree \(h_t \approx \nabla_F \mathcal{L}\)
Key Insight: Boosting performs gradient descent in function space.
$$F_t(x) = F_{t-1}(x) - \eta \cdot \nabla_{F} \mathcal{L}(y, F_{t-1}(x))$$
Parameter Space: Neural networks do gradient descent on weights \(\theta\).
Function Space: Boosting does gradient descent on the prediction function \(F\) itself. Each tree approximates the negative gradient.
The name "gradient boosting" comes from this: we compute the gradient of the loss w.r.t. the current predictions, then fit a tree to approximate that gradient. The tree IS the gradient descent step.
Trees are to gradient boosting what parameter updates are to SGD. Both take steps that reduce the loss. For when transformers beat trees and vice versa, see Ch 06.

When to Use

GBDT (Function-Space GD)
  • Tabular/structured data where features have clear meaning
  • When you need built-in feature selection and missing value handling
  • No need to define architecture -- tree structure is learned
  • Works well with small-to-medium data (1k-10M rows)
  • Example: credit risk scoring, tabular Kaggle, recommendation ranking
Neural Nets (Parameter-Space GD)
  • Unstructured data (images, text, audio, graphs)
  • When transfer learning from pretrained models is valuable
  • Differentiable end-to-end pipelines needed
  • Very large datasets with GPU/TPU acceleration
  • Example: image classification, language models, speech-to-text
import xgboost as xgb
import numpy as np

# Huber loss: robust to outliers
def huber_obj(y_pred, dtrain):
    y_true = dtrain.get_label()
    delta = 1.0
    diff = y_pred - y_true
    grad = np.where(np.abs(diff) <= delta, diff, delta * np.sign(diff))
    hess = np.where(np.abs(diff) <= delta, 1.0, 0.0)
    return grad, hess

dtrain = xgb.DMatrix(X_tr, label=y_tr)
params = {'max_depth': 4, 'eta': 0.1}
model = xgb.train(params, dtrain, num_boost_round=100, obj=huber_obj)

Any twice-differentiable loss works: provide gradient and hessian.

8.7 Regularization & Shrinkage

Prevent overfitting with multiple techniques: learning rate shrinkage, depth limits, subsampling rows, subsampling features.

Key Insight: Regularization prevents the ensemble from memorizing the training data.
Shrinkage (Learning Rate)
Each tree contributes less. Slows learning but improves generalization. The most important regularizer.
Max Depth
Limits tree complexity. Shallow trees = weak learners = less overfitting per tree.
Row Subsampling
Each tree sees a random subset of rows (like bagging). Reduces variance and correlation between trees.
Column Subsampling
Each tree (or split) uses a random subset of features. Inspired by Random Forests. Reduces overfitting to specific features.
In practice, XGBoost/LightGBM defaults: learning_rate=0.1, max_depth=6, subsample=1.0, colsample=1.0. Start there and tune.

⚠️ Common Mistakes

  • Mistake: Forgetting to handle categorical features properly. Fix: Use one-hot encoding for XGBoost, but prefer native categorical support in LightGBM (categorical_feature) and CatBoost. One-hot encoding with high-cardinality categoricals creates sparse, inefficient features.
  • Mistake: Applying the same regularization strength everywhere. Fix: Tune regularization based on your data size: small datasets need more regularization (lower lr, shallower trees, more subsampling); large datasets can tolerate less.

When to Use

Shrinkage + Subsampling
  • Large datasets where overfitting is the primary risk
  • Row subsampling (0.7-0.9) acts like stochastic gradient descent
  • Column subsampling (0.5-0.8) decorrelates trees like Random Forest
  • Most impactful when combined: lr=0.05, subsample=0.8, colsample=0.7
  • Example: e-commerce conversion prediction with 50M rows and 300 features
Depth + Leaf Constraints
  • Small datasets (under 10k rows) where each split must be well-supported
  • max_depth=3-4 for boosting, plus min_child_weight=10-50
  • gamma (min split gain) prunes splits that barely help
  • Symmetric trees (CatBoost) for fastest inference
  • Example: clinical prediction with 2000 patients -- tight depth limits prevent memorization
import lightgbm as lgb
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=5000, n_features=20, random_state=42)

model = lgb.LGBMClassifier(
    n_estimators=300, learning_rate=0.05,
    max_depth=4, subsample=0.8,
    colsample_bytree=0.7, reg_alpha=0.1,
    reg_lambda=1.0, random_state=42
)
model.fit(X, y)
print(f'Accuracy: {model.score(X, y):.3f}')

Combine shrinkage, subsampling, and L1/L2 regularization for best results.

8.8 Full GBDT Training Loop

End-to-end: generate data, iteratively build trees, watch the decision boundary form, and see early stopping prevent overfitting.

Chapter 8 complete. You have built gradient boosted decision trees from scratch: stumps, recursive splitting, residuals, boosting steps, ensembles, the gradient descent connection, regularization, and the full training loop. This is how XGBoost, LightGBM, and CatBoost work under the hood.

And GBDT vs neural nets (8.6) is a key decision point that depends on whether your data is tabular or unstructured — see Ch 6: Transformers.

The Full GBDT Algorithm:
$$\text{Initialize } F_0(x) = \bar{y}. \text{ For } t = 1 \ldots T: \text{ fit } h_t \text{ to } r_i = -\nabla_F L, \text{ update } F_t = F_{t-1} + \eta \cdot h_t$$
1. Initialize: Start with a constant prediction (mean of targets for regression, log-odds for classification).
2. Compute Residuals: For each sample, compute the negative gradient of the loss w.r.t. current prediction.
3. Fit Tree: Build a regression tree on the residuals (pseudo-residuals for non-MSE losses).
4. Update: Add the new tree's predictions (scaled by learning rate) to the ensemble.
5. Early Stop: Monitor validation loss. Stop when it stops improving (patience rounds).
XGBoost, LightGBM, CatBoost all follow this same algorithm. The differences are in HOW they build trees (histogram binning, leaf-wise vs level-wise growth, handling categoricals). The core idea is identical. Feature importance in GBDT is analogous to attention weights in transformers (Ch 04) -- both answer "what did the model focus on?", but through very different mechanisms.
GBDT = "fit the errors, add a little bit, repeat". Learning rate shrinks each tree's contribution, early stopping says when to stop.
from xgboost import XGBClassifier
from sklearn.model_selection import train_test_split
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=5000, n_features=20, random_state=42)
X_tr, X_val, y_tr, y_val = train_test_split(X, y, test_size=0.2)

clf = XGBClassifier(
    n_estimators=500, learning_rate=0.05,
    max_depth=4, subsample=0.8,
    colsample_bytree=0.7, early_stopping_rounds=10
)
clf.fit(X_tr, y_tr, eval_set=[(X_val, y_val)], verbose=False)
print(f'Best iteration: {clf.best_iteration}')
print(f'Accuracy: {clf.score(X_val, y_val):.3f}')

This is the standard production GBDT recipe: tune lr, depth, subsampling + early stopping.

When to Use

GBDT for Production
  • Tabular data with mixed types (numeric, categorical, missing)
  • When model must train in minutes, not hours
  • Feature importance and SHAP for model governance
  • Example: fraud detection pipeline, real-time credit decisions, search ranking
GBDT for Competitions
  • Kaggle tabular tasks -- GBDT dominates leaderboards
  • Ensemble XGBoost + LightGBM + CatBoost for diversity
  • Aggressive hyperparameter search with Optuna/Hyperopt
  • Example: stacking 3 GBDT variants with a linear meta-learner for top leaderboard spots
When NOT to Use GBDT
  • Images, text, audio -- use neural networks instead
  • Online learning / streaming data (trees are batch learners)
  • When smooth decision boundaries are required
  • Example: image classification, real-time personalization with streaming updates

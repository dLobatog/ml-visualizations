1.1 Scalar Operations & the Computation Graph

Every neural network is a computation graph of simple operations. Type values, pick an expression, and watch the graph build node by node with real numbers flowing through.

Key Insight: Every expression decomposes into a directed acyclic graph (DAG) of primitive operations.
$$f(a,b,c) = a \cdot b + c \quad\Rightarrow\quad \text{multiply}(a,b) \rightarrow \text{add}(\_, c)$$
Computation Graph: Nodes are operations or values. Edges carry data forward. This structure enables automatic differentiation.
Why It Matters: PyTorch, JAX, and TensorFlow all build computation graphs under the hood. Understanding this is understanding how autograd works.
When asked "how does autograd work?", start here: every operation records its inputs and outputs in a DAG. Backprop walks this DAG in reverse.
Forward pass = evaluate the graph left-to-right. Backward pass = differentiate right-to-left. Same graph, two directions.
import torch

a = torch.tensor(2.0, requires_grad=True)
b = torch.tensor(3.0, requires_grad=True)
c = torch.tensor(1.0, requires_grad=True)

y = a * b + c  # PyTorch builds the DAG automatically
print(y)       # tensor(7., grad_fn=<AddBackward0>)
print(y.grad_fn.next_functions)  # shows the graph edges

Every operation on tensors with requires_grad=True is recorded in a dynamic computation graph.

1.2 The Chain Rule — Backpropagation by Hand

Gradients flow backward through the graph. At each node: local derivative times upstream gradient. Step through one node at a time and watch the chain rule in action.

Forward values
Backward gradients
Key Insight: The chain rule decomposes a complex derivative into a product of simple local derivatives.
$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot \frac{\partial y}{\partial x} \quad \text{(upstream grad} \times \text{local grad)}$$
Multiply node \(y = a \cdot b\): \(\frac{\partial y}{\partial a} = b\), \(\frac{\partial y}{\partial b} = a\). The local gradient is the "other" input.
Add node \(y = a + b\): \(\frac{\partial y}{\partial a} = 1\), \(\frac{\partial y}{\partial b} = 1\). Addition distributes the gradient equally.
The chain rule is backprop. There is no separate algorithm. Backpropagation is just the chain rule applied systematically from output to input through the computation graph.
At each node: (gradient flowing in) times (local derivative) = (gradient flowing out to each parent).
import torch

x = torch.tensor(2.0, requires_grad=True)
w = torch.tensor(3.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)

y = x * w + b
y.backward()  # chain rule applied automatically

print(w.grad)  # dy/dw = x = 2.0
print(x.grad)  # dy/dx = w = 3.0
print(b.grad)  # dy/db = 1.0

backward() walks the DAG in reverse, applying the chain rule at each node.

1.3 Autograd Engine — Automatic Differentiation

We don't need to compute gradients by hand. The graph structure encodes everything. Watch a tiny autograd engine (like micrograd) run topological sort and fill in gradients automatically.

Key Insight: Autograd = topological sort + reverse traversal + chain rule at each node.
$$\text{topo\_order} = \text{reverse}(\text{topological\_sort}(\text{graph}))$$
Topological Sort: Process nodes in dependency order. Each node's gradient depends only on already-computed downstream gradients.
Why Order Matters: Toggle "Break It" above to disable topo sort. Watch how processing nodes in the wrong order gives incorrect gradients.
When to Use
Reverse Mode (Backprop)
One backward pass computes ALL parameter gradients. Cost: O(1) backward passes regardless of number of parameters. Used in neural networks.
Forward Mode
One forward pass computes gradient w.r.t. ONE input. Cost: O(n) passes for n inputs. Used when outputs >> inputs (rare in ML).
Reverse-mode autodiff (backprop) is efficient because one backward pass gives gradients for ALL parameters. This is why we can train models with millions of parameters.
import torch

x = torch.randn(3, requires_grad=True)
y = (x ** 2).sum()  # scalar loss

y.backward()
print(x.grad)  # dy/dx = 2*x for each element

# Gradients accumulate! Always zero them:
x.grad.zero_()
z = (x ** 3).sum()
z.backward()
print(x.grad)  # now dz/dx = 3*x^2

Always call .zero_() before backward() in training loops to prevent gradient accumulation across steps.

1.4 A Single Neuron

Key Insight: A single neuron computes a linear decision boundary in its input space.
$$y = \sigma(w_1 x_1 + w_2 x_2 + b)$$
Decision Boundary: \(w_1 x_1 + w_2 x_2 + b = 0\) is a line. The weights define its orientation, the bias shifts it.
Activation Function: Introduces nonlinearity. Without it, stacking layers would still give a linear function.
Activation Functions
ReLU
\(\max(0, x)\). Simple, fast, avoids vanishing gradients. Can "die" (output always 0).
Sigmoid
\(\frac{1}{1+e^{-x}}\). Output in (0,1). Used for probabilities. Saturates, causing vanishing gradients.
Tanh
\(\frac{e^x - e^{-x}}{e^x + e^{-x}}\). Output in (-1,1). Zero-centered. Still saturates at extremes.
A single neuron can only learn linearly separable patterns (like AND, OR). It cannot learn XOR. That is why we need multiple neurons and layers.
Weights = direction of the boundary. Bias = offset from origin. Activation = the nonlinear "squish."
import torch
import torch.nn as nn

neuron = nn.Linear(2, 1)  # 2 inputs, 1 output (includes bias)
relu = nn.ReLU()

x = torch.tensor([[1.0, 0.5]])
out = relu(neuron(x))  # y = ReLU(w1*x1 + w2*x2 + b)

print(f"Weights: {neuron.weight.data}")
print(f"Bias: {neuron.bias.data}")
print(f"Output: {out.item():.4f}")

nn.Linear computes the weighted sum + bias. Wrap it with an activation for a complete neuron.

1.5 Stacking Neurons: The MLP Forward Pass

Multiple neurons in multiple layers can learn nonlinear decision boundaries. Watch data pulse through the network and see the boundary curve. Toggle "Break It" to remove activations and watch the boundary become linear.

Key Insight: An MLP with at least one hidden layer and nonlinear activations is a universal function approximator.
$$h = \text{ReLU}(W_1 x + b_1), \quad y = W_2 h + b_2$$
Without Activations: \(W_2(W_1 x + b_1) + b_2 = (W_2 W_1)x + (W_2 b_1 + b_2)\) — still linear! Depth without nonlinearity is useless.
Universal Approximation: With enough hidden neurons, an MLP can approximate any continuous function on a compact set (Cybenko 1989, Hornik 1991).
If asked "why do we need activation functions?", demonstrate: without them, any deep network collapses to a single linear transformation. Compose N linear maps = still one linear map.
import torch
import torch.nn as nn

mlp = nn.Sequential(
    nn.Linear(2, 16),
    nn.ReLU(),
    nn.Linear(16, 1),
)

x = torch.randn(1, 2)
y = mlp(x)  # forward pass through all layers
print(f"Output: {y.item():.4f}")
print(f"Parameters: {sum(p.numel() for p in mlp.parameters())}")

nn.Sequential chains layers. Without ReLU, this collapses to a single linear layer.

1.6 MLP Backward Pass & Gradient Flow

Watch gradients flow backward through the network. Toggle between sigmoid and ReLU to see vanishing gradients in action. Add layers to make the problem worse.

4
Key Insight: Sigmoid's maximum derivative is 0.25. After L layers, gradients shrink by at most \(0.25^L\).
$$\text{Sigmoid: } \sigma'(x) = \sigma(x)(1-\sigma(x)) \leq 0.25 \quad\Rightarrow\quad \text{grad} \leq 0.25^L$$
Vanishing Gradients: With sigmoid and 10 layers: \(0.25^{10} \approx 0.000001\). First layer barely learns.
ReLU Fixes This: ReLU derivative is 1 for positive inputs. Gradients pass through unchanged (when active).
Vanishing gradients was the main obstacle to training deep networks before ReLU (Glorot & Bengio 2010) and residual connections (He et al. 2015).
import torch
import torch.nn as nn

# Deep sigmoid network
layers = [nn.Linear(64, 64) for _ in range(10)]
model = nn.Sequential(*[m for l in layers for m in (l, nn.Sigmoid())])

x = torch.randn(1, 64)
loss = model(x).sum()
loss.backward()

for i, layer in enumerate(layers):
    grad_norm = layer.weight.grad.norm().item()
    print(f"Layer {i}: grad norm = {grad_norm:.6f}")

Gradient norms shrink exponentially with sigmoid. Replace with ReLU to fix.

Key Insight: The learning rate controls the step size along the negative gradient direction.
$$\theta_{t+1} = \theta_t - \eta \nabla_\theta \mathcal{L}(\theta_t)$$
Too Small: Slow convergence, may get stuck in local minima or saddle points. Training takes forever.
Too Large: Overshoots the minimum, oscillates, or diverges entirely. Loss goes up instead of down.
When asked about learning rate tuning: start with 1e-3 (Adam default). Use learning rate schedulers (cosine, warmup) in practice. LR is the single most important hyperparameter.
import torch

w = torch.tensor([5.0, -3.0], requires_grad=True)
lr = 0.1

for step in range(5):
    loss = (w[0] - 1)**2 + 10*(w[1] - 2)**2  # elongated valley
    loss.backward()
    with torch.no_grad():
        w -= lr * w.grad
        w.grad.zero_()
    print(f"Step {step}: w={w.data.tolist()}, loss={loss.item():.4f}")

This is the core update rule. In practice, use torch.optim instead of manual updates.

1.8 The Training Loop — Putting It All Together

Watch a neural network learn in real-time. Four panels show the network, decision boundary, loss curve, and training phase simultaneously. You built every piece of this system from scratch.

0.10
Chapter 1 complete. You have built a complete neural network training system from scratch: computation graphs, automatic differentiation, neurons, layers, backpropagation, and gradient descent. Everything that follows — transformers, GNNs, language models — is variations on this same loop: forward, loss, backward, update.
The Training Loop: Four steps, repeated thousands of times.
$$\text{Forward} \rightarrow \text{Loss} \rightarrow \text{Backward} \rightarrow \text{Update} \rightarrow \text{Repeat}$$
1. Forward Pass: Compute predictions by passing data through the network.
2. Compute Loss: Measure how wrong the predictions are (MSE, cross-entropy, etc.).
3. Backward Pass: Compute gradients of the loss w.r.t. all parameters via backpropagation.
4. Update Weights: Adjust parameters in the direction that reduces the loss: \(\theta \leftarrow \theta - \eta \nabla \mathcal{L}\).
Optimizers
SGD
Vanilla gradient descent. Simple but can oscillate in ravines and converge slowly.
SGD + Momentum
Adds velocity term: \(v_t = \beta v_{t-1} + \nabla\mathcal{L}\). Smooths out oscillations, accelerates in consistent directions.
Adam
Adaptive LR per parameter using first and second moment estimates. Default choice in practice. Robust to hyperparameters.
When asked "walk me through training a neural network": forward pass, compute loss, backward pass (backprop), weight update (optimizer step), repeat. That is the entire loop.
import torch
import torch.nn as nn

model = nn.Sequential(nn.Linear(2, 16), nn.ReLU(), nn.Linear(16, 1))
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
loss_fn = nn.MSELoss()

X = torch.randn(100, 2)
y = (X[:, 0] * X[:, 1]).unsqueeze(1)

for epoch in range(100):
    pred = model(X)           # 1. Forward
    loss = loss_fn(pred, y)   # 2. Loss
    loss.backward()           # 3. Backward
    optimizer.step()          # 4. Update
    optimizer.zero_grad()     # Reset grads

Four steps repeated: forward, loss, backward, update. This loop trains every neural network.

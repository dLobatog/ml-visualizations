4.1 The Problem: Why MLPs Struggle with Sequences

Fixed-size context windows miss long-range dependencies. An MLP with context=3 trying to predict "was" only sees "the door ___" -- it can't see the subject "The cat" 8 tokens back. We need selective attention.

3
Key Insight: MLPs process a fixed window of tokens. Long-range dependencies (like subject-verb agreement) are invisible unless the window is enormous.
$$\text{MLP input} = [x_{t-k}, \ldots, x_{t-1}] \quad \text{where } k = \text{context size}$$
The Scaling Problem: Parameters grow as \(O(k \cdot d)\). For context=1000, d=768: that is 768,000 parameters just for the input layer.
What We Need: A mechanism that can selectively look at any position in the sequence, regardless of distance. This is attention.
When asked "why attention over RNNs/MLPs?": MLPs have a fixed receptive field. RNNs process sequentially (slow, vanishing gradients). Attention looks at ALL positions in parallel with learned relevance weights.
import torch
import torch.nn as nn

# MLP can only see a fixed window of tokens
context_size = 5
d_model = 64
mlp_input = nn.Linear(context_size * d_model, 128)  # O(k*d) params!

# Attention sees ALL tokens regardless of distance
seq_len = 100
x = torch.randn(1, seq_len, d_model)
attn = nn.MultiheadAttention(d_model, num_heads=4, batch_first=True)
out, weights = attn(x, x, x)  # every token attends to all 100
print(f"Output shape: {out.shape}")  # (1, 100, 64)

Attention scales to any sequence length without adding parameters. MLPs cannot.

4.2 Attention as Soft Dictionary Lookup

Before the math: attention is just a "soft" version of a dictionary lookup. Instead of exact-match-or-error, every key gets a similarity score, and the output is a weighted blend of all values.

Key Insight: Attention = soft dictionary. The query doesn't need an exact match. Every key contributes proportionally to its similarity.
$$\text{output} = \sum_{i} \text{softmax}(\text{similarity}(q, k_i)) \cdot v_i$$
Hard Lookup: Exact match or KeyError. Binary: you either find it or you don't.
Soft Lookup: Continuous similarity scores. Every key contributes. The output is a blend of all values weighted by relevance.
This IS attention. Everything else (Q, K, V projections, scaling, multi-head) is just making the similarity scores learnable and stable.
import torch
import torch.nn.functional as F

# 4 key-value pairs
keys = torch.randn(4, 8)    # 4 keys, dim=8
values = torch.randn(4, 8)  # 4 values
query = torch.randn(1, 8)   # 1 query

# Soft dictionary lookup
scores = query @ keys.T             # similarity scores
weights = F.softmax(scores, dim=-1) # normalize to sum=1
output = weights @ values            # weighted blend

print(f"Weights: {weights.data}")    # how much each key matched
print(f"Output: {output.shape}")     # (1, 8) blended value

This is the core attention mechanism: score, normalize, blend. Everything else builds on this.

4.3 Self-Attention: Q, K, V Projections

Q, K, V are learned linear projections that make the soft lookup trainable. Each token's embedding is projected into three different spaces: what am I looking for (Q), what do I contain (K), and what do I offer (V).

Callback to 4.2: Remember the soft dictionary? Q is the query, K is the key, V is the value. We have just made all three learnable via weight matrices \(W_Q, W_K, W_V\).
Key Insight: Without learned projections, attention is just dot-product similarity of raw embeddings. The W matrices let the model learn what to look for vs what to advertise vs what to return.
$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$
Query (Q): "I'm looking for..." -- encodes what this token wants to find in other tokens.
Key (K): "I contain..." -- encodes what this token advertises to other tokens.
Value (V): "Here's my content..." -- the actual information that gets passed when attended to.
Q/K/V is the most asked attention question. Key answer: Q and K determine HOW MUCH to attend (the routing). V determines WHAT information flows. They are decoupled on purpose.
import torch
import torch.nn as nn

d_model, d_k = 64, 16
W_Q = nn.Linear(d_model, d_k, bias=False)
W_K = nn.Linear(d_model, d_k, bias=False)
W_V = nn.Linear(d_model, d_k, bias=False)

x = torch.randn(1, 5, d_model)  # 5 tokens
Q = W_Q(x)  # "what am I looking for?"
K = W_K(x)  # "what do I contain?"
V = W_V(x)  # "what do I offer?"

scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)
print(f"Q,K,V shapes: {Q.shape}")  # (1, 5, 16)

W_Q, W_K, W_V are learned projections. They decouple routing (Q,K) from content (V).

4.4 Scaling & Softmax: Why \(\sqrt{d_k}\) Matters

Without scaling, dot products grow large with embedding dimension, pushing softmax into saturation (nearly one-hot). Dividing by \(\sqrt{d_k}\) keeps gradients healthy.

1.0
Key Insight: When \(d_k\) is large, dot products grow as \(O(\sqrt{d_k})\) in expectation. Softmax of large values saturates, killing gradients.
$$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$
Without Scaling (scale=1)
Scores: [45.2, -38.7, 51.0, 12.3]. Softmax: [0.003, 0.000, 0.997, 0.000]. Nearly one-hot -- can only attend to ONE token. Gradient of softmax near 0.
With √d_k Scaling (scale=8)
Scores: [5.65, -4.84, 6.38, 1.54]. Softmax: [0.29, 0.01, 0.59, 0.05]. Smooth distribution -- model can blend information from multiple tokens.
Think of it like temperature in a softmax. Large dot products = very cold = spiky. Dividing by √d_k = warming up = smoother. The sweet spot keeps gradients flowing.
import torch
import torch.nn.functional as F

d_k = 64
Q = torch.randn(1, 5, d_k)
K = torch.randn(1, 5, d_k)
V = torch.randn(1, 5, d_k)

# Without scaling: dot products are O(sqrt(d_k)) in magnitude
raw_scores = Q @ K.transpose(-2, -1)
print(f"Unscaled std: {raw_scores.std():.2f}")  # ~8

# With scaling: unit variance
scaled_scores = raw_scores / (d_k ** 0.5)
print(f"Scaled std: {scaled_scores.std():.2f}")    # ~1

attn = F.softmax(scaled_scores, dim=-1) @ V

Dividing by sqrt(d_k) keeps softmax inputs near unit variance, preventing gradient saturation.

4.5 Multi-Head Attention: Parallel Perspectives

One head captures one pattern. Multiple heads learn to attend to different things -- syntax, coreference, punctuation. Their outputs are concatenated and projected.

1
Head 1: Local syntax
Head 2: Subject-verb
Head 3: Coreference
Head 4: Positional
Key Insight: Each head operates in a lower-dimensional subspace (\(d_k = d_{model} / h\)). This is cheaper than one big head AND captures diverse patterns.
$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W_O$$
Why Multiple Heads? A single head can only produce one attention pattern per position. Multiple heads let each position attend to different things simultaneously.
Concatenation + W_O: After all heads compute independently, their outputs are concatenated and projected through \(W_O\) to blend the perspectives.
Common follow-up: "Why not just use a bigger single head?" Because a single attention pattern per position is a bottleneck. Multi-head is like an ensemble -- each head specializes on a different linguistic relation.
import torch
import torch.nn as nn

d_model, n_heads = 64, 4
mha = nn.MultiheadAttention(d_model, n_heads, batch_first=True)

x = torch.randn(2, 10, d_model)  # batch=2, seq_len=10
# Self-attention: Q=K=V=x
output, attn_weights = mha(x, x, x)

print(f"Output: {output.shape}")         # (2, 10, 64)
print(f"Attn weights: {attn_weights.shape}")  # (2, 10, 10)
# Each head uses d_k = d_model/n_heads = 16

Each head operates in d_model/n_heads dimensions. Outputs are concatenated and projected by W_O.

4.6 Causal Masking: No Peeking at the Future

For autoregressive generation, position \(i\) can only attend to positions \(\leq i\). Without the mask, the model cheats by copying future tokens.

Key Insight: The causal mask prevents information leakage from future tokens. Without it, the model can trivially copy the next token, achieving ~0 training loss but failing at generation.
$$\text{mask}_{ij} = \begin{cases} 0 & \text{if } j \leq i \\ -\infty & \text{if } j > i \end{cases}$$
With Causal Mask
Each position only sees past + present. Must actually LEARN to predict the next token from context. Generalization works.
Without Mask (Broken)
Position 3 can see position 5 -- the answer it's trying to predict! Training loss drops to ~0 (cheating). Generation produces garbage because there's no future to peek at.
Causal masking is what makes GPT-style models autoregressive. BERT uses NO mask (bidirectional). T5 uses causal mask in the decoder only. Know which architecture uses which masking strategy.
import torch
import torch.nn.functional as F

seq_len, d_k = 5, 16
Q = torch.randn(1, seq_len, d_k)
K = torch.randn(1, seq_len, d_k)
V = torch.randn(1, seq_len, d_k)

scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)

# Causal mask: -inf above diagonal
mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))

attn = F.softmax(scores, dim=-1)  # future positions become 0
print(attn[0])  # lower-triangular attention pattern

torch.triu creates the upper-triangular mask. masked_fill with -inf zeros them after softmax.

4.7 The Complete Attention Block

Assemble everything: multi-head causal attention + LayerNorm + residual connections + FFN. Toggle each component to see what breaks.

Chapter 4 payoff: You have built the attention block -- the core building block of every transformer. In Ch 5, you will see that putting this same mechanism on graph edges gives you GATs. In Ch 6, you will stack these blocks into a full transformer.
Key Insight: The attention block is the fundamental unit of the transformer. Every component serves a critical purpose.
$$\text{output} = x + \text{FFN}(\text{LayerNorm}(x + \text{MHA}(\text{LayerNorm}(x))))$$
LayerNorm: Stabilizes training by normalizing activations. Without it: gradient magnitudes vary wildly across layers.
Residual Connection: Enables gradient flow in deep stacks. Without it: gradients vanish after ~6 layers. With it: hundreds of layers work.
FFN (Feed-Forward): Applies a nonlinear transformation per-position. Attention alone is linear in V -- the FFN adds expressive power.
If asked "what are the components of a transformer block?": LayerNorm, Multi-Head Attention, Residual Add, LayerNorm, FFN, Residual Add. Pre-norm (LN before attention) is now standard over post-norm.
The block formula: (1) normalize, (2) attend, (3) add residual, (4) normalize, (5) feed-forward, (6) add residual. Six steps, one block. Stack N of these = a transformer.
import torch
import torch.nn as nn

class TransformerBlock(nn.Module):
    def __init__(self, d_model=64, n_heads=4):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, 4*d_model), nn.GELU(),
            nn.Linear(4*d_model, d_model))
    def forward(self, x):
        x = x + self.attn(self.ln1(x), self.ln1(x), self.ln1(x))[0]
        x = x + self.ffn(self.ln2(x))
        return x

Pre-norm transformer block: LayerNorm before attention and FFN. Residuals around both.

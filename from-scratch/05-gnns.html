5.1 Graphs as Data: Nodes, Edges, Features

Click on the canvas to add nodes. Drag from one node to another to create edges. Each node has a feature vector shown as a colored bar. The adjacency matrix updates live.

Click to add node
Drag node-to-node for edge

Key Insight: Unlike images (grid) or text (sequence), graph data has irregular structure. Each node can have a different number of neighbors.
$$A_{ij} = \begin{cases} 1 & \text{if edge between node } i \text{ and } j \\ 0 & \text{otherwise} \end{cases}$$
Adjacency Matrix A: A symmetric matrix where entry (i,j) = 1 if there is an edge between nodes i and j. This is the fundamental representation of graph structure.
Node Features X: Each node carries a feature vector. For molecules: atom type, charge. For social networks: user profile. The feature matrix X has shape (N, F).
Graphs are everywhere: social networks, molecules, knowledge bases, code ASTs, traffic networks. The key challenge is that we need architectures that handle variable-size neighborhoods.
A graph = (Nodes with features, Edges defining structure). The adjacency matrix A encodes who is connected to whom.
import torch
from torch_geometric.data import Data

# Define a 4-node graph: edges + node features
edge_index = torch.tensor([[0,1,1,2,2,3],[1,0,2,1,3,2]], dtype=torch.long)
x = torch.randn(4, 16)  # 4 nodes, 16-dim features
data = Data(x=x, edge_index=edge_index)
print(data)  # Data(x=[4,16], edge_index=[2,6])

PyG uses COO format: edge_index has shape [2, num_edges].

5.2 Message Passing: The Core Idea

Click any node to be the "focus." Watch neighbors send colored envelopes along edges, then aggregate and update. Use the layer slider to see the receptive field grow.

1
Target node
Neighbors (sending)
Envelopes (messages)
Updated embedding
Key Insight: One round of message passing = one GNN layer. Each node "collects mail" from its neighbors, aggregates, and updates.
$$h_v^{(l+1)} = \sigma\!\left(W^{(l)} \cdot \text{AGG}\!\left(\{h_u^{(l)} : u \in \mathcal{N}(v)\}\right)\right)$$
1-hop (1 layer): Each node knows about its direct neighbors.
k-hop (k layers): Each node knows about nodes up to k edges away. The receptive field grows with depth.
"Message passing" is the unifying framework for most GNNs: GCN, GAT, GraphSAGE, GIN all differ only in how they create messages and aggregate them.
Message passing in 4 steps: (1) Highlight target, (2) Neighbors send messages, (3) Aggregate (mean/sum/max), (4) Update with W and activation.
import torch
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import add_self_loops

class MyGNN(MessagePassing):
    def __init__(self, in_ch, out_ch):
        super().__init__(aggr='mean')  # mean aggregation
        self.lin = torch.nn.Linear(in_ch, out_ch)
    def forward(self, x, edge_index):
        return self.propagate(edge_index, x=self.lin(x))
    def message(self, x_j):  # x_j = neighbor features
        return x_j

MessagePassing handles gather/scatter; override message() and aggregate().

5.3 Aggregation Functions: Mean vs Sum vs Max

Same graph, three columns. Watch how different aggregations produce different embeddings. The key experiment: degree-2 node vs degree-10 node with identical neighbor features.

Mean aggregation
Sum aggregation
Max aggregation
Key Insight: Different aggregations have different properties. Sum preserves degree info. Mean is degree-invariant. Max captures the most salient feature.
Sum Aggregation
Degree-aware: a node with 10 neighbors gets 5x larger embedding than a node with 2 neighbors (same features). Can distinguish multi-sets.
Mean Aggregation
Degree-invariant: both get the same embedding regardless of neighbor count. Loses structural information about degree.
Max Aggregation: Captures the most extreme feature value among neighbors. Good for detecting the presence of a feature, not its frequency.
GIN (Graph Isomorphism Network) proves that sum aggregation is the most expressive -- it can distinguish graph structures that mean and max cannot.
from torch_geometric.nn import GCNConv, SAGEConv, GINConv
import torch.nn as nn

# GCN uses mean (symmetric-normalized)
gcn = GCNConv(16, 32)
# GraphSAGE supports mean, max, lstm
sage = SAGEConv(16, 32, aggr='max')
# GIN uses sum (most expressive)
gin_nn = nn.Sequential(nn.Linear(16, 32), nn.ReLU())
gin = GINConv(gin_nn)  # sum aggregation by default

Pass aggr='mean'|'sum'|'max' to most PyG layers.

5.4 GCN: Graph Convolutional Network

The GCN formula step by step: add self-loops, normalize by degree, multiply features, weight matrix, activation. Compare GCN vs naive message passing.

Key Insight: GCN uses symmetric normalization to prevent degree-dependent scaling.
$$H^{(l+1)} = \sigma\!\left(\hat{D}^{-1/2}\hat{A}\hat{D}^{-1/2} H^{(l)} W^{(l)}\right) \quad\text{where}\quad \hat{A} = A + I$$
Self-loops (A + I): Each node also receives its OWN message, not just from neighbors. This preserves the node's own features across layers.
Symmetric norm: \(\hat{D}^{-1/2}\hat{A}\hat{D}^{-1/2}\) normalizes by both sender and receiver degree. This is a weighted mean where weight depends on both nodes' connectivity.
GCN is a first-order approximation of spectral graph convolutions. The normalized adjacency acts as a low-pass filter on the graph signal -- it smooths features between connected nodes.
GCN in one line: "Average your neighbors (including yourself), normalize by degree, then transform with a learned weight matrix."
import torch.nn.functional as F
from torch_geometric.nn import GCNConv

class GCN(torch.nn.Module):
    def __init__(self, in_dim, hid, out_dim):
        super().__init__()
        self.conv1 = GCNConv(in_dim, hid)
        self.conv2 = GCNConv(hid, out_dim)
    def forward(self, data):
        x = F.relu(self.conv1(data.x, data.edge_index))
        x = F.dropout(x, p=0.5, training=self.training)
        return self.conv2(x, data.edge_index)

GCNConv adds self-loops and applies symmetric normalization automatically.

5.5 GAT: Graph Attention Network

Not all neighbors are equally important. Edge thickness = learned attention weight. Compare GCN (degree-weighted) vs GAT (attention-weighted) side by side.

1
GCN weights (degree-based)
GAT weights (learned attention)
Callback to Ch 4: Remember Q/K/V attention on sequences? GAT is the same idea on graph edges. Instead of attending to all positions in a sequence, each node attends to its neighbors.
Key Insight: GAT learns which neighbors matter most. The attention mechanism assigns different weights to different neighbors.
$$\alpha_{ij} = \frac{\exp\!\left(\text{LeakyReLU}\!\left(\mathbf{a}^T [Wh_i \| Wh_j]\right)\right)}{\sum_{k \in \mathcal{N}(i)} \exp\!\left(\text{LeakyReLU}\!\left(\mathbf{a}^T [Wh_i \| Wh_k]\right)\right)}$$
Transform: Apply shared weight matrix W to both source and target node features.
Concatenate + Score: Concatenate transformed features [Wh_i || Wh_j], apply attention vector a, then LeakyReLU.
Normalize: Softmax over all neighbors to get attention coefficients that sum to 1.
Multi-head attention in GATs: run K independent attention heads and concatenate (or average) their outputs. This stabilizes learning and captures different types of neighbor importance.
from torch_geometric.nn import GATConv

# 8 attention heads, concat output
gat1 = GATConv(16, 8, heads=8, concat=True)   # out: 64
# Final layer: average heads instead of concat
gat2 = GATConv(64, 7, heads=1, concat=False)  # out: 7 classes

# Forward: out = gat2(F.elu(gat1(x, edge_index)), edge_index)

Set concat=False on the final layer to average attention heads.

5.6 Over-Smoothing: The Depth Problem in GNNs

Too many GNN layers make all node embeddings converge to the same vector. Watch the heatmap go uniform. Toggle fixes: residual connections, skip connections, jumping knowledge.

1
Diverse embeddings (good)
Converged embeddings (bad)
Fix applied
Key Insight: Over-smoothing is why most GNNs use only 2-3 layers, unlike deep CNNs with 100+ layers.
$$\text{Diversity}(L) = \frac{1}{N^2}\sum_{i \neq j} \|h_i^{(L)} - h_j^{(L)}\|_2 \quad \xrightarrow{L \to \infty} \quad 0$$
Residual Connections
h' = GNN(h) + h. Skip connection preserves original information. Inspired by ResNets.
Jumping Knowledge
Concatenate outputs from ALL layers: [h^(1) || h^(2) || ... || h^(L)]. The model selects the best layer per node.
If asked "why are GNNs shallow?", explain over-smoothing: repeated message passing is like repeatedly averaging -- everything converges. This is fundamentally different from depth problems in CNNs.
from torch_geometric.nn import GCNConv
import torch.nn.functional as F

class ResGCN(torch.nn.Module):
    def __init__(self, dim, num_layers=8):
        super().__init__()
        self.convs = torch.nn.ModuleList(
            [GCNConv(dim, dim) for _ in range(num_layers)])
    def forward(self, x, edge_index):
        for conv in self.convs:
            x = x + F.relu(conv(x, edge_index))  # residual
        return x

Residual connections let deep GNNs (8+ layers) avoid over-smoothing.

5.7 Graph-Level Tasks: Pooling & Readout

Go from node embeddings to a single graph vector. Compare mean, sum, and hierarchical pooling. See the full molecule classification pipeline.

Key Insight: For graph-level tasks (graph classification, property prediction), we need a readout function that compresses all node embeddings into one vector.
$$h_G = \text{READOUT}\!\left(\{h_v^{(L)} : v \in G\}\right) \quad\text{e.g.,}\quad h_G = \frac{1}{|V|}\sum_{v \in V} h_v^{(L)}$$
Mean Pooling: Average all node embeddings. Size-invariant but loses information about graph size.
Sum Pooling: Sum all node embeddings. Preserves graph size information. Provably more expressive (GIN paper).
Hierarchical Pooling: Progressively coarsen the graph (e.g., DiffPool). Learns which nodes to group. More powerful but more complex.
The molecule classification pipeline: molecular graph -> GNN layers -> readout -> MLP classifier -> property prediction. This is the workhorse of computational chemistry and drug discovery.
from torch_geometric.nn import GCNConv, global_mean_pool
import torch.nn.functional as F

class GraphClassifier(torch.nn.Module):
    def __init__(self, in_dim, hid, n_classes):
        super().__init__()
        self.conv1 = GCNConv(in_dim, hid)
        self.conv2 = GCNConv(hid, hid)
        self.lin = torch.nn.Linear(hid, n_classes)
    def forward(self, data):
        x = F.relu(self.conv1(data.x, data.edge_index))
        x = F.relu(self.conv2(x, data.edge_index))
        x = global_mean_pool(x, data.batch)  # graph-level
        return self.lin(x)

global_mean_pool averages node embeddings per graph in a batch.

5.8 GraphSAGE: Inductive Learning & Sampling

Scale GNNs to huge graphs by sampling neighbors. Adjust the sampling budget K and watch the computation graph change. Compare inductive vs transductive.

5
Target node
Sampled neighbors (layer 1)
Sampled neighbors (layer 2)
New node (inductive)
Chapter 5 Payoff: You've built GNNs from the ground up: message passing, GCN, GAT, and learned to handle their failure modes. The same attention mechanism from Ch 4 powers GATs -- and the same principle of learning from structure will continue into later chapters.
Key Insight: GraphSAGE samples a fixed number of neighbors at each layer, enabling mini-batch training on massive graphs.
$$h_v^{(l)} = \sigma\!\left(W \cdot \text{AGG}\!\left(\{h_u^{(l-1)} : u \in \mathcal{S}_K(\mathcal{N}(v))\}\right)\right)$$
Transductive (GCN)
Requires the full graph at training time. Cannot generalize to unseen nodes without retraining.
Inductive (GraphSAGE)
Learns a function that samples and aggregates. Can immediately compute embeddings for new, unseen nodes.
K=2 (small budget): Fast but noisy. High variance across samples. Good for very large graphs where speed matters.
K=25 (large budget): Accurate but slow. Low variance. Approaches full-neighborhood aggregation (GCN-like).
GraphSAGE's key innovation: instead of learning embeddings (transductive), it learns an aggregation function (inductive). This is why it scales to graphs with millions of nodes and can handle new nodes at test time.
from torch_geometric.nn import SAGEConv
from torch_geometric.loader import NeighborLoader

# Neighbor sampling: 25 neighbors at layer 1, 10 at layer 2
loader = NeighborLoader(data, num_neighbors=[25, 10],
                        batch_size=512, shuffle=True)

class SAGE(torch.nn.Module):
    def __init__(self, in_dim, hid, out_dim):
        super().__init__()
        self.conv1 = SAGEConv(in_dim, hid)
        self.conv2 = SAGEConv(hid, out_dim)
    def forward(self, x, edge_index):
        x = F.relu(self.conv1(x, edge_index))
        return self.conv2(x, edge_index)

NeighborLoader enables mini-batch training on million-node graphs.

Core Concept: Markov Decision Processes (MDPs) formalize sequential decision-making. An MDP has states S, actions A, transition dynamics P(s'|s,a), rewards R(s,a,s'), and discount γ ∈ [0,1].

V(s) = E[Σ γ^t R_t | starting from s under optimal policy]
V(s):State value function; expected cumulative discounted reward from state s under optimal policy
E[·]:Expectation over randomness in transitions and rewards
Σ γ^t R_t:Sum of discounted rewards; return G_t = R_t + γR_{t+1} + γ²R_{t+2} + ...
γ:Discount factor ∈ [0,1); how much to value future rewards relative to immediate. Typical: 0.99
R_t:Reward at time step t; scalar signal of immediate performance
π*:Optimal policy; maximizes V(s); can be deterministic or stochastic
Markov Property:Future depends only on current state, not history (memoryless)
Policy π(a|s):Maps states to action distributions; π* maximizes expected return

Key Mathematical Intuition:

V(s) = E[R(s,a,s') + γV(s') | a~π(s)]
V(s):Current state value
R(s,a,s'):Expected reward for action a taken in state s, transitioning to s'
γ:Discount factor ∈ [0,1); weights future value. Typical: 0.99
V(s'):Value of successor state s'; enables dynamic programming recursion
a~π(s):Action sampled from policy π(a|s); distributes expectation over action probabilities

Bellman equation: state value equals immediate reward plus discounted successor value.

Value V(s)
Expected return from state; enables value iteration
Q-function Q(s,a)
Expected return from state+action; Q = E[R + γV(s')]
Low γ (→0)
Myopic: prioritizes immediate rewards
High γ (→1)
Farsighted: balances long-term rewards
Why does Markov assumption matter? It enables dynamic programming solutions; without it, you'd need full history, making problems intractable.
Think: Bellman = backwards induction. Each state's value = what you get now + discounted value of successor states.

Interview Gotchas: Markov property is strong (not all problems satisfy it); Value iteration rate depends on γ and space size; Large spaces need function approximation; Exploration-exploitation is orthogonal to MDPs.

Production Tips: Verify Markov assumption holds. Use γ=0.99 for long horizons. Function approximation essential for continuous spaces. Monitor episode returns. Always simulate offline before deployment.

Core Concept: Q-learning is a model-free, off-policy temporal-difference algorithm for learning optimal action-values Q(s,a). Learns from experience without knowing P or R.

Q(s,a) ← Q(s,a) + α[R + γ max Q(s',a') - Q(s,a)]
Q(s,a):Action-value function; expected return from taking action a in state s, then following optimal policy
α:Learning rate ∈ (0,1]; step size for each update. Typical: 0.01-0.1. Lower = more stable, slower convergence
R:Immediate reward received after taking action a in state s
γ:Discount factor ∈ [0,1); devalues future rewards. Typical: 0.99
max Q(s',a'):Maximum Q-value over all possible actions a' in next state s' (greedy lookahead)
δ (TD Error):R + γ max Q(s',a') - Q(s,a); measures prediction error, drives updates toward better estimates
Off-Policy:Uses greedy max for update; behavior policy can differ from target policy

Key Insight vs. SARSA:

Q-Learning (Off-Policy)
Uses max action in target; optimistic; enables off-policy updates
SARSA (On-Policy)
Uses actual action taken; pessimistic; slower but safer
Q-learning converges in tabular case if every s,a pair visited infinitely often with decaying learning rate. In practice, function approximation breaks these guarantees.
Q-Learning: Maximize future Q values (greedy bootstrap). The max makes it optimistic and off-policy—you explore but learn about the best policy.

Key Gotchas: Overestimation bias from max (max_a Q is biased up); Function approximation breaks convergence; Off-policy divergence if policies drift; ε-greedy is inefficient.

Production Solutions: Double DQN (separate target/online networks for bias reduction); Experience replay (decorrelates samples); Slow target network updates (stable targets); Policy gradients for continuous control; Monitor Q-value distributions for explosions; Always simulate before deployment.

Core Concept: Policy gradient methods directly optimize policy π_θ(a|s) instead of learning value functions. Enables stochastic policies and continuous control.

∇J(θ) = E[∇log π_θ(a|s) · Q(s,a)]
∇J(θ):Gradient of objective function J with respect to policy parameters θ
π_θ(a|s):Parameterized policy; probability of action a given state s, with parameters θ
∇log π_θ(a|s):Score function; gradient of log probability. Gradient flows through stochastic policy
Q(s,a):Action-value estimate; controls variance of gradient. Can use returns or learned Q
E[·]:Expectation over states and actions during rollout

Policy Gradient Theorem: gradient increases probability of high-value actions

REINFORCE:Uses full return G_t = Σγ^k R_{t+k} as Q estimate; high variance
Log-Trick:∇π = π ∇log π allows backprop through stochastic policy
Advantage:A(s,a) = Q(s,a) - V(s) reduces variance without bias (baseline cancels)

Algorithm Variants:

REINFORCE
Monte-Carlo returns; simple but high variance
Actor-Critic
Critic estimates V(s); lower variance via advantage
PPO
Clipped objective; stable large updates
TRPO
KL constraint on updates; theoretically principled
Log-probability trick is key: you can't backprop through stochasticity, but you can backprop through log(π)—differentiate the log, then multiply by the result.
Actor-Critic = actor (policy) makes decisions, critic (value) evaluates them. Critic reduces variance of policy gradients via advantage estimation.

Key Gotchas: REINFORCE high variance without baselines; Critic bias if poorly trained; High sample complexity; Local optima common; Off-policy corrections can explode.

Production Best Practices: Use PPO/TRPO for stability (clip/trust region updates); Actor-critic more sample-efficient; GAE balances bias-variance; Monitor policy entropy; Test exhaustively (can exploit simulator); Continuous control is PG sweet spot.

Core Concept: Multi-armed bandits isolate exploration-exploitation tradeoff. K arms with unknown reward distributions; no states, no sequential dependence.

Regret_T = E[Σ R*(a) - Σ R(a_t)] over T rounds
Regret_T:Cumulative performance gap; sum of optimal rewards minus actual rewards. Lower = better exploration-exploitation balance
R*(a):Reward of optimal arm in round; R* = max_a μ_a (best expected payoff)
R(a_t):Actual reward from chosen arm a_t at round t
T:Total number of rounds; longer horizon → larger regret. Regret rate (regret/T) → 0 = good algorithm
E[·]:Expectation over arm reward distributions and algorithm randomness
ε-Greedy:Exploit best arm (1-ε); explore uniform (ε). Regret: O(εT + (1-ε)K log T)
UCB:Q̂(a) + c√(ln T / n(a)) = empirical + uncertainty bonus. Regret: O(log T)
Thompson Sampling:Bayesian posterior; sample rewards, play best. Regret: O(log T)

Regret Analysis:

ε-Greedy
O(εT): linear term is wasted; suboptimal
UCB
O(log T): exploration bonus shrinks; theoretically optimal
Thompson Sampling
O(log T): posterior concentrates; empirically fastest
Lower bound on regret is Ω(log T). Only UCB and Thompson Sampling achieve this; ε-greedy is fundamentally suboptimal (linear waste term).
UCB = empirical mean + exploration bonus that shrinks as arm is played. Thompson = posterior samples naturally balance exploration (high uncertainty) and exploitation (posterior peaks).

Key Gotchas: ε-greedy regret is linear in T (bad for long horizons); UCB constant c requires tuning; Thompson Sampling needs Bayesian inference; Contextual bandits are realistic but harder.

Production Best Practices: Use Thompson Sampling for A/B testing/ad serving (auto-balances, easy to explain); UCB for explicit control; Avoid ε-greedy unless trivial; Contextual variants (LinUCB, ContextTS) for recommendations; Monitor arm frequencies; Cold-start: all arms equally unknown.

Core Concept: RLHF aligns LLMs with human preferences via reinforcement learning. No explicit reward labels; instead, learn from preference comparisons.

Loss = -E[min(r_t·Â, clip(r_t, 1-ε, 1+ε)·Â)] - β·KL(π_new || π_ref)
Loss:Objective to minimize (negative because we maximize reward and penalize divergence)
r_t:Probability ratio = π_new(a_t|s_t) / π_ref(a_t|s_t); >1 means new policy more likely than reference
Â:Advantage estimate; reward_model(s_t,a_t) - baseline. Positive = good actions, negative = bad
clip(r_t, 1-ε, 1+ε):Clips ratio to prevent too-large policy changes. Typical ε = 0.2; prevents instability
β:KL penalty weight ∈ [0.05, 1]; higher β = stricter adherence to reference. Typical: 0.1-1.0
KL(π_new || π_ref):Kullback-Leibler divergence; measures distributional distance. Prevents mode collapse to reward gaming
π_ref:Reference policy (usually SFT model); keeps new policy from drifting too far

PPO with KL penalty: maximize reward while staying close to reference model

Pipeline Step 1:SFT on human examples to bootstrap quality
Pipeline Step 2:Collect human preferences (A vs B comparisons)
Pipeline Step 3:Train reward model via Bradley-Terry logistic regression
Pipeline Step 4:RL fine-tune LLM on reward signal with KL regularization

Bradley-Terry Model:

P(y ≻ x) = σ(R(x,y))
P(y ≻ x):Probability that response y is preferred over response x; ∈ [0,1]
σ(·):Sigmoid activation σ(z) = 1/(1+e^{-z}); maps real numbers to [0,1]
R(x,y):Scalar reward difference from learned reward model; R(x,y) = r_θ(y) - r_θ(x)
r_θ:Learned reward model (neural network); outputs scalar reward for text pair

Preference probability = sigmoid of reward difference; trainable via cross-entropy

KL penalty is crucial: without it, LLM mode-collapses to trivial outputs that maximally exploit the reward model. KL keeps it close to SFT baseline.
RLHF = teach model to please humans. Bradley-Terry = learn what humans like. PPO+KL = apply RL while staying sensible. Three-stage dance between SFT, reward, and policy.

Key Challenges: Reward hacking (exploiting model weaknesses); Expensive annotation with labeler bias; Reward model overfits or goes off-distribution; KL tuning critical; Computational cost high.

Production Best Practices: Multiple annotators, measure agreement; Diverse preference data; Monitor RM accuracy on held-out; Use β ≈ 0.1-1.0; Test on multiple benchmarks; Iteratively improve RM as LLM improves; Static RM becomes suboptimal over time.

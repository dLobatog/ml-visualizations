1. Markov Decision Process

An MDP is the basic language of reinforcement learning: state, action, reward, transition, and a discount factor that decides how much the future matters.

Core Concept: Markov Decision Processes (MDPs) formalize sequential decision-making. An MDP has states S, actions A, transition dynamics P(s'|s,a), rewards R(s,a,s'), and discount \(\gamma \in [0,1]\).

$$V(s) = \mathbb{E}\left[\sum \gamma^t R_t \;\middle|\; \text{starting from } s \text{ under optimal policy}\right]$$
\(V(s)\):State value function; expected cumulative discounted reward from state \(s\) under optimal policy
\(\mathbb{E}[\cdot]\):Expectation over randomness in transitions and rewards
\(\sum \gamma^t R_t\):Sum of discounted rewards; return \(G_t = R_t + \gamma R_{t+1} + \gamma^2 R_{t+2} + \ldots\)
\(\gamma\):Discount factor \(\in [0,1)\); how much to value future rewards relative to immediate. Typical: 0.99
R_t:Reward at time step t; scalar signal of immediate performance
\(\pi^*\):Optimal policy; maximizes \(V(s)\); can be deterministic or stochastic
Markov Property:Future depends only on current state, not history (memoryless)
Policy \(\pi(a|s)\):Maps states to action distributions; \(\pi^*\) maximizes expected return

Key Mathematical Intuition:

$$V(s) = \mathbb{E}[R(s,a,s') + \gamma V(s') \;|\; a \sim \pi(s)]$$
\(V(s)\):Current state value
\(R(s,a,s')\):Expected reward for action a taken in state s, transitioning to s'
\(\gamma\):Discount factor \(\in [0,1)\); weights future value. Typical: 0.99
\(V(s')\):Value of successor state \(s'\); enables dynamic programming recursion
\(a \sim \pi(s)\):Action sampled from policy \(\pi(a|s)\); distributes expectation over action probabilities

Bellman equation: state value equals immediate reward plus discounted successor value.

Value \(V(s)\)
Expected return from state; enables value iteration
\(Q\)-function \(Q(s,a)\)
Expected return from state+action; \(Q = \mathbb{E}[R + \gamma V(s')]\)
Low \(\gamma\) (\(\to 0\))
Myopic: prioritizes immediate rewards
High \(\gamma\) (\(\to 1\))
Farsighted: balances long-term rewards
When to Use
Value Iteration
When state space is small and model (transitions, rewards) is known. E.g., grid worlds, simple game AI, routing problems. Guaranteed optimal policy via dynamic programming. Scales poorly beyond ~10K states.
Policy Iteration
When convergence speed matters and policy evaluation is cheap. E.g., inventory management, resource allocation. Converges in fewer iterations than value iteration but each iteration costs more (full policy eval).
Model-Free Methods
When transition dynamics are unknown or state space is too large for tabular methods. E.g., robotics, game playing, real-world control. Use Q-learning, policy gradients, or actor-critic instead.
Why does Markov assumption matter? It enables dynamic programming solutions; without it, you'd need full history, making problems intractable.
👉 Try this: In the MDP visualization, verify Markov property: P(s'|s, a) depends only on s and a, not history. This independence enables Bellman equations, value iteration, and policy gradients. Without Markov: infinite history needed (intractable).
Think: Bellman = backwards induction. Each state's value = what you get now + discounted value of successor states.

Interview Gotchas: Markov property is strong (not all problems satisfy it); Value iteration rate depends on γ and space size; Large spaces need function approximation; Exploration-exploitation is orthogonal to MDPs.

Production Tips: Verify Markov assumption holds. Use γ=0.99 for long horizons. Function approximation essential for continuous spaces. Monitor episode returns. Always simulate offline before deployment.

Q-learning updates values from partial experience instead of waiting for full rollouts. That is the key efficiency win of temporal-difference learning.

$$Q(s,a) \leftarrow Q(s,a) + \alpha[r + \gamma \cdot \max_{a'} Q(s',a') - Q(s,a)]$$

Core Concept: Q-learning is a model-free, off-policy temporal-difference algorithm for learning optimal action-values Q(s,a). Learns from experience without knowing P or R.

$$Q(s,a) \leftarrow Q(s,a) + \alpha[R + \gamma \max_{a'} Q(s',a') - Q(s,a)]$$
\(Q(s,a)\):Action-value function; expected return from taking action \(a\) in state \(s\), then following optimal policy
\(\alpha\):Learning rate \(\in (0,1]\); step size for each update. Typical: 0.01-0.1. Lower = more stable, slower convergence
\(R\):Immediate reward received after taking action a in state s
\(\gamma\):Discount factor \(\in [0,1)\); devalues future rewards. Typical: 0.99
\(\max_{a'} Q(s',a')\):Maximum Q-value over all possible actions a' in next state s' (greedy lookahead)
\(\delta\) (TD Error):\(R + \gamma \max_{a'} Q(s',a') - Q(s,a)\); measures prediction error, drives updates toward better estimates
Off-Policy:Uses greedy max for update; behavior policy can differ from target policy

Key Insight vs. SARSA:

Q-Learning (Off-Policy)
Uses max action in target; optimistic; enables off-policy updates
SARSA (On-Policy)
Uses actual action taken; pessimistic; slower but safer
When to Use
Q-Learning / DQN
When action space is discrete and moderate-sized. E.g., Atari games, board games, recommendation systems with finite actions. Off-policy enables experience replay for sample efficiency. DQN scales to visual inputs.
SARSA (On-Policy)
When safety during learning matters (cliff-walking scenarios). E.g., robot navigation near hazards, trading systems where exploration cost is high. More conservative than Q-learning -- avoids risky actions during training.
Policy Gradient Instead
When action space is continuous or very large. E.g., robotic arm control (joint angles), autonomous driving (steering), dialogue systems. Q-learning fails with continuous actions (cannot compute max_a Q).
Q-learning converges in tabular case if every s,a pair visited infinitely often with decaying learning rate. In practice, function approximation breaks these guarantees.
👉 Try this: Run Q-Learning with α=0.1 (stable) and α=0.9 (noisy). Convergence guarantee requires tabular (small state space), exploration (ε-greedy), and learning rate decay. With neural networks (Deep Q-Network), only approximation—divergence possible. Always monitor for instability.
Q-Learning: Maximize future Q values (greedy bootstrap). The max makes it optimistic and off-policy—you explore but learn about the best policy.

Key Gotchas: Overestimation bias from max (max_a Q is biased up); Function approximation breaks convergence; Off-policy divergence if policies drift; ε-greedy is inefficient.

Production Solutions: Double Deep Q-Network (separate target/online networks for bias reduction); Experience replay (decorrelates samples); Slow target network updates (stable targets); Policy gradients for continuous control; Monitor Q-value distributions for explosions; Always simulate before deployment.

import gymnasium as gym
                import numpy as np
                
                env = gym.make("FrozenLake-v1", is_slippery=False)
                Q = np.zeros((env.observation_space.n, env.action_space.n))
                alpha, gamma, epsilon = 0.1, 0.99, 0.1
                
                for ep in range(2000):
                    s, _ = env.reset()
                    done = False
                    while not done:
                        a = env.action_space.sample() if np.random.random() < epsilon else np.argmax(Q[s])
                        s2, r, term, trunc, _ = env.step(a)
                        Q[s, a] += alpha * (r + gamma * np.max(Q[s2]) - Q[s, a])
                        s, done = s2, term or trunc
                
                print("Learned Q-table:\n", Q)

Tabular Q-learning converges for small discrete state/action spaces.

Adjust learning rate to see gradient step magnitude. High rates overshoot, low rates creep slowly. Watch the policy optimize toward high-reward regions. REINFORCE is simple: sample trajectories, compute log-prob gradient, weight by return. Key insight: you can't backprop through stochasticity, but you can through log-probability.

$$\nabla J(\theta) = \mathbb{E}[\nabla \log \pi(a|s,\theta) \cdot G_t] \quad \leftarrow \text{Policy Gradient Theorem}$$
$$\theta \leftarrow \theta + \alpha \cdot \nabla \log \pi(a|s,\theta) \cdot G_t$$

Core Concept: Policy gradient methods directly optimize policy \(\pi_\theta(a|s)\) instead of learning value functions. Enables stochastic policies and continuous control.

$$\nabla J(\theta) = \mathbb{E}[\nabla \log \pi_\theta(a|s) \cdot Q(s,a)]$$
\(\nabla J(\theta)\):Gradient of objective function \(J\) with respect to policy parameters \(\theta\)
\(\pi_\theta(a|s)\):Parameterized policy; probability of action \(a\) given state \(s\), with parameters \(\theta\)
\(\nabla \log \pi_\theta(a|s)\):Score function; gradient of log probability. Gradient flows through stochastic policy
\(Q(s,a)\):Action-value estimate; controls variance of gradient. Can use returns or learned Q
\(\mathbb{E}[\cdot]\):Expectation over states and actions during rollout

Policy Gradient Theorem: gradient increases probability of high-value actions

REINFORCE:Uses full return \(G_t = \sum \gamma^k R_{t+k}\) as \(Q\) estimate; high variance
Log-Trick:\(\nabla \pi = \pi \nabla \log \pi\) allows backprop through stochastic policy
Advantage:\(A(s,a) = Q(s,a) - V(s)\) reduces variance without bias (baseline cancels)

Algorithm Variants:

REINFORCE
Monte-Carlo returns; simple but high variance
Actor-Critic
Critic estimates \(V(s)\); lower variance via advantage
Proximal Policy Optimization (PPO)
Clipped objective; stable large updates
Trust Region Policy Optimization (TRPO)
KL constraint on updates; theoretically principled
When to Use
REINFORCE
When simplicity is valued and sample efficiency is not critical. E.g., quick prototypes, episodic tasks with short horizons, NLP fine-tuning with SCST. Simple to implement but high variance -- add baseline to reduce it.
PPO (Proximal Policy Optimization)
Default choice for most continuous control and LLM alignment tasks. E.g., RLHF for ChatGPT, robotic locomotion, game AI. Clipped objective prevents catastrophic updates. Best stability-performance tradeoff in practice.
Actor-Critic (A2C/A3C)
When you need lower variance than REINFORCE with moderate complexity. E.g., real-time game playing, multi-agent environments. Critic provides per-step advantage estimates via GAE, reducing variance at cost of critic bias.
Log-probability trick is key: you can't backprop through stochasticity, but you can backprop through log(π)—differentiate the log, then multiply by the result.
👉 Try this: REINFORCE uses ∇_θ log π(a|s) · R (score function estimator). This works because ∇ log π = (∇π) / π—clever algebraic trick. High-variance (full trajectory return). Modern methods: A3C reduces variance with baseline (critic). Proximal Policy Optimization uses clipped objective instead.
Actor-Critic = actor (policy) makes decisions, critic (value) evaluates them. Critic reduces variance of policy gradients via advantage estimation.

Key Gotchas: REINFORCE high variance without baselines; Critic bias if poorly trained; High sample complexity; Local optima common; Off-policy corrections can explode.

Production Best Practices: Use Proximal Policy Optimization / Trust Region Policy Optimization for stability (clip/trust region updates); Actor-critic more sample-efficient; Generalized Advantage Estimation balances bias-variance; Monitor policy entropy; Test exhaustively (can exploit simulator); Continuous control is PG sweet spot.

import torch

                        return torch.softmax(self.net(x), dim=-1)

Core Concept: Multi-armed bandits isolate exploration-exploitation tradeoff. K arms with unknown reward distributions; no states, no sequential dependence.

$$\text{Regret}_T = \mathbb{E}\left[\sum R^*(a) - \sum R(a_t)\right] \text{ over } T \text{ rounds}$$
\(\text{Regret}_T\):Cumulative performance gap; sum of optimal rewards minus actual rewards. Lower = better exploration-exploitation balance
\(R^*(a)\):Reward of optimal arm in round; \(R^* = \max_a \mu_a\) (best expected payoff)
\(R(a_t)\):Actual reward from chosen arm a_t at round t
T:Total number of rounds; longer horizon → larger regret. Regret rate (regret/T) → 0 = good algorithm
\(\mathbb{E}[\cdot]\):Expectation over arm reward distributions and algorithm randomness
\(\epsilon\)-Greedy:Exploit best arm \((1-\epsilon)\); explore uniform \((\epsilon)\). Regret: \(O(\epsilon T + (1-\epsilon)K \log T)\)
Upper Confidence Bound (UCB):Q̂(a) + c√(ln T / n(a)) = empirical + uncertainty bonus. Regret: O(log T)
Thompson Sampling:Bayesian posterior; sample rewards, play best. Regret: \(O(\log T)\)

Regret Analysis:

ε-Greedy
\(O(\epsilon T)\): linear term is wasted; suboptimal
Upper Confidence Bound (UCB)
O(log T): exploration bonus shrinks; theoretically optimal
Thompson Sampling
O(log T): posterior concentrates; empirically fastest
When to Use
Thompson Sampling
Best default for most bandit problems. E.g., A/B testing, ad selection, clinical trials, recommendation ranking. Bayesian posterior naturally balances exploration/exploitation. Easy to explain to stakeholders. O(log T) regret.
UCB (Upper Confidence Bound)
When deterministic behavior is preferred and theoretical guarantees matter. E.g., resource allocation with strict audit requirements, systematic exploration. No randomness in action selection -- easier to debug and verify.
Contextual Bandits
When side information (context) is available per decision. E.g., personalized recommendations (user features), news article selection, dynamic pricing. LinUCB or neural contextual bandits leverage features for better decisions.
Lower bound on regret is \(\Omega(\log T)\). Only UCB and Thompson Sampling achieve this; ε-greedy is fundamentally suboptimal (linear waste term).
👉 Try this: Switch between ε-greedy (fixed exploration) and UCB. ε-greedy with ε=0.1 wastes 10% of budget on suboptimal arms (linear regret). UCB allocates adaptively—arms with high uncertainty get more trials. Cumulative regret log(T) vs T—huge difference at scale.
UCB = empirical mean + exploration bonus that shrinks as arm is played. Thompson = posterior samples naturally balance exploration (high uncertainty) and exploitation (posterior peaks).

Key Gotchas: ε-greedy regret is linear in T (bad for long horizons); UCB constant c requires tuning; Thompson Sampling needs Bayesian inference; Contextual bandits are realistic but harder.

Production Best Practices: Use Thompson Sampling for A/B testing/ad serving (auto-balances, easy to explain); UCB for explicit control; Avoid ε-greedy unless trivial; Contextual variants (LinUCB, ContextTS) for recommendations; Monitor arm frequencies; Cold-start: all arms equally unknown.

import numpy as np
                
                n_arms, n_rounds = 5, 10000
                true_means = np.random.randn(n_arms)
                counts = np.zeros(n_arms)
                values = np.zeros(n_arms)
                
                for t in range(1, n_rounds + 1):
                    if t <= n_arms:
                        arm = t - 1  # pull each arm once
                    else:
                        ucb = values + np.sqrt(2 * np.log(t) / counts)
                        arm = np.argmax(ucb)
                    reward = np.random.randn() + true_means[arm]
                    counts[arm] += 1
                    values[arm] += (reward - values[arm]) / counts[arm]
                
                print(f"Best arm: {np.argmax(values)}, True best: {np.argmax(true_means)}")

UCB balances exploration (uncertainty bonus) and exploitation (empirical mean).

$$\mathcal{L} = -\mathbb{E}[\min(r_t \cdot \hat{A},\; \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \cdot \hat{A})] - \beta \cdot \text{KL}(\pi_{\text{new}} \| \pi_{\text{ref}})$$

Core Concept: Reinforcement Learning from Human Feedback aligns large language models with human preferences via reinforcement learning. No explicit reward labels; instead, learn from preference comparisons.

$$\mathcal{L} = -\mathbb{E}[\min(r_t \cdot \hat{A},\; \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \cdot \hat{A})] - \beta \cdot \text{KL}(\pi_{\text{new}} \| \pi_{\text{ref}})$$
Loss:Objective to minimize (negative because we maximize reward and penalize divergence)
\(r_t\):Probability ratio \(= \pi_{\text{new}}(a_t|s_t) / \pi_{\text{ref}}(a_t|s_t)\); >1 means new policy more likely than reference
\(\hat{A}\):Advantage estimate; reward_model(s_t,a_t) - baseline. Positive = good actions, negative = bad
\(\text{clip}(r_t, 1-\epsilon, 1+\epsilon)\):Clips ratio to prevent too-large policy changes. Typical \(\epsilon = 0.2\); prevents instability
\(\beta\):KL penalty weight \(\in [0.05, 1]\); higher \(\beta\) = stricter adherence to reference. Typical: 0.1-1.0
\(\text{KL}(\pi_{\text{new}} \| \pi_{\text{ref}})\):Kullback-Leibler divergence; measures distributional distance. Prevents mode collapse to reward gaming
\(\pi_{\text{ref}}\):Reference policy (usually supervised fine-tuned model); keeps new policy from drifting too far

Proximal Policy Optimization with KL penalty: maximize reward while staying close to reference model

Pipeline Step 1:Supervised fine-tuning on human examples to bootstrap quality
Pipeline Step 2:Collect human preferences (A vs B comparisons)
Pipeline Step 3:Train reward model via Bradley-Terry logistic regression
Pipeline Step 4:Reinforcement learning fine-tune language model on reward signal with KL regularization

Bradley-Terry Model:

$$P(y \succ x) = \sigma(R(x,y))$$
\(P(y \succ x)\):Probability that response y is preferred over response x; ∈ [0,1]
\(\sigma(\cdot)\):Sigmoid activation \(\sigma(z) = \frac{1}{1+e^{-z}}\); maps real numbers to [0,1]
\(R(x,y)\):Scalar reward difference from learned reward model; \(R(x,y) = r_\theta(y) - r_\theta(x)\)
\(r_\theta\):Learned reward model (neural network); outputs scalar reward for text pair

Preference probability = sigmoid of reward difference; trainable via cross-entropy

When to Use
RLHF (Full Pipeline)
When aligning LLMs to human preferences at scale. E.g., ChatGPT, Claude, Gemini. Three-stage: SFT then reward model then PPO. Gold standard for instruction-following and helpfulness. Expensive but most proven approach.
DPO (Direct Preference Optimization)
When you want alignment without training a separate reward model. E.g., smaller-scale fine-tuning, faster iteration cycles. Directly optimizes policy from preferences. Simpler pipeline, fewer hyperparameters, but less flexible than full RLHF.
Constitutional AI Instead
When human annotation is too expensive or slow to scale. E.g., rapid safety iteration, principle-based alignment. Model self-critiques using constitution principles. No human labelers needed after principles are defined.
KL penalty is crucial: without it, the language model mode-collapses to trivial outputs that maximally exploit the reward model. KL keeps it close to the supervised fine-tuning baseline.
👉 Try this: Set KL coefficient = 0 (reward-only optimization). The language model exploits reward model, outputs degenerate (e.g., repeat "yes" forever if human likes agreement). KL=0.1: balanced Pareto frontier. KL=1.0: plays it safe, boring outputs. Find your coefficient based on alignment criticality.
Reinforcement Learning from Human Feedback = teach model to please humans. Bradley-Terry = learn what humans like. Proximal Policy Optimization + KL = apply reinforcement learning while staying sensible. Three-stage dance between supervised fine-tuning, reward, and policy.

Key Challenges: Reward hacking (exploiting model weaknesses); Expensive annotation with labeler bias; Reward model overfits or goes off-distribution; KL tuning critical; Computational cost high.

Production Best Practices: Multiple annotators, measure agreement; Diverse preference data; Monitor reward model accuracy on held-out; Use β ≈ 0.1-1.0; Test on multiple benchmarks; Iteratively improve reward model as language model improves; Static reward model becomes suboptimal over time.

import torch
                import torch.nn as nn
                
                def ppo_loss(new_log_probs, old_log_probs, advantages, clip_eps=0.2, beta=0.1, ref_log_probs=None):
                    ratio = torch.exp(new_log_probs - old_log_probs)
                    clipped = torch.clamp(ratio, 1 - clip_eps, 1 + clip_eps)
                    policy_loss = -torch.min(ratio * advantages, clipped * advantages).mean()
                    kl_penalty = 0.0
                    if ref_log_probs is not None:
                        kl_penalty = (torch.exp(new_log_probs) * (new_log_probs - ref_log_probs)).mean()
                    return policy_loss + beta * kl_penalty
